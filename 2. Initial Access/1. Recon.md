##  Recon-ng

- [Recon-ng](https://github.com/lanmaster53/recon-ng)
- Recon-ngは作戦やOSINTタスクに役立つ様々な断片的な情報を見つけるために使用する
- 収集されたデータはすべて、ワークスペースに関連するDBに自動保存される。
- 例えば、後で行うポートスキャンのためのホストアドレスを発見したり、フィッシング攻撃用の連絡先メールアドレスを収集したりする

- Recon-ngを起動する
```zsh
recon-ng
```
- プロンプトが`[recon-ng][default] >`で成功

- ここから、使用するインストール済みモジュールを選択する。ただし、recon-ngを初めて実行する場合は、必要なモジュール（複数可）をインストールする必要がある

###### 流れ

1. プロジェクト用のワークスペースを作成する  
2. データベースに開始情報を挿入する  
3. マーケットプレイスでモジュールを検索し、インストールする前にそのモジュールについて学習する
4. インストールされているモジュールをリストアップし、1つロードする  
5. ロードしたモジュールを実行する

#### ①ワークスペースの作成

- 作成
```zsh
workspaces create [Workspace name]
```
- 作成したワークスペースのプロンプトに自動で変わる

- 起動時に作成したワークスペースを使用する
```zsh
recon-ng -w [Workspace name]
```

#### ②DBにデータを挿入する

- Reconでは、1つの情報から始めて、それを新たな情報に発展させていくことになる。
- 例えば、企業名から調査を開始し、それをもとにドメイン名、連絡先、プロフィールを発見する。

- ターゲットのドメイン名わかっていて、それを現在アクティブのワークスペースのDBに挿入したい場合。
- DBのテーブル名を確認する
```recon-ng
db schema
```
![ 100](../../../画像ファイル/Pasted%20image%2020230609151806.png)
- テーブルの中身を閲覧する
```recon-ng
db query select * from domains
```
- データを挿入する
```recon-ng
db insert domains
domain (TEXT): [domain name]
notes (TEXT):
```

#### ③Recon-ng Marketplaceでモジュールの検索と理解

- ドメイン名を入手している状態だとして、次のステップは、ドメインを他のタイプの情報に変換するモジュールを検索すること
- マーケットプレイスから適切なモジュールを検索する

##### マーケットプレイスの使い方に関連するコマンド

- `marketplace search`: すべての利用可能なモジュールを表示する
- `marketplace search [Keyword]`: キーワードで利用可能なモジュールを検索
- `marketplace info [Module]`: 当該モジュールに関する情報を入手
- `marketplace install [Module]`: 指定されたモジュールをRecon-ngにインストール
- `marketplace remove [Module]`:

#####  モジュールについて

- discovery, import, recon そして reportingなどのカテゴリがあり、さらに情報変換の種類によってサブカテゴリがある。
- サブカテゴリがどのような情報を入手するか示している。
- `whoxy_whoisn`のように、=="K"の列に`*`が付いているものはキーが必要==で、関連するサービスを利用するためのキーがない限り、このモジュールは使用できない。詳細は→[1. Recon](#Keys)
- =="D"列に`*`が付いているものはサードパーティのPythonライブラリが必要==。依存関係があるので
 ![](../../../画像ファイル/Pasted%20image%2020230609152847.png)

##### 手順

- モジュールを検索
```recon-ng
marketplace search [keyword]
```
- モジュールについて調べる
```recon-ng
marketplace info [Module]
```
- モジュールをインストール
```recon-ng
marketplace install [Module]
```
- モジュールはフルパスではなくても、最下層の名前だけ指定すればいい。上の画像だと、`cencys_companiese`だけでいい。

#### ④モジュールをロードし実行する

- インストールされているすべてのモジュールをリストアップ
```recon-ng
modules search
```
- 特定のモジュールをメモリにロードする
```recon-ng
modules load [Module]
```
 ![ 300](../../../画像ファイル/Pasted%20image%2020230609154912.png)
- ロードしたモジュールの詳細を表示
```recon-ng
list
```
- ロードしたモジュールに設定できるオプションをリストアップ
```recon-ng
options list
```
- オプションの値を設定
```recon-ng
options set [option] [value]
```
- 実行する
```recon-ng
run
```
- Ctrl+Cでロードしたモジュールをunloadする

#### Keys

- "K"列に`*`がある一部のモジュールは、各サービスAPIのキーがないと使用できない。
- keyをリストアップ
```recon-ng
keys list
```
- keyの追加
```recon-ng
keys add [KEY_NAME] [KEY_VALUE]
```
- keyの削除
```recon-ng
keys remove [KEY_NAME]
```

## Maltego

### 基本

- [Maltego](https://www.maltego.com/)
- 用途はRecon-ngと一緒だがより視覚的
- マインドマップとOSINTを融合させたアプリケーション。
- 最初に入手した情報、例えば、ドメイン名、会社名、人名、メールアドレスなどから始める。
- そして、この情報の断片をトランスフォームしていく
- Maltegoで収集した情報は、後のステージで利用することができる。例えば、収集した会社情報、連絡先名、メールアドレスは、非常に合法的に見えるフィッシングメールを作成するために使用することができる
- Maltegoのグラフ上の各ブロックは、エンティティとして考える。エンティティは、それを説明する値を持つことができる。
- Maltegoの用語では、トランスフォームとは、特定のエンティティに関連する情報を取得するためにAPIに問い合わせるようなコードの一部のこと。
- ![](../../../画像ファイル/Pasted%20image%2020230609170039.png)
- ==Maltegoで利用可能なトランスフォームの中には、ターゲットシステムに積極的に接触するものもある==ので、Passive Reconにとどめたい場合は、そのトランスフォームがどのように機能するかを事前に知っておく
- すべてのトランスフォームは、いくつかの新しい値を導き出すかも。

### 手順例

  ![ 400](../../../画像ファイル/Pasted%20image%2020230609165839.png)

- New > Entity Paletteに"DNS name"と入力し、エンティティをダブルクリックしてcafe.thmredteam.comと入力する
- 使用するトランスフォームに基づいて新しい種類のエンティティを取得する。
- たとえば、エンティティを右クリック > All Transformsをクリック > 検索タブに"IP"と入力 > To IP Address \[DNS\]をクリックする
 ![ 300](../../../画像ファイル/Pasted%20image%2020230609170510.png)
 - IPアドレスを表示してくれる。
 ![ 300](../../../画像ファイル/Pasted%20image%2020230609170554.png)

- IPアドレスの一つを右クリックしてTo DNS Name from passive DNS (Robtex)というトランスフォームを選択する
- ![ 300](../../../画像ファイル/Pasted%20image%2020230609171313.png)

- このようにして、ターゲットマシンの情報収集を効率的にわかりやすく行える。
