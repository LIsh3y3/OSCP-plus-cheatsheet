### Information disclosureとは

- 機密性の高い情報を意図せず開示してしまうこと
- あらたなエクスプロイトの糸口になることも多い。

###### 基本的なinformation disclosureの例

- [3. 一般的な情報開示ソース](3.%20一般的な情報開示ソース.md)

- `robots.txt`ファイルやディレクトリのリスト
- ライブラリやCMSのバージョン情報
- 一時的なバックアップに対するアクセスでソースコードを流出
- Errorメッセージによるテーブル名やカラム名の表示 [4-2. Error-based SQLi](../SQLi/4.%20Blind%20SQLi/4-2.%20Error-based%20SQLi.md)
- APIキー、IPアドレス、DBのクレデンシャルなどをソースコードに記載
- 異なるアプリケーションの動作による情報窃取 [2. パスワードログインの脆弱性](../Authentication/2.%20パスワードログインの脆弱性.md#Usernameの列挙)

### Information disclosureへの注意点と対策

- [ ] 情報漏えいの有無を単体の問題としてではなく、漏えいした情報の影響と悪用可能性に主眼を置く
- [ ] どのような情報が機密情報とされるのか、組織全体で共有しておくこと
- [ ] QA(Quality Assesment)やビルドプロセスにおいて、情報漏洩につながるコードをチェックする
- [ ] エラーメッセージに違いを出さない。なるべく一般的なエラーメッセージを使用する
- [ ] 本番環境でデバッグまたは診断機能が無効になっていることを再確認
- [ ] 資産管理で構成情報を把握しておくこと。必要のないサービスは無効化
