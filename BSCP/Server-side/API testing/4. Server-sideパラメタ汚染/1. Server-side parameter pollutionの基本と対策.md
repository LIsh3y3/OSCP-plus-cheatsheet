
## Server-sideパラメータ汚染とは

![[Pasted image 20231219163527.png]]

- システムには、ユーザからは直接アクセスできない内部APIが存在する場合がある
- Webアプリが内部APIのリクエストを発行する時のこと
- ユーザの入力を適切にサニタイジングせずにリクエストに組み込むことで起きる
	- 考え方が少し似てる：([1. SSRFの基本](../../SSRF/1.%20SSRFの基本.md))
- (別名HTTP parameter pollution。厳密には意味合いがことなる)

###### 可能なこと

- 既存のパラメタの上書き
- アプリケーションの動作を操る
- データへの不正アクセス

##### 対象のパラメタ

- 基本全て
- クエリパラメタ
- formフィールド
- ヘッダ
- URLパスのパラメタ

## Server-side parameter pullutionの対策

- [ ] エンコードする必要のない文字を許可リストに登録する
- [ ] 許可リスト以外の入力はサーバからリクエストを送る前にエンコードする
- [ ] すべての入力が意図されたフォーマットと構造か検証する
