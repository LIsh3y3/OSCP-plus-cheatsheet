
## ①APIエンドポイントの特定

### ドキュメントの活用

###### APIの公開ドキュメントを確認する

 - APIが外部の開発者による使用の目的で公開している
 - 機械が読めるJSON/XML/YAMLの形式もある
 - 機械が読める形式のドキュメント閲覧方法
	 - Burp Scanner(Pro)
	 - BApp: [📕](../../Misc/📕.md#OpenAPI) Parser([リンク](https://portswigger.net/bappstore/6bf7574b632847faaaa4eb5e42f1757c))
	 - [Postman](https://www.postman.com/)や [SoapUI](https://www.soapui.org/)などの有料ツール

###### 非公開のAPIドキュメントを探し当てる

 - Burp Scanner(pro)
-  [api_wordlist](https://github.com/chrislockard/api_wordlist)のcommon_paths.txtを使用しファジング

### 隠されたAPIエンドポイントの発見

###### 基本的な方法

- Burp Scanner(pro)を使ってアプリケーションをクロールする
- APIと関連してそうなエンドポイントを探す(`/api`等)

- JavaScriptファイルも閲覧し、APIエンドポイントへの参照やパラメタがないか確かめる
	- [JS Link Finder](https://portswigger.net/bappstore/0e61c786db0c4ac787a08c4516d52ccf)(pro)を使用し、JavaScriptファイルを手動で検証することも可能

###### ファジング

- APIエンドポイントの[api_wordlist](https://github.com/chrislockard/api_wordlist)を使用してTurbo Intruder
- `PUT /api/user/update`
- `update` -> `delete`, `add`など

##### Tips

- エンドポイントを特定したらベースのpathも必ず閲覧すること
- 例：`/api/example/v1/users/123`
	- `/api/example/v1`
	- `/api/example`
	- `/api`

- `http://examle/ui`も確認してみる。APIのドキュメントが掲載されている可能性がある

## ②APIエンドポイントとのインタラクト

- APIとやりとりして挙動やErrorを確認しながら、以下のことを確認する

### 使用可能なHTTPメソッドの特定

- APIエンドポイントは他のページと異なるHTTPメソッドが使用可能かもしれない
- すべてのHTTPメソッドを検証する
	- Intruderで"HTTP verbs"(pro)リストを使用。(proじゃないので自作したものを用意済)

###### ⚠️注意
- HTTPメソッドテスト時は重要でないオブジェクトを対象にテストすること
	- 例：`DELETE /api/task/not_important`: 重要でないタスクオブジェクトをDeleteする

###### HTTPメソッド例

|メソッド|意味|
|--|--|
|PATCH|部分的な変更をリソースに適用する|
|OPTIONS|リソース上で使用できるHTTPメソッドの種類に関する情報を取得|
|PUT|リクエストしたペイロードでリソースのすべてを置き換える([3. (実践無)RCEなしでの悪用&PUTメソッドのファイルアップロード](../File%20upload%20vuln/3.%20(実践無)RCEなしでの悪用&PUTメソッドのファイルアップロード.md#PUTメソッドによるファイルアップロード))|

- [Mozilla:HTTP request methods](https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods)

### 使用可能なContent-Typeの特定

###### アプリが意図してない`Content-Type`を使用することで以下を可能にする

- Errorによる役立つ情報の開示
- 欠陥のある防御のバイパスや処理ロジックの悪用(XMLを使用している時はインジェクションされ易い)
	- [1. File upload脆弱性の基本と対策](../File%20upload%20vuln/1.%20File%20upload脆弱性の基本と対策.md)
	- [1. XXE injectionの基本と対策](../XXE%20injection/1.%20XXE%20injectionの基本と対策.md)
	- [2. パスワードログインの脆弱性](../Authentication/2.%20パスワードログインの脆弱性.md#1回のリクエストで複数のパスワード推測実践)
	- [3. NoSQL operator injection](../NoSQLi/3.%20NoSQL%20operator%20injection.md)

###### 使用可能なContent-Typeの特定方法

1. `Content-Type`ヘッダを変更する
2. リクエストボディを`Content-Type`に合わせた形に変更する
	- BApp: Content type converterを使用する
