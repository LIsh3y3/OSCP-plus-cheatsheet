
### 基本

- クエリ文字列を直接注入せずとも、アプリケーションによってクエリと解釈される文法や文字で間接的にクエリを注入することで、SQLiペイロードのWAFフィルタリングを回避することが可能
- 例えば、JSONやXMLフォーマットの入力を受け取ってこれをDBのクエリとして使用するアプリケーションがある
- 例えば、以下のXMLベースのSQLインジェクションは、SELECTのS文字をエンコードするためにXMLエスケープシーケンスを使用している
```xml
<stockCheck>
	<productId>123</productId>
	<storeId>999 &#x53;ELECT * FROM information_schema.tables</storeId> </stockCheck>
```
- これはSQLインタプリタに渡される前にサーバーサイドでデコードされる。

### 実践

- 解答見た[filter bypass via XML encoding](https://portswigger.net/web-security/learning-paths/sql-injection/sql-injection-in-different-contexts/sql-injection/lab-sql-injection-with-filter-bypass-via-xml-encoding#)
1. アプリケーションの機能の中で、XMLで実行しているものを見つける
```xml
<?xml version="1.0" encoding="UTF-8"?>
<stockCheck>
	<productId>1</productId>
	<storeId>1</storeId>
</stockCheck>
```

2. バックエンドシステムと通信しているリクエスト(GET=product 1やPOST)をRepeaterへ送っておくこと

3. UNION攻撃を行う[2. SQLi UNION攻撃](2.%20SQLi%20UNION攻撃.md)
```sql
1 UNION SELECT NULL--
```
- `Attack detected`と表示される

4. Hackvertorで元々のクエリの値も含め、追加したクエリをhex_entitiesでエンコードする。
- HTML encodeでも可能だが、より難読化するうえに高可読性はhex_entities

- Requestを右クリック-> Extension -> Hackvertor -> Encode -> hex_entitiesを選択
- hex_entities
	- HTMLやXMLでエンコードするときに使用する
	- XMLエンコードだけではフィルタリングに引っかかる場合、hex(16進数)でエンコードする
```http
<storeId><@hex_entities>
	1 UNION SELECT NULL--<@/hex_entities>
</storeId>
```
- NULLと表示される
- **反応の違いを必ず確認**

5. カラム数を特定し、データ型も特定
- [2. SQLi UNION攻撃](2.%20SQLi%20UNION攻撃.md#準備)

6. [3. DBの構成情報の探索](3.%20DBの構成情報の探索.md)に取り組む

7. 一つのカラムで文字列型、　RDB・テーブル・カラム名を把握したので[2. SQLi UNION攻撃](2.%20SQLi%20UNION攻撃.md#1つのカラムで複数のデータを抽出する方法)を行う
