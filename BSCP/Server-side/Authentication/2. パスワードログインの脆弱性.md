## ブルートフォース攻撃とは

###### ユーザ名のブルートフォースの脆弱性の例

- ログインせずに閲覧できる情報にユーザ名が含まれている
- 推測が容易な文字列を使用している
- メールアドレスを直接ユーザ名としている
- レスポンスでメールアドレスを開示している。
	- たまに、レスポンスに管理者やITサポートなどの高特権ユーザーのメールアドレスが含まれていることがある。

###### パスワードのブルートフォースの脆弱性の例

- 厳しいパスワードポリシが設定されていても、「人の脆弱性」を利用すると効率的にブルートフォースが可能。

###### 人の脆弱性

- 覚えやすく楽なパスワードを設定する。
- 自分の覚えているパスワードを無理やりパスワードポリシーに適用させるように設定する。
	- 例: `mypassword` -> `Mypassword1!` or `Myp4$$w0rd`

- 定期的な変更を要求するシステムに対しても、簡単な変更しかしない
	- 例: `Mypassword1!` -> `Mypassword2`

## Usernameの列挙

###### ブルートフォース以外での列挙方法

- ログインフォームで、有効なユーザ名を入力した時とそうでないときのError messageの差を利用し、有効なユーザ名をリストアップする
- Registerフォームで「すでに使用されているユーザ名です」をリストアップする
- 応答時間の差異を強調するため、過度に長い値を入力する

###### ブルートフォース中の注目点

- Status Code
- Error message:
	- 見た目がほとんど同じでもResponceの空白の数が異なったり、誤字脱字があると存在するかしないかの区別につながる。
- Responce time:
	- レスポンスタイムが異なる＝推測されたユーザー名が正しいかもしれない兆候と考える
	- 過度に長いパスワードを入力することで、レスポンスタイムの差異を強調する

## アカウントロックを回避しながらのブルートフォースステップ

- レスポンスタイムの違いでusernameを列挙する場合

1. `X-Forwarded-For`を使用して送信元IPアドレスを詐称し、アカウントロックを回避する
2. 100文字くらいのパスワードを設定する。

- ==⚠️==：IntruderのMax fraction digitsを0にする：整数に限定したいので
```http
POST /login HTTP/2
...
X-Forwarded-For: §§

username=§§&password=peterpeter... //100peter
```

###### ブルートフォース対策

1. IPアドレスブロック -> [2. パスワードログインの脆弱性](2.%20パスワードログインの脆弱性.md#1.IPアドレスブロックのバイパス)
2. アカウントロック-> [2. パスワードログインの脆弱性](2.%20パスワードログインの脆弱性.md#2.%20アカウントロックのバイパス)
3. user rateの制限 -> [2. パスワードログインの脆弱性](2.%20パスワードログインの脆弱性.md#3.%20User%20rate制限のバイパス)

### 1.IPアドレスブロックのバイパス

- ログイン成功によりIPアドレスブロックのための失敗回数が無効化されることがある
- 自身のアカウントのクレデンシャルでログインできるようワードリストに定期的に自身のクレデンシャルを挿入する。

###### 0. IPアドレスブロックの条件を確認する

- 何回のログイン試行でブロックされるか？
- 失敗回数のリセット条件は？
	- この例では、3回目のログイン試行でブロックされるとする

###### 1. IPアドレスブロックの条件を回避するwordlist作成のためのスクリプト

- usernameをcarlosとwienerで繰り返させる
- password listに2行ごとに正規のパスワードを注入する(3回目でブロックされるので)
```python
print("######## The following are the usernames:#######")
for i in range(150):
	if i % 3:
		print("carlos")
	else:
		print("wiener")
print("######## The following are the passwords:########")
with open('[path to passwords list]', 'r') as f:
	lines = f.readlines()

i = 0
for pwd in lines:
	if i % 3:
		print(pwd.strip('\n'))
	else:
		print("peter")
		print(pwd.strip('\n'))
		i = i+1
	i = i+1
```

###### 3. 出力されたリストを使用してIntruder

- payload settingsでmaximum concurrent requestを1にすること
- 必要であれば`X-Forwarded-For`を使用する

### 2. アカウントロックのバイパス

###### アカウントロックの弱点

- [クレデンシャルスタッフィング](https://www.nec-solutioninnovators.co.jp/ss/insider/security-words/52.html)
- パスワードスプレー攻撃やリバートブルートフォース
- 攻撃者は特定のアカウントではなく、どれでもいいからアカウントにアクセスしたいので、アカウントロックはあまり意味ない。
###### Tips

- アカウントがロックされたというエラーメッセージがユーザ名の列挙につながるかも。ロックする対象は存在するアカウントのみかも。
	- あえてロックをしたいなら、IntruderのNull Payloadを使用する

#### アカウントロックのバイパス概要

1. 有効そうなusernameの候補リストを作成する
	- [2. パスワードログインの脆弱性](2.%20パスワードログインの脆弱性.md#Usernameの列挙)もしくは一般的なusernameを元に作成

2. アカウントロック回数分の数で、短いパスワードリストを作成する：
	- 少なくとも1人のユーザは使用してそうなパスワード
	- 3回のログイン失敗でアカウントロックになるなら、3つのパスワードで1つのリストを作成する

3. Inturderを使い、ユーザごとにステップ2で作成したパスワードリストでブルートフォース
	- アカウントロックをトリガーすることなく、任意のアカウントを侵害できる可能性が高まる

### 3. User rate制限のバイパス

- 一定時間内でログイン試行が一定回数繰り返されたときに、IPアドレスブロックされる

###### ロック解除の基本的な方法

- 一定時間経過後に自動的に解除
- 管理者による手動解除
- CAPTCHAに成功した後、ユーザが手動で行う

#### User rate制限のバイパス

- 見かけ上のIPアドレスをスプーフィングする
- 1回のリクエストで複数のパスワードを推測する

#### JSONによる1回のリクエストで複数のパスワード推測実践

![[Pasted image 20231123164016.png | 300]]
1. `Content-Type: application/json`に変更する
2. ログインリクエストをRepeaterへ
3. 複数のパスワードを送信する
```json
{
	"username":"carlos",
	"password": [
		"123456",
		"password",
		...
		"moscow"
	]
}
```

- 上記の形(`"password",`)にpasswordlistを変更するためのスクリプト
```python
with open('password_list.txt', 'r') as f:
	lines = f.readlines()

i = 0
for pwd in lines:
	print("\"" + pwd.strip('\n') + "\",")
	i = i + 1
```
