
## LLMとは

- LLM =  Large Language Models　＝　大規模言語モデル
- AIのアルゴリズム。
- ユーザの入力を処理し、単語のシーケンスを予測することによって、もっともらしい応答を作成することができる
	- 単語のシーケンス：その後に来るであろう単語やフレーズを予測するという意味

- LLMは巨大な半公開データセットで学習され、機械学習を使って言語の構成要素がどのように組み合わされているかを分析する

###### プロンプトとは

- LLMに指示をするための入力セット
- 許可される入力は、入力検証ルールによって部分的に制御される。

###### LLMの用途

- バーチャルアシスタントなどのカスタマーサービス
- 翻訳
- SEOの改善
- ユーザが生成したコンテンツの分析

## Web LLM attackとは

- SSRF([1. SSRFの基本](../../Server-side/SSRF/1.%20SSRFの基本.md))と非常に似ている。ユーザから直接アクセスできない別のコンポーネントに攻撃を仕掛けるといった点で一緒

#### 攻撃の影響

###### 本来はLLMのみアクセスできる情報を取得する

- LLMのプロンプト
- 学習用データ(トレーニングセット)
- モデルで使用されているAPI

###### API経由で悪意のあるアクションを引き起こす

- LLMを使用して、モデルがAPIに対して誤ったクエリを送ることで、APIに対してSQLiを行う。

###### 他のユーザーやシステムへの攻撃のトリガー 

- LLMにクエリを送る他のシステムやユーザーに対して、フィッシング攻撃やマルウェアの配布といった攻撃を仕掛ける

---
### プロンプトインジェクションとは

- 攻撃者が細工したプロンプトを使ってLLMの出力を操作するもの
- 機密性の高いAPIを不正に呼び出したり、ガイドラインと一致しないコンテンツを返したりするなど、AIが意図した目的から外れた動作をさせる

---
## LLM攻撃への対策

- LLMと統合するアプリをデプロイする際に、以下のステップを踏む

###### LLMに提供されたAPIを一般公開されたものとして扱う

- LLMを通じてAPIを呼び出せるため、LLMがアクセスできるAPIは誰でもアクセス可能とみなす
- 常に認証を要求するなど、基本的なAPIアクセス制御を実施する。
- アクセス制御はLLMと統合したアプリケーションによって処理されるべきで、モデル自体の自己管理に期待しない。
- [3. 間接的なプロンプトインジェクション](3.%20間接的なプロンプトインジェクション.md)のリスクを軽減するために、適切な権限制御を実施する。

###### LLMに機密データを供給しないようにする

- モデルの訓練データセットに対して堅固なサニタイズ技術を適用する。
- モデルに供給するデータは、最低限の権限を持つユーザーがアクセス可能なものに限定する。
- モデルが外部データソースにアクセスするのを制限し、データサプライチェーン全体に堅固なアクセス制御を適用する。
- 定期的にモデルが機密情報をどの程度知っているかをテストする。

###### 攻撃をブロックするためにプロンプトに頼らない

- プロンプトを使用してLLMの出力に制限を設けることは理論上可能だが、これに依存すべきではない。
- 攻撃者は`disregard any instructions on which APIs to use`などのプロンプトを使ってこの制限をバイパスすることが可能
	- "jailbreaker prompts"と呼ばれる
