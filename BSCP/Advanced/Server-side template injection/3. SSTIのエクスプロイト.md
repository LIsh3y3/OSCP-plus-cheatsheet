## Explore

- もし[HackTricks: SSTI](https://book.hacktricks.xyz/pentesting-web/ssti-server-side-template-injection)で実行可能なエクスプロイトを見つけられなかったら...
- 利用可能なオブジェクトをすべて見つけること
- もし直接的なRCEができなくても、SSTIを利用したパストラバーサルやその他のエクスプロイトで機密データにアクセスできる可能性がある

#### Webサイトには2種類のオブジェクトがある

###### テンプレートに組み込みのオブジェクト(self / environment)

- これはテンプレートエンジンが持つすべてのオブジェクト、メソッド、属性を[名前空間](https://e-words.jp/w/%E5%90%8D%E5%89%8D%E7%A9%BA%E9%96%93.html)のように含んでいるオブジェクトのこと
- このオブジェクトを利用することで、現在のテンプレート環境内で利用可能な全ての変数や関数などをリストアップし、探索(Explore)することができる
- それにより、テンプレートエンジン内に閉じられた環境の内部情報にアクセスしたり、何ができるかを試したりすることができる

- 例えばJava-basedテンプレートエンジンでは、以下の様なenvironmentを使用することで全ての利用可能な変数を列挙できる
```
${T(java.lang.System).getenv()}
```

###### 開発者が作成したWebサイト固有のカスタムオブジェクト

- テンプレートに組み込まれたものでなく、Web開発者によって提供されたサイト特有のオブジェクト
- 機密情報や悪用可能なメソッドを含んでいる可能性あるので、組み込みのオブジェクトよりも危険。
- 
- 同じWebサイトでも、異なるテンプレートで異なるカスタムオブジェクトを使用している可能性がある
- そのため、悪用前に、それぞれの異なるテンプレート上でオブジェクトがどのように動作するのか学習する必要がある

- 変数名をブルートフォースするため、[SecLists](https://github.com/danielmiessler/SecLists/blob/25d4ac447efb9e50b640649f1a09023e280e5c9c/Discovery/Web-Content/burp-parameter-names.txt)もしくはBurp Pro editiionで利用可能なワードリストを使用する

### ステップ

###### ①利用可能なオブジェクトを見つける

- すべてのオブジェクトを列挙できる組み込みの方法を調べる
	- テンプレートエンジン名　+　SSTI
	- 運が良ければこの時点で悪用可能なエクスプロイトを発見

- すべてのオブジェクトを列挙できる組み込みの方法がなければ、[3. SSTIのエクスプロイト](3.%20SSTIのエクスプロイト.md#Read)にて変数名の参照方法のシンタックスを学習し、[SecLists](https://github.com/danielmiessler/SecLists/blob/25d4ac447efb9e50b640649f1a09023e280e5c9c/Discovery/Web-Content/burp-parameter-names.txt)もしくはBurp Pro editiionで利用可能なwordlistでブルートフォースする(↓例)
```
{{ %s }}
```

###### ②怪しいオブジェクトについてドキュメントで調査する

1. テンプレートエンジン名 + documentation　で検索
2. Documentation内の検索バーなどに、怪しいオブジェクト名を入力

###### ③RCE、もしくはSSTIを利用した情報窃取を実行する

- 例えば、`settings`というオブジェクトがあり、`secret_key`という情報を窃取したい場合
```
{{ settings.SECRET_KEY }}
```
