## Detect

0. 反映箇所を明らかにする。
	- エントリーポイントに入力した値の反映箇所が、==入力ページと異なるページの場合==もあるので(持続型XSSの考えに近い)

1. エントリーポイントに特殊文字を注入しファジングする
```
${{<%[%'"}}%\
```
- 通常と異なる反応(エラーなど）があったら、SSTIの可能性アリ。反応がなくても[2. SSTIの検出とテンプレートエンジンの特定](2.%20SSTIの検出とテンプレートエンジンの特定.md#Contextの特定)へ
- Lucky: テンプレートエンジンが表示 -> エクスプロイトへ

**重要**
- エントリーポイントは本来の正規の値の後ろの場合もある(↓例)
```
blog-post-author-display=user.nickname
```
	↓
```
blog-post-author-display=user.nickname${{<%[%'"}}%\
```


### Contextの特定

#### Plaintext context

反映箇所が平文のケース(例：`render('Hello ' + username)`)

2. テンプレートエンジンごとに異なるシンタックスで数学演算を行う(以下==1個ずつ試す==)
```
{{7*7}}
${7*7}
<%= 7*7 %>
${{7*7}}
#{7*7}
*{7*7}
```
- 出力箇所が入力ページと別のページになる場合もあるので1つずつ

###### 具体例

- テンプレートが以下のようなコードを持つとき
```
render('Hello ' + username)
```
- 数学演算を注入
```
http://TARGET_NET/?username=${7*7}
```
- もし出力に`Hello 49`と含まれていれば、これはサーバサイドで評価されていることを示す

#### Code context

反映箇所がコード中のケース(↓例)
```js
greeting = getQueryParameter('greeting')
engine.render("Hello {{"+greeting+"}}", data)
```

###### 3. XSS脆弱性ではないことを確認する

- 単純に任意のHTMLを注入する
```
[parameter]=[有効なvalue値]<tag>
```
- XSS脆弱性ではない場合の反応：
	- 空白のエントリ
	- エンコードされたtag
	- エラーメッセージ

###### 4.SSTI脆弱性であることを確認する

- 「テンプレート構文でステートメントから抜け出し」、任意のHTMLを注入する
```
[parameter]=[有効なvalue値]}}<tag>
```
- SSTIの脆弱性アリの場合
	- 出力が任意のHTMLとともに正しくレンダリングされる

- 脆弱性ナシの場合
	- エラーや空白
	- [2. SSTIの検出とテンプレートエンジンの特定](2.%20SSTIの検出とテンプレートエンジンの特定.md#Identify)でテンプレートエンジンを特定した後、もう一度正しいシンタックスで再度試す

##### 基本概念と具体例

###### デフォルトの動作

- 例：ユーザ入力がテンプレート式の中に置かれるコード
```php
greeting = getQueryParameter('greeting')
engine.render("Hello {{"+greeting+"}}", data)
```
- URL
```
http://TARGET_NET/?greeting=data.username
```
- `Hello Carlos`と表示される

###### XSSではなくSSTIであることを確かめる

- XSSでないことを検証
```
http://TARGET_NET/?greeting=data.username<tag>
```
	↓
```
data.username<tag>
```

- SSTI脆弱性であることを検証
```
http://TARGET_NET/?greeting=data.username}}<tag>
```
	↓
```
Hello Carlos<tag>
```

- もしエラーや空白なら、テンプレート言語のシンタックスがあっているか確認する。
- テンプレート言語のシンタックスがあっていてもエラーや空白なら、SSTIは不可能

## Identify

1. Fuzzingでエラーメッセージが詳細に出ているなら、そこからテンプレートエンジンを特定 → 終了
2. `<%=foobar%>` と入力 → erbっぽいエラーメッセージが出てたら → 特定完了
3. 以下の決定木（緑が計算された・赤がそのまま出力された）を試し、特定完了
