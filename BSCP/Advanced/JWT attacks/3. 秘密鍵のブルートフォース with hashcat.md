## HashcatによるJWT署名ブルートフォース

###### 基本

- [hashcat examples](https://hashcat.net/wiki/doku.php?id=example_hashes)から、JWTのHash-Modeは16500
- attack modeはStarightで0
- 秘密鍵のwordlist [jwt.secrets.list](https://github.com/wallarm/jwt-secrets/blob/master/jwt.secrets.list)　(`forBurpSuite/burp_wordlists`にダウンロード済み)

```zsh
hashcat -a 0 -m 16500 [JWT] [jwt.secrets.listへのファイルパス]
```

- ↓結果のフォーマット
```
[JWT]:[算出した秘密鍵]
```

### ステップ

1. JWTは認証・認可等で発行されるので、ログイン等の操作を実行する
2. JWTをコピーし、ターミナル上でhashcatを使ってブルートフォースする
```zsh
hashcat -a 0 -m 16500 [JWT] [jwt.secrets.listへのファイルパス]
```
- `jwt.secrets.list`は`forBurpSuite/burp_wordlists`に用意している
- 一度算出した値は`--show`を追加して実行すれば見れる

3. 入手した秘密鍵で、JWT用に[1. JWT attackの基本](1.%20JWT%20attackの基本.md#4.%20JWT署名用の鍵を追加する)で署名用の鍵を作成する
	- "New Symmtric key"(アルゴリズムがHS...などの共通鍵の場合)
	- key sizeはそのまま。自動アップデートされる
	- `"k"`プロパティにはbase64urlencodeされた秘密鍵を指定する(自動でされる)

4. JSON Web tokenからペイロードを目的の値に編集する
5. . Signでステップ3で作成した鍵を使用してsignする(Don't modify headerを選択)
6. リクエストをsend
