## JWTとは

- JSON web token
- JWTは認証、セッション管理、アクセス制御で最も一般的に使用される
- **Claim**(ユーザに関する情報)を送信する。`"key":"value"`形式
- 古典的なsession tokenとは異なり、すべてのデータはクライアント側のJWTに保存される
	- 分散された同じ機能をもつ異なるサーバに対して同じセッションを使用できるメリットがある

#### JWT format

- トークンはheader, payload, signature(署名）の3つで構成される
- それぞれ`.`(ドット)で区切られる
- ==ヘッダとペイロードはJSONオブジェクトをbase64urlでエンコードしたもの==

- ヘッダ：トークン自体のメタデータ
- ↓ペイロード：Claim
```json
{
    "iss": "portswigger",
    "exp": 1648037164,
    "name": "Carlos Montoya",
    "sub": "carlos",
    "role": "blog_author",
    "email": "carlos@carlos-montoya.net",
    "iat": 1516239022
}
```
- このデータはトークンにアクセスできるなら誰でも読み・書き可
	- つまり安全性は==暗号署名==に依存する

### ⭐️JWT signature

1. ヘッダとペイロードを`.`で結合する
2. 1の値をハッシュ化する
3. ハッシュを秘密鍵で署名(暗号化)する
4. 得られた署名をbase64urlencodeして、`.`でヘッダとペイロードの後に加える

#### JWT vs JWS vs JWE

![[Pasted image 20240202104149.png | 300]]

- JWTは一般的にJWSを指している事が多い。文脈により異なる
- JWT: 2者間で転送できるJSONオブジェクトとしてClaimを表現するフォーマットを定義しているだけ
###### JWS (JSON Web Signature): 

- JWTの情報をデジタル署名またはHMAC(messeage authentication code)で保護する方法を定義している。
- つまり、JWSはJWTの完全性と認証を保証するためのメカニズムを提供する。
- ==署名されたJWTはJWS==と呼ばれ、トークンの内容が改ざんされていないことを受信者が検証できるようにする

###### JWE (JSON Web Encryption)

- JWEは、JWTの情報を暗号化する方法を定義する。
- JWEはトークンの内容を保護するために、情報全体を暗号化する。JWSはエンコードだけ
- これにより、トークンの内容が秘密に保たれ、トークンを解読できるのは適切な鍵を持つ受信者のみとなる

## JWT attackとは？

- 悪意のある目標を達成するために、ユーザが修正したJWTをサーバに送信すること。
- 目的は、すでに認証されている別のユーザになりすまし、認証とアクセス制御をバイパスすること
- PrivEscにつながったり完全なアカウントの侵害につながる

###### 脆弱性の原因

- ペイロードの値改ざん後、署名が異なるのに検証していない
- 検証はするが、秘密鍵が漏洩・推測・ブルートフォースなどにより有効な署名を攻撃者に作成される

## JWTのBurp Suiteでの扱い方

- [Working with JWTs in Burp Suite](https://portswigger.net/burp/documentation/desktop/testing-workflow/session-management/jwts)
- BApp: JWT EditorでJWTが使用されているリクエストは緑にハイライトされる
- JWTのヘッダ部、ペイロード部のセクションをそれぞれ選択すると、Inspectorでdecodeされた値を見ることが可能
- InspectorでdecodeしたらJWT Editorで編集する

#### JWT EditorでJWTを編集する方法

1. JWT Editorによってフラグ(緑ハイライト)が立てられらたリクエストをRepeaterへ
2. JSON Web tokenタブを開く
![[Pasted image 20240202161421.png | 200]]
3. ヘッダ、ペイロードを任意の値に編集する

###### 4. JWT署名用の鍵を追加する

1. JWT Editorを開く
2. 右パネルから生成したい鍵を選択する
3. ダイアログが新しく開く
	- ”Generate"で新しい鍵を生成する
	- もしくは既存の秘密鍵をSpecify secret:にペーストする
4. 鍵を必要な値に編集する
5. OKを選択して鍵を保存する

###### 5. Signする

- Signを押すと[1. JWT attackの基本](1.%20JWT%20attackの基本.md#4.%20JWT署名用の鍵を追加する)で作成した鍵を選択できる
