## オブジェクトの属性の操作基本実践(現実では滅多にない)

###### 基本概念の具体例

- HTTPリクエスト内に`User`オブジェクトをsession cookieに保存されていることを検出する
- これを攻撃者がデコードすると、以下のバイトストリームが算出される
```php
O:4:"User":2:{s:8:"username";s:6:"carlos";s:7:"isAdmin";b:0;}
```

- 攻撃者が`"isAdmin:b:1;`に書き換え、エンコードし直してsession cookieを上書きする
- ターゲットのWebサイトはこの値を検証しないので、このユーザに特権アクセスを許可する

###### ステップ

1. session cookieをハイライトし、Inspectorでハイライトすることでデコードする
2. デコードした内容を任意のデータに編集し、"Apply changes"で上書き
3. このデータでsession cookieを上書きする

## 属性のデータ型の変更操作

###### 基本原理

- 意図されてないデータ型を注入することでロジックの破壊や意図せぬ挙動をトリガーする
	- [1. Business logic脆弱性の基本と対策](../../Server-side/Business%20logic%20vuln/1.%20Business%20logic脆弱性の基本と対策.md)
- 例えば、以下のようなphpコードの場合
```php
$login = unserialize($_COOKIE)
if ($login['password'] == $password) {
// log in successfully
}
```
- [3. シリアライズされたオブジェクトの操作による悪用](3.%20シリアライズされたオブジェクトの操作による悪用.md#phpの緩い比較(%20==))を悪用し、`$password`に`0`を引き渡すことで、ログインをバイパスできる
	- (本来のpasswordが数字から始まる場合を除き)

- シリアライズされたオブジェクトのデータ型ラベル(`s`や`b`など）やlengthの変更も忘れずに
	- [2. 安全でないデシリアライズの検出とデータ形式](2.%20安全でないデシリアライズの検出とデータ形式.md#phpのシリアライズ後の形式)

###### phpの緩い比較(\==)

- PHPでは文字列を数値と比較する場合、文字列の先頭部分にある数値を取り出して比較を行う
```php
"5" == 5 //true
"string" == 0 //true
"1string" == 0 //false
"5string" ==5 //true
```

###### ステップ

1. シリアライズされたオブジェクトを発見し、Inspectorでデコードする
2. オブジェクトのデータ型を意図されていないものに変更する
	- 編集後はシリアライズドオブジェクトの形を整える([2. 安全でないデシリアライズの検出とデータ形式](2.%20安全でないデシリアライズの検出とデータ形式.md#phpのシリアライズ後の形式))

3. 上書きする

## アプリの機能を使う

###### 基本概念と具体例

- ユーザがアクセス可能な機能を使い、手動で危険なメソッドを呼び出し、任意のデータに対してメソッドを実行させる
	- エクスプロイトを自作し、データを自動で危険なメソッドに渡す方法は[4.マジックメソッドによる悪用](4.マジックメソッドによる悪用.md)
- デシリアライズされたオブジェクトのデータに対し、アプリの機能が危険な操作を実行することがある
- 意図されていないデータを渡し、アプリがそのデータを使用して機能を実行することで被害をもたらす

- 例えば、ユーザ削除の機能があるとする
- ユーザ削除ボタンを押すと、ユーザのプロフィール画像にアクセスして削除するとする
```php
function deleteUser($user) {
	...
    $file_path = $user->image_location; //ファイルパスを取得
    unlink($file_path);// 取得したファイルパスにあるファイルを削除。
    ...
}
```
- このとき`$user`オブジェクトがシリアライズされたオブジェクトから作成された場合、攻撃者は`image_location`に削除させたい任意のファイルパスを設定する
- 結果、アプリの機能が攻撃者から渡された任意のファイルを削除する

###### ステップ

1. シリアライズされたオブジェクトを発見し、Inspectorでデコードする
2. アプリの操作に利用されそうなデータをオブジェクトから探す。
	- ファイル削除など

3. アプリの機能実行の対象として、攻撃者の任意のデータにオブジェクトを編集する
4. "Apply changes"で上書きする。
5. アプリの機能を実行し、攻撃者が注入したデータに対する処理を実行する
