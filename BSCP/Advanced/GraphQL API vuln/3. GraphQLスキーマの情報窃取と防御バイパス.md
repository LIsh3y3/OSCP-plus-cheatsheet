## スキーマ情報窃取の基本

### ①イントロスペクションを使用する

1. まずはイントロスペクションが有効か以下のクエリで検証する(💡:Scannerではイントロスペクションが有効かわかる。有効なら"GraphQL introspection enabled "となる)
```
    #Introspection probe request

    {
        "query": "{__schema{queryType{name}}}"
    }
```

2. Burpでのイントロスペクションクエリの方法：[Accessing GraphQL API schemas using introspection](https://portswigger.net/burp/documentation/desktop/testing-workflow/working-with-graphql#accessing-graphql-api-schemas-using-introspection)に従いさらに情報収集する

- ⚠️：イントロスペクションが有効なのにクエリが実行されない場合、以下を削除する
	- `onOperation`
	- `onFragment`
	- `onField`

###### 基本

- `__schema`フィールドをクエリする。
- 通常のクエリと同様、データ構造でレスポンスに欲しい情報を指定できる
![[Pasted image 20240115094131.png | 200]]

### ②イントロスペクションが無効な時はサジェスチョン機能を使用する

###### サジェスチョンとは

- Apollo GraphQLプラットフォームの機能
- サーバがエラーメッセージでクエリの修正を提案するもの(クエリが少し間違っているがまだ認識できる場合に限る)
- 例えば、`There is no entry for 'productInfo'. Did you mean 'productInformation' instead?`
- これにより、スキーマの有用な情報を得ることができる

###### サジェスチョンを使った情報収集方法

- [Clairvoyance](https://github.com/nikitastupin/clairvoyance)というCLIツールを使用
or
- Burp Scanner " -> "GraphQL suggestions enabled"となる。

### ⭐️③ [GraphQL voyager](https://graphql-kit.com/graphql-voyager/)を使って、結果をわかりやすくする

- スキーマのビジュアライズ用

###### 基本の使い方
- CHANGE SCHEMA -> INTROSPECTION -> GraphQLレスポンスをペースト
	- ⚠️レスポンスのヘッダ部は含まない

- 左下のqueryをクリックすると、他にもmutationやフィールドごとの表示に切り替えられる
- 右下のRESETを押すと画面のアップ等がリセットされる
- `〇〇[object]`の形式はレスポンスが複数オブジェクトが返る
- 逆に`User`など`[]`がないデータ構造は1つのオブジェクトのみが返るので、==引数にidなどを指定する必要がある==
![[Pasted image 20240115164730.png]]

- ==青ハイライトはオブジェクト==(黒はスカラーだと思う）例えば下図のDeleteOraganizationUserInputをクリックすると右に`id:int`と表示される。つまり、`{id: xxx}`と入力する必要がある
![[Pasted image 20240115184759.png | 300]]

- 青文字はオブジェクト
- ミューテーション例：

					username
      }
	}
}

## Introspection防御のバイパス

###### 基本原理

- `__schema`キーワードをフィルタで防御されている

### バイパス方法

1. RepeaterのRawタブでGraphQLリクエストを右クリック->GraphQL -> Set Introspection query
2. GraphQLタブに移動する

3. `__schema`の後ろに以下の文字のいずれかを注入してリクエスト
	- スペース
	- 改行
	- コンマ(`,`)

これでうまくいかない場合、以下に変更してクエリする
