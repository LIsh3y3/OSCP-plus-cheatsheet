###### 基本原理

- ほとんどのGraphQLエンドポイントはブルートフォース攻撃を防ぐためにrate limitを備えている
- しかし、一部では、HTTPリクエストの数に基づいて制限をかける脆弱な対策をしている
	- 本来は実行された操作の数でrate制限をかけるべき

###### 具体例

- 店舗の割引コードが有効かどうかをチェックする一連のエイリアスされたクエリを示す。
- この操作は、一度に膨大な数の割引コードをチェックするために使用される可能性があるにもかかわらず、単一のHTTPリクエストであるため、レート制限をバイパスする可能性がある
```
  #Request with aliased queries

    query isValidDiscount($code: Int) {
        isvalidDiscount(code:$code){
            valid
        }
        isValidDiscount2:isValidDiscount(code:$code){
            valid
        }
        isValidDiscount3:isValidDiscount(code:$code){
            valid
        }
    }
```
