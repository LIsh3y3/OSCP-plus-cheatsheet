###### 具体例

- productリストを取得するクエリ
```
    #Example product query

    query {
        products {
            id
            name
            listed
        }
    }
```
- レスポンス
```
 #Example product response

    {
        "data": {
            "products": [
                {
                    "id": 1,
                    "name": "Product 1",
                    "listed": true
                },
                {
                    "id": 2,
                    "name": "Product 2",
                    "listed": true
                },
                {
                    "id": 4,
                    "name": "Product 4",
                    "listed": true
                }
            ]
        }
    }
```

- このことから①idが割り当てられている②id3が欠けていることがわかる
- 欠けているid3を引数に取り、クエリを送る
```
 #Query to get missing product

    query {
        product(id: 3) {
            id
            name
            listed
        }
    }
```
- そうすると、公開しているつもりのない情報が開示される
```
 #Missing product response

    {
        "data": {
            "product": {
            "id": 3,
            "name": "Product 3",
            "listed": no
            }
        }
    }


```

###### ステップ

1. クエリとそのレスポンスを確認する
2. id等、予測可能な値をデータ構造の引数に取り、クエリを投げてみる
	- 適宜、エラーメッセージから引数に何を指定する必要があるのか探る
