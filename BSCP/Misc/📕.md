###### REST API
- リソースごとにエンドポイント(URI)を定義することで、リソースを一意に識別する
- リソースに対して、HTTPメソッド(GET, POST, PUT, DELETEなど）で操作する
- データは一般的にJSONで返ってくる
- セッション管理を不要とすること
- 人間がGUIだと操作しやすいように、サーバからするとREST APIのシステムは操作しやすい
###### 例
- `https://www.web-security-academy.net/admin/delete?username=carlos`

##### ステート

- サブステートはその親であるステートがアクティブな時にのみ、アクティブになることができるステート

##### Cookie
- サーバが次回のクライアント識別のために、クライアントに保存させるテキストファイル
- クライアントは次回訪問時にこれをサーバに提出する。サーバはCookieに含まれる識別子を頼りに、前回の状態を引き継いでクライアントに機能を提供する
- Cookieには==SessionID==や==userID==などが保存されている。
- 整理券番号のようなもの。バレたら困る認証情報は入れない。
- つまり、Cookieを削除してWebサイトがおかしな挙動をする場合、状態はサーバに保存されていると考える

##### CORS

##### プリフライトリクエスト
- 「シンプルなリクエスト」(徳丸本p.88)以外のリクエストをクロスドメイン(異なるドメイン)へ送信する場合

##### バイナリセーフの関数
- どんなバイナリでも正しく処理できるという意味
- ヌルバイト（`%00`）などを正しく解釈することができる。正しく解釈すれば、ヌルバイトを文字列の終端とみなさない。
- ヌルバイト攻撃の根本対策
- しかし、ファイルシステムなどではヌルバイトを許可しないので、バイナリセーフの関数でヌルバイトを正しく解釈してファイルシステムに渡すと、'filename%00'となりおかしな挙動をトリガーする可能性があるので現実的にバイナリセーフの関数だけではうまくいかない。

##### イベントリスナー
- Web開発において指定されたタイプのイベントが発生したときに実行されるJavaScriptの関数またはプロシージャのこと。
- イベントとはユーザーのアクション（クリック、キーボード入力等）、Webページの状態変化（ページの読み込みが完了したとき等）、や==Webメッセージの受信==など、さまざまなものがある。

- イベントリスナーは`addEventListener`メソッドを使用してHTML要素に追加されるのが一般的
- 要素の属性としても設定することができる。
- イベントリスナーは、そのイベントが発火した際に、何らかのアクションを起こしたり、データを処理したりするために使用される
- 同じイベントに対し複数の処理を割り当てることができる。

##### イベントハンドラ
- イベントリスナーはイベントの発生を待ち受けし、イベントハンドラーは発生したイベントに応答して動作するコード

##### 名前空間

- 名前（name）を空間（space）ごとにグループ化して、重複を許容する仕組み
- 目的：
	- 同じ名前のものを区別できるようにする
	- 管理しやすくする（整理整頓）
		- 名前空間を使わないとプログラムやDB全体で 「名前が一意でなければならない」ので、users1、users2、users_backup みたいにごちゃごちゃになる。
- つまりは「苗字」のようなもの
	- 「苗字」が違えば同じ下の名前でも区別ができる

###### コードでの例

```c#
namespace CompanyA {
    class Logger {
        void Log() {}
    }
}

namespace CompanyB {
    class Logger {
        void Log() {}
    }
}
```
- CompanyA.Logger と CompanyB.Logger は別物。
- 名前は同じ Logger だけど、属する空間が違うから問題なし。

##### Training data
- LLMが学習するために使用されるデータセットのこと。
- このデータには、文章、書籍、ウェブページ、対話記録など、多種多様なテキスト情報が含まれている

##### 間接入力
- モデルが直接的なユーザーのクエリ（prompt）以外から受け取る入力のこと。
- これには、[📕](📕.md#Training%20data)の他に、モデルが関連づけられたメタデータ、モデルの設計時に設定された仕様やパラメーター、さらにはモデルの更新履歴や以前のクエリなどが含まれている
