
###### DOM-based open redirectionとは

- スクリプトが攻撃者の制御可能なデータをsinkに書き込むことで、クロスドメインのナビゲーションをトリガーする
- 被害者がアクセスした場合に任意の外部ドメインへのリダイレクトされてしまうURLを攻撃者が構築する。
```js
let url = /https?:\/\/.+/.exec(location.hash);
if (url) {
  location = url[0];
}
```
- `/https?:\/\/.+/.exec(location.hash);`: 正規表現。URLのフラグメントに記載されたHTTPまたはHTTPSのURLを`url`変数に格納する
- `location　= url[0];` ：locationプロパティに新しい値を代入すると、その値へリダイレクトする

---
###### DOM-based open redirectionの影響

- Webサイトを訪れたユーザに対してフィシングを容易にするために使われるテクニック
- 信頼できると見せかけて、実際は悪意のあるサイトにユーザを誘導するリンクを作成することができる。
- 有効なTLS証明書をもつ本物のアプリケーションURLを使用することが可能

- 更にリダイレクションAPIに渡すURLの始まりの部分を制御できる場合、この脆弱性はJavaScriptのインジェクション攻撃へと発展する。
- 攻撃者は`javascript:`擬似プロトコルを使用したURLを作成して、そのURLがブラウザによって処理される際に任意のコードを実行する(XSS)ことができる

---
## DOM-based open redirectionの悪用手順

- 持続型XSSをBack to blogに仕掛けるイメージ

1. リダイレクト処理を探す
	- open redirectと関わりのあるパラメタ名は`redirect=`、 `path=`、`goto＝`、`url`等

2. 開発者ツールのSourcesにて[0. sinkとsourceのリスト](0.%20sinkとsourceのリスト.md#open%20redirection)で検索し、入力によって動的にURLを生成している箇所を見つける
- 例
```html
<a href='#' onclick='returnUrl = /url=(https?:\/\/.+)/.exec(location); location.href = returnUrl ? returnUrl[1] : "/"'>Back to Blog</a>
```

3. 被害者をリダイレクトしたい目的のURLを生成する
4. 以下をブラウザの検索バーに入力する
```
https://TARGET_NET/post?postId=4&url=https://EXPLOIT_DOMAIN/
```
