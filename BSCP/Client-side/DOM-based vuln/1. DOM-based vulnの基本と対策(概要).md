### 基本

- HTMLの内容を操作（追加・削除など）するために使うインターフェース。
- Document ≒ HTML
- ブラウザはHTMLの構文を解析してDOMツリーと呼ばれる構造体を生成する
	- DOMツリーの内容を変えることで、DOMツリーの元となるHTMLも書き換わる
- JSによりブラウザ画面に表示される内容を書き換えるために使うことが多い
```html
<html>
  <head>
    <meta charset="utf-8">
    <title>Top Page</title>
  </head>
  <body>
    <p>ようこそ</p>
  </body>
</html>
```

## Taint flowの脆弱性

- Taint(汚染された) flow：　ユーザによる入力が汚染されたもの(悪意あるもの)とみなす。その汚染された入力の流れのこと
- ==DOMの脆弱性の基本的な原因==は、攻撃者が入力したデータがクライアント側の(脆弱な)コードで処理されるその方法によるもの(taintの流れ：ソース -> シンク -> シンクによる処理)
- 💡DOM-based vulnにつながる一般的な[ソースのリスト](https://portswigger.net/web-security/dom-based#:~:text=attack%2C%20for%20example.-,Common%20sources,-The%20following%20are)
- - ⭐️DOM-based vulnにつながる[シンクのリスト](https://portswigger.net/web-security/dom-based#:~:text=Which%20sinks%20can%20lead%20to%20DOM%2Dbased%20vulnerabilities%3F)
	- **シンクが見つかったらこのリンクを参照すること**

### ソース(source)

- 入り口
- 潜在的に攻撃者が制御可能なデータを受け入れるJavaScript[📕](../../Misc/📕.md#プロパティ)
- ユーザからの入力が取得される場所や方法
- URLの一部（パラメータやフラグメント）、HTMLフォームの入力フィールド、cookie、ファイルパス等、多くある
- sourceからのデータはその後、sinkに流れ込み、DOMに統合される
- `location.href`など
- [2. Web messageをソースとしたDOM XSS](2.%20Web%20messageをソースとしたDOM%20XSS.md)


### シンク(sink)

- 出口
- 攻撃者が制御可能なデータを渡されると、望ましくない効果を引き起こす可能性のあるJavaScript関数またはDOMオブジェクトのこと
- ユーザからの入力データ(sourceから取得）によりJavaScriptを生成、実行してしまう箇所
- `innerHTML`、`document.write`、`eval`など
- 渡される引数をJavaScriptとして処理する

##### DOM Clobbering(Expert 割愛)

- 詳細：[DOM clobbering](https://portswigger.net/web-security/dom-based/dom-clobbering)
- Expertsなので割愛
- DOM XSSが不可能な場合でも、あるHTMLページにおいて`id`や`name`属性がHTMLフィルタによって許可されている場合、このテクニックが利用可能

## DOM-based vulnの対策の概要

- 絶対解はない。
- 回避する最も効果的な方法は、ソースからの信頼できないどんなデータもシンクに送信値を動的に変更することを避けること
- どうしても実装上必要なら、クライアント側のコードの中に防御を実装する

1. データの検証: アプリケーションに提供されるデータが安全だと確認するため、ホワイトリストを用意し、そのリストにある項目だけを受け入れる
    
2. データのサニタイジング: データを使う前に、そのデータから悪意のあるコードを取り除く、または無力化する処理を行う。これにより、悪意あるデータが実行されるリスクを減らす
    
3. データのエンコード : 特定の文脈に挿入される前に、データを適切な形式に変換します。JavaScriptのエスケープ処理、HTMLエンティティのエンコーディング、URLエンコーディングなどがあるが、これらは正確な順序で適用する必要がある
