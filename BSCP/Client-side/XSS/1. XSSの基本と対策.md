
## XSSとは

- 基本的に、JavaScriptを含むページをブラウザで閲覧すると勝手にJavaScriptが実行されるという特性を悪用する
- 悪意あるJavaScriptを自分もしくは正規のWebサイトに仕込み、被害者のアクセスを待つ。
- 被害者がアクセスすると、被害者のブラウザ上で悪意あるJSが実行される
- 正規のユーザがWebサイト上でできることは、XSSを使えば==何でも==できる

- 対象オリジンにJavaScirptを送り込み、[📕](../../Misc/📕.md#同一オリジンポリシー)制限を回避して意図を実行する
- CSSを活用しフォームを上書きするなど、JSを使用しないケースもある(徳丸本p.128)

---
## XSS proof of concept

- 自分のブラウザに以下のペイロードを注入することで、ほとんどのXSS脆弱性を確認できる

###### 基本

- `alert()`関数を使用。
- 見逃しにくいし影響が少ない

###### Chrome version 92以降(2021/07/20~)

- `print()`関数を使用
- `<iframe>`タグ内では`alert()`が動作しないケースがある
- なぜ`print()`なのかについて([リンク](https://portswigger.net/research/alert-is-dead-long-live-print))

##### コンテキストは重要

- どのようなペイロードを注入するかは脆弱性の存在するコンテキストによる
- [XSS contexts](../../../OSCP/Cheatsheet/Web%20App/XSS%20contexts.md)

## 🛡️XSSの対策

### ⭐️出力時にデータをエンコードする

###### レスポンスの文字エンコーディング指定

 - `header('Content-Type: text/html; charset=UTF-8');`

###### 複数のエンコーディングを正しい順序で適用する

- 例えば、ユーザー入力をイベントハンドラ内に安全に埋め込むには、JavaScriptのコンテキストとHTMLのコンテキストの両方を扱う。
- そのため、まず入力をUnicodeエスケープし、次にHTMLエンコードする
```html
<a href="#" onclick="x='この部分を2段階に分けてエスケープする'">test</a>
```

#### PHPでの方法

###### HTMLで特別な意味をもつ記号文字をエスケープする

- phpの`htmlspecialchars()`を使用し、第二引数に`ENT_QUOTES`を使用
```php
<?php echo htmlentities($input, ENT_QUOTES, 'UTF-8');?>
```

- 属性値はさらに`"`(ダブルクォート)で囲む

###### 入力がJSの文字列コンテキスト内では、英数字以外をUnicodeエスケープする

- テンプレートエンジンを使用するか、以下の自作関数(PortSwigger提唱）を使用する
```php
<?php
```
```php
function jsEscape($str) {
    $output = '';
    $str = str_split($str);
    for($i=0;$i<count($str);$i++) {
        $chrNum = ord($str[$i]);
        $chr = $str[$i];
        if($chrNum === 226) {
            if(isset($str[$i+1]) && ord($str[$i+1]) === 128) {
                if(isset($str[$i+2]) && ord($str[$i+2]) === 168) {
                    $output .= '\u2028';
                    $i += 2;
                    continue;
                }
                if(isset($str[$i+2]) && ord($str[$i+2]) === 169) {
                    $output .= '\u2029';
                    $i += 2;
                    continue;
                }
            }
        }
        switch($chr) {
            case "'":
            case '"':
            case "\n";
            case "\r";
            case "&";
            case "\\";
            case "<":
            case ">":
                $output .= sprintf("\\u%04x", $chrNum);
            break;
            default:
                $output .= $str[$i];
            break;
    }
    }
    return $output;
}
?>
```

- 使い方
```js
<script>x = '<?php echo jsEscape($_GET['x'])?>';</script>
```

#### JavaScriptでの方法

###### 文字列をHTMLエンティティに変換する

```js
function htmlEncode(str){
    return String(str).replace(/[^\w. ]/gi, function(c){
        return '&#'+c.charCodeAt(0)+';';
    });
}
```

- 使い方
```js
<script>document.body.innerHTML = htmlEncode(untrustedValue)</script>
```

###### 入力がJSの文字列コンテキスト内の場合(Unicodeエスケープ)

```js
function jsEscape(str){
    return String(str).replace(/[^\w. ]/gi, function(c){
        return '\\u'+('0000'+c.charCodeAt(0).toString(16)).slice(-4);
    });
}
```

- 使い方
```js
<script>document.write('<script>x="'+jsEscape(untrustedValue)+'";<\/script>')</script>
```

- [5. DOM based XSS](5.%20DOM%20based%20XSS.md#jQuery)

### 入力を検証する

- 無効な入力はブロックする。
- 無効な入力を有効なものに変換する処理はエラーを起こしやすいので可能な限り避けるべき。
- ブラックリストではなくホワイトリストを使用すること。
###### 安全なプロトコルか

- 例えば、ユーザが入力したURLをレスポンスで返す場合、HTTPやHTTPSなどのプロトコルか。
- `javascript`や`data`を許可すると悪用される

###### 開発者の意図した値か

- 例えば数値が入力されると意図しているフォームで実際に整数が入力されたか検証する

---
### 安全なHTMLのみ許可する

###### HTMLタグやCSSの入力を許可しない

- どうしても許可するなら、HTML PurifierやDOMPurifyというライブラリを使用する。
- しかし、完璧な解決策ではないしXSSの脆弱性がある場合がある。アップデートも注意深く監視する必要がある
