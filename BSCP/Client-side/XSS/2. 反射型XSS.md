
## 反射型XSSとは

- 攻撃用JSが攻撃対象サイトとは別のサイト(罠サイトやメールのURL)にある場合のXSS
- 入力された値をそのままレスポンスで返す場合
- 被害者を罠サイトに誘導する必要があるので[3.持続型XSS](3.持続型XSS.md)に比べ影響は小さい
- 攻撃者のサイトに悪意あるHTMLをホストし、被害者にDeliverする

![[Pasted image 20231221115732.png | 400]]

###### 具体例

- URLパラメタに入力された値で検索するWebアプリ
```
https://insecure-website.com/search?term=gift
```
- このWebアプリは入力された値をレスポンスに表示する
```html
<p>You searched for: gift</p>
```

- エスケープ処理などの対策を何も施していない場合、攻撃者が以下のようなペイロードを注入できる
```
https://insecure-website.com/search?term=<script>/*+Bad+stuff+here...+*/</script>
```
- Webアプリは悪意あるスクリプトをレスポンスに表示する
```html
<p>You searched for: <script>/* Bad stuff here... */</script></p>
```
- もし被害者のブラウザで攻撃者の用意した罠サイトのURLをクリックしたとき、このスクリプトが被害者のコンテキストで実行される

## 反射型XSSの検出とテスト方法

### 自動での検出

- 大部分はBurp suiteのScanner(pro)で迅速かつ確実に発見可能

### 手動での検出とテスト

###### 1. すべての入力箇所を探す

- URLのクエリ文字列のパラメタ
- bodyのメッセージ部のパラメタ
- JSON, XML, Cookieなどにあるパラメタ
- URL上のファイルパス

###### 2.各入力箇所に一意のランダムな値を送信し、入力値がレスポンスに反映されるか確認

- DOM Invaderでcanaryを注入していく

OR
 Intruder
- Attack Type: Battering ram
- Payload types: Numbers
	- Number range->
		- Sequence
		- From: 0
	- Number format->
		- Base: Hex
		- Min integer digits: 8

- 送信した内容が含まれたレスポンスにフラグを立てる
	- Settings -> Grep - Payloads -> search responses for payload settingsに☑️ 

###### 3.レスポンスから反射型XSSのコンテキストを決定する

- [XSS contexts](../../../OSCP/Cheatsheet/Web%20App/XSS%20contexts.md)

###### 4. 初期候補のpayloadをテストする

- ステップ2でフラグが立ったリクエストをRepeaterへ送る
- ステップ２で生成したランダムな値の前後にXSSペイロードの初期候補を入力しsend
- ランダムな値でSearchし、ハイライトされた箇所付近を探しうまくいったか確認する

###### 5. 代わりのペイロードをテストする

- ステップ4のペイロードがうまくいかない場合
- ステップ3で決定したコンテキストを元にペイロードを変更する
- 詳しくは[XSS contexts](../../../OSCP/Cheatsheet/Web%20App/XSS%20contexts.md)

###### 6. 実際のブラウザ上で有効なペイロードが実行されるか確認する

- 有効なペイロードをRepeaterではなく実際のブラウザ上で使用する。
- `alert(document.domain)`など
- この際、==攻撃者は自身のサーバにペイロードをホストし、罠を被害者へ送信する==

##### 反射型XSSの基本的なpayload形式

1. ユーザの介入なし(no user interaction)
```html
<iframe src="[target_site]?[vuln_param]="><[payload]" onload=this.style.width='100px'>
```

2. 介入あり
```html
<script>
location = '[target_site]?[vuln_param]=[payload]'
</script>
```

3.  攻撃者自身の罠サイトへのリンクなどを注入したあと、攻撃者自身でpayloadをトリガーする場合
```url
[target_site]?[vuln_param]=[payload]
```

- ＊target_siteにはhttpなどスキーマから始まるURLが入る
