### 基本

- Meterpreterは、ハッカーが侵入したシステムに対して、遠隔操作でコントロールを行うためのツールの一つで、Metasploitのペイロードの一種
- 特殊なシェルの一種で独自のコマンドを使用する
- ターゲットシステム上で動作するが、ターゲットシステム上にインストールされるわけではない。メモリ上で動作し、ターゲット上のディスクにファイルを書き込むことはない。
- この機能は、ウイルススキャン時に検出されないようにするのが目的。
- デフォルトでは、ほとんどのウイルススキャンソフトはディスク上の新しいファイルをスキャンする。そこで、Meterpreterはメモリ（RAM）内で動作し、ターゲットシステム上のディスクにファイル（例：meterpreter.exe）を持たない。こうすることで、Meterpreterはプロセスとして認識され、==ウイルススキャンを回避する==
- その証拠として、meterpreterプロンプトで`getpid`（プロセスの一意のIDを表示）コマンドを実行するとIDが表示される。つまりプロセスとして認識されている
- ターゲットマシン上で動作するMeterpreterから攻撃者への通信はTLSで暗号化されているので、==IDSやIPSから検知されない==
- ほとんどのアンチウイルスソフトウェアが検知できるとはいえ、ステルス性は結構高い

- [What is the shell](../../../Cheatsheet/Common/What%20is%20the%20shell.md#ペイロードの種類%20Staged%20vs%20Stageless)にもあるようにstaged payloadのほうがステルス性が高い
- Meterpreterのバージョンは`msfvenom --list payloads ｜ grep meterpreter`で表示できる
- androidやapple, linuxなど、使用できるバージョンが表示される
- どのバージョンを使用するかは以下の3点から判断する
	-  対象OS：対象OSはLinux？Windows？Mac？Android？
	- ターゲットシステムで利用可能なコンポーネント： Pythonがインストールされているか？ PHPで出来たウェブサイトか？ 等
	- ターゲットと確立できるネットワーク接続の種類：　平文のTCP接続を許可しているか？HTTPSの逆接続しかできないのか？IPv6アドレスはIPv4アドレスほど厳重に監視されていないのか？など

- すべてのエクスプロイトにおいてMeterpreterを利用できるとは限らない。

### Meterpreter command

- meterpreterプロンプトで`help`と入力すると使用可能なコマンドが表示される
- バージョンごとに異なるので、`help`で確認するのはいいこと
- さまざまなコマンドがあるが、カメラが必要な`webcam_snap`などはカメラがないマシンには無効

### Post Exploitation w/ Meterpreter

- `getuid`と入力すると、`Server username: NT AUTHORITY\SYSTEM`のように、現在実行されているユーザを表示する。これにより、ターゲット・システムで想定される特権レベルを知ることができる
- `getpid`で現在のセッションのPIDを表示するが、実際はmeterpreterのPIDは見つからない＝ステルス性が高い
- `ps`コマンドで実行中のプロセスを一覧表示する。"PID"列には、Meterpreterを別のプロセスにmigrateする際に必要なPID情報も表示されている

###### migrate

- `ps`コマンドで表示したPIDを使用して
```
migrate <PID>
```
とすることで、例えば、ターゲット上で動作しているワープロ（word.exe、notepad.exeなど）を見つけたら、そのプロセスに移行して、ユーザーからこのプロセスに送られるキーストロークのキャプチャを開始するなどが可能になる
- また、移行先のプロセスがSYSTEMプロセスである場合、そのプロセスは通常、システムの一部として常に実行されているため、Meterpreterが予期せぬエラーで停止する可能性を低減することができる
- つまり、そのプロセスとMeterpreterがやりとりしやすくなる
- **注意**：　高権限ユーザー（例：SYSTEM）から低権限ユーザー（例：Webサーバー）が起動するプロセスに移行した場合、ユーザー権限を失う可能性がある。この場合、権限を取り戻すことができない場合があるので、==同じUserのプロセスにmigrateすること==。一番安全なのは、**services.exe**

###### hashdump

```
hashdump
```
- これで、SAM(Security Account Manager）データベースの内容を一覧表示する。このデータベースはWindowsシステム上のユーザーのパスワードをNTLMフォーマットで保存している
- Pass-the-hash攻撃へ[[Attactive Directory#Domain Privilege Escalation]]

###### shell

```
shell
```
- shellコマンドは、ターゲットシステム上で通常のコマンドラインシェルを起動する
- `CTRL+Z`でmeterpreterに戻る

#### 実践
- `load`コマンドを使用することで、pythonやkiwiなどの追加ツールも使用可能になる
- kiwiとmimikatzは作成者が一緒。mimikatzはWindows向けのポストエクスプロイトツールの1つで、パスワードやKerberosチケットなどを含む認証情報を取得し、悪意のある攻撃に利用する。両方ともWindowsセキュリティに関する検証を行うためのツールであり、パスワードやハッシュ値をダンプすることができるが、kiwiはMetasploitの拡張機能として提供されており、より高度な攻撃手法を実行することができる
- post exploitationフェーズでPriv Escや水平移動、情報収集などをmetasploitなら充実のサポートがある
- 実際にエクスプロイトを成功させ、meterpreterプロンプトに入った後、`help`コマンドで実行可能なコマンドを調べたり、Kiwiを`load`してクレデンシャルを入手したり
- `sysinfo`は対象マシンのさまざまな情報を露わにする

- コマンドだけでは入手できない情報は、例えばsmb shareの一覧などは
1. 現在のmeterpreterを`CTRL + Z`でバックグラウンド化する。→ sessionの一つになる
2. `search`コマンドで適切なモジュールを選択し、`show options`で`SESSION`が出たら、そこにセッションIDをセットする
3. モジュールを実行
4. セッションに戻るには`sessions -i <SessionID>`

- パスワード入手は
1. まず`hashdump`をためす
2. 十分な権限がないと表示されたら、`help`コマンドの結果にもあるように`getsystem`で権限昇格してからもう一度dumpする
3. それでも失敗したらIDSやFWによって検知されているので、`migrate`を考える
4. 例えばNTLMハッシュは`lsass.exe`にあるので、そのプロセスIDにmigrateしてからdumpする
5. Hashcatでモードを検索する。だめなら、ハッシュ値を直接[crackstation](https://crackstation.net/)　に入力してみる

- ファイルのパスに空白が含まれている時、meterpreterプロンプトから`cd`するときは`""`（クウォテーション）を含む
