# RDP接続

- 条件：ローカルでRemote Desktop UsersもしくはAdministratorsグループに所属しているユーザー
- ドメインポリシーによっては、上記のグループに所属していなくても使える
```zsh
# ADの場合、/d:<domain>を追加
# 接続ができない場合、/network:wan /audio-mode:1 /timeout:50000 を追加するか、sudo ifconfig tun0 mtu 1250
xfreerdp3 /dynamic-resolution +clipboard /cert:ignore /v:$TargetIP /u:<username> /p:'<pw>'
```

- 代替手段としてのrdesktop
	- これは古いWindowsサーバーに対して接続性が良い場合がある
```sh
rdesktop -u <username> -p <password> -g 1280x720 -P -z $TargetIP
```
- `-g`: 解像度の指定
- `-P`: ビットマップキャッシュを有効化
- `-z`: 圧縮を有効化

# RDPの有効化

- 条件：ローカル管理者権限
- (WinRMで十分なため、必要性は低い)
```powershell
# RDPを有効化してファイアウォールルールも設定
Set-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' -Name 'fDenyTSConnections' -Value 0
Enable-NetFirewallRule -DisplayGroup 'Remote Desktop'

# 確認→0なら成功
Get-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' -Name 'fDenyTSConnections'
```
- →RDP接続（`/d`は不要）

# トラブルシューティング

## Xfreerdpがうまく接続できない

- mtuを下げる（デフォルト1,500）
```zsh
sudo ifconfig tun0 mtu 1200
```

- 環境変数ではなく、IPアドレス直指定する
	- 環境変数が原因かどうかは断定できないが、失敗することがある
```zsh
⭕️xfreerdp3 /v:192.168....
❌xfreerdp3 /v:$TargetIP
```

- `/etc/hosts` に FQDN の名前解決登録をし、ドメイン名を指定して接続

## 入力が英字配列のキーボードになってしまう→対策なし

- Xfreerdpからできる対策はなし
- マシンの環境では、ログオン後インターネット接続が使えないので、日本語packのインストールもできない
