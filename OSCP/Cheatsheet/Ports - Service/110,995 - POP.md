# 留意点

- POP3はデフォルトで「ダウンロード後にサーバーから削除」する設定が多い。不用意にメッセージを取得すると、サーバー上の証拠（メール）を消してしまう可能性があるため、実務では注意が必要

# POP3の基本

- POP3（Post Office Protocol version 3）は、リモートサーバーからローカルクライアントにメールを受信・取得するためのプロトコル
- クライアント・サーバーモデルで動作
- 通常はメールをダウンロードしてサーバーから削除する（IMAPとの大きな違い）
- デフォルトポート：
    - 110：==暗号化なし==
    - 995：SSL/TLS暗号化（POP3S）

---

# 接続方法

標準的な接続（Port 110）
```zsh
nc -nvC <TargetIP> 110
# または
telnet <TargetIP> 110
```
- ↓出力例
```zsh
# <サーバーソフト> (OS)
+OK Dovecot (Ubuntu) ready.
```

SSL/TLS接続（Port 995）
```zsh
openssl s_client -connect <TargetIP>:995 -crlf -quiet
```

curlを使用したメール操作
```zsh
# メール一覧を表示
curl -u <username>:<password> pop3://<TargetIP>/

# 特定のメール(ID:1)を表示
curl -u <username>:<password> pop3://<TargetIP>/1

# POP3S（証明書チェックを無視）
curl -u <username>:<password> pop3s://<TargetIP>/ --insecure
```

# Exploit

## ユーザー名の列挙

POP3にはSMTPのようなVRFYコマンドはないが、レスポンスの差異で推測可能な場合がある（接続後）→ [25,465,587 - SMTP](25,465,587%20-%20SMTP.md)で列挙する方が簡単
```zsh
USER admin
# +OK (存在する可能性が高い)
# -ERR (存在しない、あるいはサーバーの設定で隠蔽されている)
```
- 💡レスポンスの「時間差」を利用して列挙が可能な場合もある
- 💡ユーザー名の候補としてWebサイトから収集できる人物名以外にも、部門名（例：hr、it等）が使える
- ユーザー名は基本的に小文字・大文字問わない
- ユーザー名は`@`以降のドメイン名は不要で列挙できる（実際は`admin@gmail.com`でも、`admin`で有効）

## 認証情報のブルートフォース

⚠️fail2banなどの防御機構により、アカウントロックされる可能性有り

方法A：[🐉THC-Hydra](../../Tools/🐉THC-Hydra.md#サービスに対する辞書攻撃)
```zsh
# 110
hydra -V -f -l <username> -P <PW_wordlist> <TargetIP> pop3

# 995
hydra -V -f -S -l <username> -P <PW_wordlist> -s 995 <TargetIP> pop3
```

方法B：Nmapスクリプトによるブルートフォース
```zsh
nmap -p 110 --script pop3-brute <TargetIP>
```

## スニッフィング

スタンドアロンマシンに対しては使わない

パケットキャプチャ
	kaliのtun0を通過する、TargetIPを送信先/受信とするport 110の通信をキャプチャ
```zsh
# ターミナルに出力
# imapもopenであれば
sudo tcpdump -i tun0 host <TargetIP> and \( port 110 or port 143 \) -A

# wireshark用
sudo tcpdump -i tun0 host <TargetIP> and port 110 -w pop3.pcap
```
-  →[🦈Wireshark](../../Tools/🦈Wireshark.md)でキャプチャしたデータを開き、認証情報を特定する
	- フィルタ: `pop` または `tcp.port == 110` 
	- TCPストリームを追跡: `AUTH PLAIN` または `AUTH LOGIN` とやり取りしているパケットを右クリックし、`[Follow]` -> `[TCP Stream]`を選択

# 侵入後の操作（Post-Exploitation）

## 基本的なメール取得操作

```zsh
USER <username>     # ユーザー名入力
PASS <pw>     　　　 # パスワード入力

LIST                # メール一覧とサイズ確認
RETR <1>            # 1番目のメールを取得
```

## メールの一括ダウンロード（自動化）

認証情報を入手後、ループ処理で全てのメールをローカルに保存する。
```zsh
for i in {1..100}; do
  curl -u <username>:<password> "pop3://<TargetIP>/$i" > email_$i.eml 2>/dev/null
done
```

ダウンロードした`.eml`ファイルからキーワードを検索する
```zsh
# パスワード関連の検索
grep -r "password\|credential\|username" *.eml

# URLの抽出
grep -Eiorh 'https?://[^\s]+' *.eml

# メールアドレスの抽出
grep -Eiorh '\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b' *.eml
```

## 設定不備の確認

- POPサーバーのログ設定（`auth_debug`等）が有効な場合、サーバー側のログにパスワードが平文で記録されている可能性がある
	- →サーバー内のログファイルにアクセス可能な場合は要確認

# POP3コマンド一覧

| コマンド   | 説明                           | 使用例             |
| ------ | ---------------------------- | --------------- |
| `USER` | ユーザー名の入力                     | `USER username` |
| `PASS` | パスワードの入力                     | `PASS password` |
| `STAT` | メールの総数と合計サイズを表示              | `STAT`          |
| `LIST` | 各メールの番号とサイズを一覧表示             | `LIST`          |
| `RETR` | 指定した番号のメールを表示                | `RETR 1`        |
| `TOP`  | メールのヘッダーと指定した行数のみ表示          | `TOP 1 10`      |
| `UIDL` | メールの固有ID（UID）を表示             | `UIDL`          |
| `DELE` | メールに削除マークを付ける                | `DELE 1`        |
| `RSET` | 削除マークを解除する                   | `RSET`          |
| `CAPA` | サポートしている機能（Capabilities）を表示  | `CAPA`          |
| `QUIT` | ログアウト（この時DELEマークされたものが削除される） | `QUIT`          |
