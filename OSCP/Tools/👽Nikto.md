# Niktoとは

あらゆるタイプのWebサーバーの構成ミスや既知の脆弱性を検出するための、オープンソースのWebサーバー・スキャナ。

数千項目に及ぶテストを高速に実行し、以下のような問題をあぶり出す:
- ​サーバーの構成ミス: 不要なディレクトリの露出や、デフォルト設定のままの管理画面など
- ​古いバージョンの検出: セキュリティホールがある古いApacheやNginxなどのバージョン特定
- ​危険なファイルの存在: config.php.bak のような一時ファイルや、テスト用のスクリプト
- ​HTTPヘッダーの不備: XSS対策などのセキュリティヘッダーが欠落していないかのチェック
- ​既知の脆弱性: 特定のサーバーソフトウェアに対するCVE（共通脆弱性識別子）に関連する問題
- 参考記事：[Niktoの解説 - 個人ブログzenn](https://zenn.dev/oreha_gao/scraps/e413e7e88fb929)

> [! WARNING]
> - サブディレクトリのスキャンは自動で実行しないため、指定する必要がある
> 	- `-h <TargetIP>`は、`http:/<TargetIP>`をスキャンするが、`http://<TargetIP>/xxx`はスキャンしない
> - 過剰なリクエスト、かつ、過検知、見過ごしも非常に多いため、OSCP+ではほとんど使わなかった

# 目的

- Webサーバーが公開しているファイル、CGIスクリプト、ディレクトリなどから、Webサーバの以下の脆弱性を探す：
	- 機密性の高いファイル  
	- 古いサーバーやプログラム（例：脆弱なWebサーバーのインストールなど）  
	- よくあるサーバやソフトウェアの設定ミス（ディレクトリインデックス、cgiスクリプト、x-ssプロテクト）

- ✅ 良い使い方：
	- 初期偵察での情報収集
	- サーバー設定の確認
	- 見落としがないかのチェックリスト
	- 例えば、SQL injectionについて多く報告

- ❌ 悪い使い方：
	- 結果をそのまま信じる(==偽陽性が非常に多い==)
	- 全ての項目を調査する

# 標準スキャン

```shell
# -C：CGIディレクトリを探索
nikto -Format htm -o nikto.html --maxtime=180s -C all -h <TargetIP>
```
- faviconやデフォルトのパスなどから何のアプリケーションが使われているか割り出せる
- どのHTTPメソッドが使われているかわかる
	- →DELETEなどは悪用可能

---

# 複数のホストとポートをスキャンする

特定のホストのポートをスキャンするとき、`-p`を使用し、複数のポートをスキャンできる
```zsh
nikto -h <TargetIP> -p <Port>, <Port>
```
- ⚠️`nikto -h http://`とやると、ポート80しか検査されない

# プラグインについて

- スキャンした後、その情報からターゲットに適したプラグインを探す
	- プラグインは`--list-plugins`でリストを閲覧できる
```zsh
nikto -h <Target> -Plugin apacheuser
```

Plugin名|	Description
---|---
apacheusers|Apache HTTP 認証ユーザーの列挙を試みる
cgi|	悪用できる可能性のあるCGIスクリプトを探す
robots|	robots.txtファイルを解析し、どのファイル/フォルダーに移動できるかを決定
dir_traversal|	Linux の /etc/passwd などのシステムファイルを探すために、ディレクトリトラバーサル攻撃（LFI など）を使おうと試みる(http://ip_address/application.php?view=../../../../../../../etc/passwd)

# スキャンの詳細化

- 基本的に出力全体を表示しないので、`-Display`フラグにプラスで目的に合わせて以下の引数を使用

| 引数  | Description                  | 使用ケース                                                          |
| --- | ---------------------------- | -------------------------------------------------------------- |
| 1   | Webサーバーから与えられたすべてのリダイレクトを表示. | Webサーバーは、特定のファイルやディレクトリに利用者を再配置したい場合があるので、これに応じてスキャンを調整する必要がある |
| 2   | 受け取った全てのCookieを表示            | セッション情報やクレデンシャルなどないか確認                                         |
| E   | 全てのエラーを出力                    | スキャンがうまくいってないときはエラーを返す                                         |

# 脆弱性カテゴリーごとのスキャン

検証する脆弱性を絞るために、`-Tuning`を使用可能
```zsh
# 例）XSSとSQLiに絞って検証
nikto -Tuning 49 -h <Target>
```

- 以下主要オプション（詳細はhelpを確認）：

| 脆弱性カテゴリー名                         | Description                                                                          | Tuningオプション |
| --------------------------------- | ------------------------------------------------------------------------------------ | ----------- |
| File Upload                       | ・Webサーバー上でファイルのアップロードが可能なものを探す<br>・これは、アプリケーションを実行するためのリバースシェルをアップロードするために使用することができる | 0           |
| Misconfigurations / Default Files | Webサーバー上の機密性の高い（設定ファイルなどアクセスすべきでない）共通ファイルを検索                                         | 2           |
| Information Disclosure            | Webサーバーやアプリケーションに関する情報を収集する（バージョン番号、HTTPヘッダーなど、後で攻撃に活用できる可能性のある情報）                   | 3           |
| Injection(XSS/Script/HTML)<br>    | インジェクション攻撃に使えそうな場所を探索                                                                | 4           |
| Command Execution                 | OSコマンドの実行を許可するもの（シェルの起動など）を探す                                                        | 8           |
| SQL Injection                     | SQLインジェクションの脆弱性があるURLパラメータを持つアプリケーションを探す                                             | 9           |
