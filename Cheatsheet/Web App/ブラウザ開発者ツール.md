# Debugger（Sourcesタブ）

JavaScriptのソースコード閲覧・動作確認に使用する。参考記事：🔗[ChromeのデベロッパーツールでJavaScriptをデバッグする方法 - ICS MEDIA](https://ics.media/entry/190517/)

## 基本機能

- JSのソースコードを閲覧できる
- `{}` を押すと読みやすい形式（整形済み）に変換される
- 変数にマウスカーソルを合わせると、その変数の現在値を確認できる
- 処理がどのように進んでいるかをステップ実行で追える

**XSS調査での活用：**

- 注入したXSSペイロードがどのようなcontextで処理されているかを把握できる
- ペイロードによって元の文法を壊していないかを確認するために使う
- 注入箇所にブレークポイントを設定し、Watchパネルに変数を追加することでcontextを把握できる

## ブレークポイントの使い方

ブレークポイントを設定すると、その処理箇所で一旦実行が止まり、青くハイライトされる。

**手順：**

1. 行番号をクリックしてブレークポイントを設定する
2. ⏸️（Pause script execution）を押して色がなくなることを確認する
3. プログラムを実行するとブレークポイントで止まる

**ステップ実行の各ボタン：**

|ボタン|名称|説明|
|---|---|---|
|▶️|リジューム実行|次のブレークポイントまで処理を飛ばす|
|↓|ステップイン|行ごとにプログラムを実行していく|
|↷|ステップオーバー|関数の内部処理をすべて実行し、次の行で止まる|
|↑|ステップアウト|現在の関数内の残りの処理をすべて実行し、呼び出し元の行へ移動する|
|→|ステップ|次の行へ進む|

## Scope / Watch パネル

|パネル|説明|
|---|---|
|Scope|現在実行中のJSのスコープ内の変数を閲覧できる。関数内のローカル変数はLocalツリーに表示される|
|Watch|`+` ボタンで変数名を登録すると、その変数の値をずっと追跡できる|

# Inspector（Elementsタブ）

ページ上の特定の場所をクリックすると、その箇所のHTML要素が表示される。

**主な用途：**

- ブラウザ上の表示・動作のシミュレーションを即時反映で確認する（実際のファイルへの変更は反映されない）
- hiddenフィールドの探索（session keyなどの機密情報が隠されている可能性がある）

## 検索の注意点

Elementsタブの検索はDOMとして構築された要素の「名前・属性・値・テキストノード」に対する**部分一致検索**であるため、HTMLの文字列そのものは検索できない。

|検索対象|ヒットするか|理由|
|---|---|---|
|`title`|✅|タグ名がDOMに存在する|
|`Sign In`|✅|テキストノードにマッチする|
|`title>Sign In`|❌|DOM構造にそのような文字列は存在しない|
|`<title>Sign In</title>`|❌|タグ構造の文字列はDOMには存在しない|
|`div`|✅|タグ名がDOMに存在する|
|`div id`|❌|属性込みの文字列はDOMには存在しない|

> [!NOTE]
> HTMLの文字列を検索したい場合は、`Ctrl + Shift + F` で開く **Search タブ** か、**Sources タブ** で検索すること。

# Network タブ

Proxyのような働きを持ち、リクエストとレスポンスの中身を閲覧できる。

レスポンスの `Server` などのヘッダー情報は重要な情報源になる。

---

# Application タブ

Storageの中からCookieの一覧を閲覧できる。

ログイン後にCookieのフラグ（`HttpOnly` / `Secure` など）を確認する用途で使う。
