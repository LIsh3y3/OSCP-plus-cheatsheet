# RDP接続

- 接続条件：ローカルでRemote Desktop UsersもしくはAdministratorsグループに所属しているユーザーの認証情報をもっていること
	- ドメインポリシーによっては、上記のグループに所属していなくても使える
```zsh
# ADの場合、/d:<domain>を追加
xfreerdp3 /dynamic-resolution +clipboard /cert:ignore /v:<target_IP> /u:<username> /p:'<password>'
```

- 代替手段としてのrdesktop
	- これは古いWindowsサーバーに対して接続性が良い場合がある
```sh
rdesktop -u <username> -p <password> -g 1280x720 -P -z <target_IP>
```
- `-g`: 解像度の指定
- `-P`: ビットマップキャッシュを有効化
- `-z`: 圧縮を有効化

# RDPの有効化

- 条件：ローカル管理者権限
- (WinRMで十分なため、必要性は低い)
```powershell
# RDPを有効化してファイアウォールルールも設定
Set-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' -Name 'fDenyTSConnections' -Value 0
Enable-NetFirewallRule -DisplayGroup 'Remote Desktop'

# 確認→0なら成功
Get-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' -Name 'fDenyTSConnections'
```
- →RDP接続（`/d`は不要）


>[!Warning]
>- 認証情報が有効であっても、RDPログイン権限がなければログインできない
>- NetExecでパスワードスプレーしてvalidだったとしても、ログインできるとは限らない

# トラブルシューティング

## Xfreerdpがうまく接続できない

- mtuを下げる（デフォルト1,500）
```zsh
sudo ifconfig tun0 mtu 1200
```

- `/etc/hosts` に FQDN の名前解決登録をし、ドメイン名を指定して接続

- RDPではなく、"RemoteAPP"の可能性があるため、`.rdp`ファイルを入手し、接続
```sh
xfreerdp <.rdp_file> /d:<domain> /u:<username>
```
- 🔗[リモートアプリ VS RDP: どう違う？ - AnyViewer](https://www.anyviewer.com/jp/how-to/remoteapp-vs-rdp-2578.html)
>[!TIP]
>RDWeb にアクセスして `.rdp` ファイルを入手するベクターがないか検討する

## 入力が英字配列のキーボードになってしまう→対策なし

- Xfreerdpからできる対策はなし
- マシンの環境では、ログオン後インターネット接続が使えないので、日本語packのインストールもできない
