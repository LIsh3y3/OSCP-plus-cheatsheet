# IMAPの基本

IMAP（Internet Message Access Protocol）は、サーバー上のメールを直接操作するためのプロトコル
- デフォルトポート：    
    - 143：==暗号化なし==
    - 993：SSL/TLS暗号化（IMAPS）
- メールの同期、フォルダ作成、サーバー上での検索などが可能
- POP3とは異なり、メールはサーバー上に保持されるため、複数のデバイスから同期して閲覧可能

>[!Warning]
>- 調査やエクスプロイトを行う際、サーバー上の既読・未読フラグなどの状態を変更してしまう可能性がある点に注意
>- IMAPコマンドは、行の先頭に「a1」「A1」などの<u>任意</u>のタグ（識別子）を付ける必要がある

---

# 接続方法

標準的な接続（Port 143）
```zsh
nc -nvC <target_IP> 143
# または
telnet <target_IP> 143
```

SSL/TLS接続（Port 993）
```zsh
openssl s_client -connect <target_IP>:993 -crlf -quiet
```

curlを使用したメール操作
```zsh
# メールボックス（フォルダ）の一覧表示
curl -u <username>:<password> imap://<target_IP>/

# INBOX内の全てのメールを検索
curl -k 'imaps://<target_IP>/INBOX?ALL' --user <user>:<pass>

# 特定のメール(Index:1)の本文を取得
curl -u <username>:<password> imap://<target_IP>/INBOX -X "FETCH 1 BODY[]"
```

# Recon

NSEを使った包括的な列挙
```zsh
sudo nmap -p 143,993 --script "imap-* or imap-ntlm-info" -sV <target_IP>
```
- `imap-ntlm-info.nse`：NTLM認証が有効な場合、WindowsのバージョンやNetBIOS名、ドメイン名などの内部情報を取得できる

サーバー情報の抽出(上記NSEで代替可)
```zsh
a1 AUTHENTICATE NTLM
# 応答：+
```
```zsh
TlRMTVNTUAABAAAAB4IIAAAAAAAAAAAAAAAAAAAAAAA=
# 応答：+ TlRMTVNTUAACAAAAC...（長い文字列）
```
- →応答をbase64でデコードすると、OSのバージョン、内部ホスト名、ドメイン名がわかる

IMAPサーバーの拡張機能確認（接続後）
```zsh
a1 CAPABILITY
```

| 出力                                                                | 意味                               | 解釈                                                                 |
| ----------------------------------------------------------------- | -------------------------------- | ------------------------------------------------------------------ |
| IMAP4rev1                                                         | IMAPの標準バージョン                     | 標準的なIMAP攻撃手法やツール（Hydra等）がそのまま通用する                                  |
| AUTH=PLAIN AUTH=LOGIN                                             | 平文認証のサポート                        | ・パスワードがBase64エンコードのみで流れる<br>・→[スニッフィング](#スニッフィング)(⚠️スタンドアロンでは使わない） |
| Pre-login capabilities listed, post-login capabilities have more. | ログイン前にできることはこれだけ、ログイン後にはもっと色々できる | -                                                                  |

# Exploit

## 認証情報のブルートフォース

方法A：[🐉THC-Hydra](../../Tools/🐉THC-Hydra.md#サービスに対する辞書攻撃)
```zsh
# 143
hydra -V -f -l <username> -P <password_wordlist> <target_IP> imap

# 993
hydra -V -f -S -l <username> -P <password_wordlist> -s 993 <target_IP> imaps
```

方法B：Nmapスクリプトによるブルートフォース
```zsh
nmap -p 143 --script imap-brute <target_IP>
```

## スニッフィング

スタンドアロンマシンに対しては使わない

パケットキャプチャ
	kaliのtun0を通過する、target_IPを送信先/受信とするport 143の通信をキャプチャ
```zsh
# ターミナルに出力
# pop3もopenであれば..and \( port 110 or port 143 \) -A とする
sudo tcpdump -i tun0 host <target_IP> and　port 143 -A

# wireshark用
sudo tcpdump -i tun0 host <target_IP> and port 143 -w imap.pcap
```
-  →[🦈Wireshark](../../Tools/🦈Wireshark.md)でキャプチャしたデータを開き、認証情報を特定する
	- フィルタ: `imap` または `tcp.port == 143`

# 侵入後の操作（Post-Exploitation）

サーバー内メールの検索でIMAPの検索機能を利用して、**機密情報を効率的に探す**
```zsh
# 接続・ログイン後
a1 LOGIN <username> <password>
a2 SELECT INBOX

# 本文に "password" を含むメールを検索
a3 SEARCH TEXT "password"

# 送信元が "admin" で、件名に "reset" を含むものを検索
a4 SEARCH FROM "admin" SUBJECT "reset"

# 2024年1月1日以降のメールを検索
a5 SEARCH SINCE 01-Jan-2024
```

メールの一括ダウンロード
```zsh
for i in {1..100}; do
  curl -u <username>:<password> "imap://<target_IP>/INBOX;MAILINDEX=$i" > email_$i.eml
done
```

# よく使うIMAPコマンド一覧

| マンド          | 説明                            | 使用例                                 |
| ------------ | ----------------------------- | ----------------------------------- |
| `CAPABILITY` | サーバーの機能を一覧表示                  | `a1 CAPABILITY`                     |
| `LOGIN`      | 認証（ユーザー名、パスワード）               | `a1 LOGIN user pass`                |
| `LIST`       | メールボックス（フォルダ）一覧を表示            | `a1 LIST "" "*"`                    |
| `SELECT`     | 操作するメールボックスを選択                | `a1 SELECT INBOX`                   |
| `STATUS`     | メールの件数、未読数などの状態を確認            | `a1 STATUS INBOX (MESSAGES UNSEEN)` |
| `SEARCH`     | 条件に合うメールを検索                   | `a1 SEARCH TEXT "keyword"`          |
| `FETCH`      | メールの内容を取得（BODY[TEXT]等で部分取得も可） | `a1 FETCH 1 BODY[]`                 |
| `CREATE`     | 新しいフォルダを作成                    | `a1 CREATE INBOX.Archive`           |
| `DELETE`     | フォルダを削除                       | `a1 DELETE INBOX.Old`               |
| `CLOSE`      | 選択中のメールボックスを閉じ、削除マーク付きメールを消去  | `a1 CLOSE`                          |
| `LOGOUT`     | セッションを終了してログアウト               | `a1 LOGOUT`                         |
